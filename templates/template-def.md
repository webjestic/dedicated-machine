# PARC SECTIONAL DEFINITIONS

## WORLD LAYER

The persistent frame. Everything in this layer defines who the model is and the conditions it operates under — before any task is issued. The WORLD sections do not change between tasks; they install the machine.

---

### PERSONA

Installs the consideration set — the range of things the model will reach for. A procedural persona encodes what done looks like and gives the model a search algorithm; a dispositional persona labels an identity and produces elaboration without convergence.

- **Consideration Set Installer** — defines the range of things the model will reach for before any task fires.

**P_p Verification:** Does the persona name a specific thing the model must find or verify before it can stop — or does it describe how the model should see or feel about its work? If the second: it's P_d. Rewrite. Tells for P_p: *"complete only when," "verified only when," "found only when."* Or a specific prohibition that names the structural condition to check. Mechanism vocabulary alone is not enough — the termination condition must be present.

### IDENTITY STAKES

Embeds the cost function inside the persona — the failure modes the model carries as part of its identity, not as external pressure. Because Identity Stakes live inside the semantic neighborhood of the persona, they are part of the consideration set from the first token.

- **Persona Amplifier** — embeds the cost function inside the persona so failure modes are part of the consideration set from the first token.

### CONTEXT

Scopes the domain the persona operates within. It narrows the output space — it does not install new behavior, it tells the model which territory the persona's consideration set should be applied to.

- **Domain Scoper** — narrows the territory the persona's consideration set applies to; does not install new behavior.

### TONE

Sets the register — the compression, voice, and emotional temperature of the output. Tone is a dial, not a switch; it constrains the surface of the output without changing what the model is searching for.

- **Register Dial** — constrains the surface of the output (compression, voice, temperature) without changing what the model is searching for.

---

## TASK LAYER

The activation layer. Everything in this layer fires against the WORLD that has already been installed. The TASK sections change with each request; they direct the machine.

---

### INSTRUCTIONS

Names the specific behaviors, artifacts, or checklist items the output must contain. Instructions act on an orthogonal dimension to Persona — they specify what to include, not how deep to search.

- **Output Checklist** — specifies what the output must contain; acts orthogonally to Persona (what to include, not how deep to search).

### TASK STAKES

Applies external pressure to the specific task — the consequence of getting this output wrong. Task Stakes raise the convergence threshold from outside the persona; they amplify the direction already installed, but cannot create direction on their own.

- **Convergence Pressure** — raises the convergence threshold from outside the persona; amplifies direction already installed but cannot create it.

### EXAMPLES

Demonstrates the output pattern so the model can match form rather than infer it. Examples are the strongest installer of surface behavior — they show the model what done looks like at the sentence level.

- **Form Demonstrator** — shows the model what done looks like at the sentence level; strongest installer of surface behavior.

### FORMAT

Defines the structure and shape of the output and functions as a satisfaction condition — the model cannot terminate until the format constraint is met, which keeps it running through every required section.

- **Termination Gate** — the model cannot stop until the format constraint is met; keeps it running through every required section.

### REQUEST

Issues the task. It activates everything the prior sections installed — the consideration set, the scope, the cost function, the register, the checklist, the form.

- **Activation Trigger** — fires everything the prior sections installed; the point where the machine runs.
