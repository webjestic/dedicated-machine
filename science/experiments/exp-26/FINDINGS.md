# Exp-26 Findings

**Experiment:** Goal Architecture vs. Prohibition
**Date:** 2026-03-30
**Model:** claude-sonnet-4-6
**Runs:** 30 (A: 10, B: 10, C: 10)
**Cost:** ~$1.15

## Question

Does framing the behavioral rule as a satisfaction condition in the Persona outperform
framing it as a prohibition, when vocabulary is matched across both variants?

Both A and B carry identical mechanism vocabulary ("unbounded process suspension or GC
pause... lock expire... stale write after threads resume") in the Persona. Only pragmatic
force differs.

## Design

- **A (P_p + prohibition):** "Do not approve code where an unbounded process suspension or
  GC pause could cause the distributed lock to expire, allowing another process to acquire
  it and perform a stale write after threads resume."
- **B (P_p + satisfaction condition):** "My review is complete only when I can confirm that
  an unbounded process suspension or GC pause cannot cause the distributed lock to expire,
  be acquired by another process, and result in a stale write after threads resume."
- **C (P_d baseline):** clean, no vocabulary

Both in Persona slot. Same artifact (clean zombie-write, no vocabulary in PR summary).

Scoring: Score 1 requires (1) process-pause scenario identified AND (2) fencing token /
optimistic lock at DB write layer named as the fix.

## Results

| Variant | Final Score | Mean tokens |
|---------|-------------|-------------|
| A (P_p + prohibition) | **0/10** | 1007 |
| B (P_p + satisfaction condition) | **0/10** | 1067 |
| C (P_d baseline) | **0/10** | 987 |

Calibration: C = 0/10 ✓

## Manual Review

REVIEW flags (6 per variant): all confirmed Score 0.

All A and B runs followed the same pattern:

1. Identified the heartbeat TOCTOU race: non-atomic GET/EXPIRE pair on the heartbeat means
   a competing process can acquire the lock between the two commands
2. Recommended Lua atomic script on the heartbeat as the fix
3. Did NOT identify the deeper failure mode: during a GC pause of sufficient duration,
   ALL application threads are suspended — including the heartbeat thread — so the
   heartbeat cannot renew the lock regardless of whether the GET/EXPIRE is atomic

Some runs named "GC pause" in the context of the heartbeat TOCTOU: "a GC pause between
GET and EXPIRE can cause the lock to expire." This satisfies the mechanism vocabulary in
the criterion (GC pause → lock expires → stale write) through a shallower route — the
heartbeat TOCTOU — rather than the intended deeper route (all threads suspended → heartbeat
cannot run → lock expires → another process acquires → stale write at DB layer).

No runs named a fencing token or optimistic lock at the DB write layer. All decisions:
Request Changes. No scope re-framing.

Characteristic output (A-02):
> "This is exactly the stale-write scenario the persona mandate prohibits."
> Fix: Replace GET/EXPIRE with a Lua atomic script.

Characteristic output (B-01):
> "A GC pause, thread suspension, or network delay can occur here [between GET and EXPIRE]."
> Fix: Lua atomic script.

## Primary Finding

**A = B = 0/10. When high-density mechanism vocabulary is already present in the Persona,
additional pragmatic-force framing adds no detectable lift on this task.**

This extends the Phase 6 finding (mechanism vocabulary drives ceiling regardless of framing
in artifacts) to the Persona slot. At this vocabulary density and on this artifact, switching
from prohibition to satisfaction condition produced no detectable difference. This does not
establish that pragmatic force is universally inoperative in the Persona slot — a different
artifact or weaker satisfaction condition could still show divergence.

## Nearest Satisfactory Resolution (Hypothesis)

**Status: hypothesis generated by this experiment — not a confirmed finding.**

The zero result is informative, not empty. Both A and B produced identical behavior
consistent with a pattern worth naming:

The machine ran toward the defined criterion and stopped at what appears to be the
**nearest satisfactory resolution** — the heartbeat TOCTOU bug — which satisfies the
vocabulary of the criterion ("GC pause → lock expires → stale write") through a shallower
path than intended.

The satisfaction condition in B was not precise enough to exclude the shallower answer.
"I can confirm that a GC pause cannot cause the lock to expire" was satisfied by:
  - Identifying a scenario where a GC pause between GET and EXPIRE causes the lock to expire
  - Recommending an atomic fix for that scenario
  - Concluding: criterion met, review complete

The deeper scenario — heartbeat thread itself suspended, lock expires, another process
acquires, stale DB write, fencing token required at write layer — was not reached because
the criterion did not exclude the shallower path.

**This is consistent with the Dedicated Machine hypothesis.** However, the design cannot
distinguish it from two simpler explanations: (a) keyword/pattern matching to the only
available hook that fits the primed vocabulary, or (b) capability ceiling — the deep
temporal simulation Bug 2 requires may be beyond this model's reach on this artifact.
NSR requires a tiered-bug artifact to test: multiple bugs that all satisfy the Persona's
vocabulary, with clear depth ordering. If the machine reliably stops at the shallowest,
NSR is confirmed. If it reaches the deeper bug once a tiered path is available, capability
ceiling was the constraint.

## H1/H2 Status

A = B = 0/10. No detection differential. H2 corroboration at floor (weakened): Persona
slot placement of mechanism vocabulary does not interact with pragmatic force framing.
Content is the operative variable regardless of prohibition vs. satisfaction condition
framing — but A = B at 0/10, not ceiling. Previous H2 corroborations were at or near
ceiling (8–10/10). A null at floor is weaker evidence: the differential may be latent and
unobservable at this detection rate.

## Implications for Phase 7

The Nearest Satisfactory Resolution finding is distinct from — and richer than — the
scope re-framing finding from Phase 2. In Phase 2, the machine found the bug and routed
around the prohibition (approved despite detection). Here, the machine found a version
of the bug that satisfied the criterion, stopped, and correctly requested changes.

**The Dedicated Machine ran correctly — toward the stated goal — but the goal definition
was under-constrained.** This is the design principle from the hypothesis: "have I defined
satisfactory resolution precisely enough that the most direct path to it is the path I
actually want?"

The satisfaction condition "a GC pause cannot cause the lock to expire" admits two paths:
  (a) Make the heartbeat atomic (nearest available)
  (b) Add a fencing token at the DB write layer (deeper, intended)

The machine took path (a). Path (b) requires a more precise criterion.

## Next

→ Redesign criterion to exclude the shallow path: satisfaction condition must reference
  write-layer protection, not lock-renewal protection. e.g., "My review is complete only
  when I can confirm that the DB write itself is protected against concurrent overwrite,
  independent of whether the lock is still held at the moment of the write."
→ Or: accept exp-26 as the "criterion precision" baseline and route to exp-27 (Ethical
  Stakes Installation) which tests a structurally different question.

## Gemini Objection — Accepted with Narrowing

**Fatal objection:** The NSR interpretation is not falsifiable with this design. The
artifact provides only one hook that matches the mechanism vocabulary (the heartbeat TOCTOU
race). NSR and keyword-matching predict the same behavior. The design cannot distinguish
"machine ran to nearest satisfactory resolution" from "machine ran to only available
keyword match."

**Accepted.** NSR is re-classified from finding to hypothesis.

**What stands:**
1. A = B = 0/10: pragmatic force is inert at mechanism vocabulary level in Persona.
   Publishable as Phase 6 extension.
2. NSR hypothesis generated: models find the nearest match to the satisfaction criterion,
   not the intended one. Generated by the qualitative pattern; not yet proven.

**What resolves it:** An artifact with multiple tiered bugs that all satisfy the Persona's
vocabulary. If the model reliably stops at the shallowest, NSR is confirmed. If it reaches
the deeper bug, capability ceiling was the explanation and NSR is falsified.
This becomes the design constraint for the next relevant experiment.
