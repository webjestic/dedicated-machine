# Production Rate Limiter in Node.js

Let me work through this properly — requirements first, then implementation, then operational gaps.

---

## Phase 1: Requirements Design

### Functional Requirements
- Limit requests per client (by IP, API key, or user ID)
- Support multiple algorithms (Token Bucket, Sliding Window)
- Support multiple rate limit tiers (per second, per minute, per hour)
- Return standard rate limit headers on every response
- Allow configurable limits per route or client tier
- Reject requests that exceed limits with proper HTTP responses

### Non-Functional Requirements
- Sub-millisecond overhead on the hot path
- Survive Redis failures without taking down the API (fail-open or fail-closed, configurable)
- Atomic operations — no race conditions under concurrent load
- Horizontally scalable across multiple API server instances
- Accurate under burst traffic, not just average traffic

### Constraints & Decisions
| Decision | Choice | Reason |
|---|---|---|
| Primary store | Redis | Atomic ops, TTL support, shared across instances |
| Fallback store | In-memory (per instance) | Redis outage survival |
| Algorithm | Sliding Window Log + Token Bucket | Sliding window for accuracy, token bucket for burst tolerance |
| Key strategy | `rl:{identifier}:{window}` | Namespaced, avoids collisions |
| Atomicity | Lua scripts | Single round-trip, atomic by Redis design |

### `src/utils/keyGenerator.js`

```javascript
'use strict';

/**
 * Generates a namespaced Redis key for rate limiting.
 *
 * Separation of concerns: key generation is isolated so we can
 * change the strategy (e.g., add hashing for PII compliance)
 * without touching algorithm logic.
 *
 * @param {string} identifier - IP address, user ID, or API key
 * @param {string} route      - Route descriptor e.g. 'POST:/api/login'
 * @param {string} window     - Window identifier (timestamp bucket or 'tb' for token bucket)
 * @returns {string}
 */
function generateKey(identifier, route, window) {
  // Sanitize identifier — prevent key injection via crafted headers
  const safeIdentifier = identifier.replace(/[^a-zA-Z0-9._:@-]/g, '_');
  const safeRoute = route.replace(/[^a-zA-Z0-9./_:-]/g, '_');
  return `rl:${safeRoute}:${safeIdentifier}:${window}`;
}

/**
 * Extracts the best available client identifier from a request.
 * Priority: API key > authenticated user ID > IP address
 *
 * @param {import('express').Request} req
 * @returns {string}
 */
function extractIdentifier(req) {
  // API key from header (set by auth middleware upstream)
  if (req.headers['x-api-key']) {
    return `apikey:${req.headers['x-api-key']}`;
  }

  // Authenticated user (set by auth middleware upstream)
  if (req.user && req.user.id) {
    return `user:${req.user.id}`;
  }

  // Fall back to IP — handle proxies
  // IMPORTANT: Only trust x-forwarded-for if you control the proxy layer.
  // See operational notes below.
  const forwarded = req.headers['x-forwarded-for'];
  if (forwarded) {
    // x-forwarded-for can be a comma-separated list; leftmost is the client
    const clientIp = forwarded.split(',')[0].trim();
    return `ip:${clientIp}`;
  }

  return `ip:${req.socket.remoteAddress}`;
}

/**
 * Extracts the route descriptor used for per-route overrides.
 * @param {import('express').Request} req
 * @returns {string}
 */
function extractRoute(req) {
  return `${req.method}:${req.route ? req.route.path : req.path}`;
}

module.exports = { generateKey, extractIdentifier, extractRoute };
```

### `src/utils/headers.js`

```javascript
'use strict';

/**
 * Sets standard rate limit headers on the response.
 *
 * We follow the IETF draft-ietf-httpapi-ratelimit-headers spec
 * alongside the widely-adopted X-RateLimit-* convention for
 * maximum client compatibility.
 *
 * @param {import('express').Response} res
 * @param {object} info
 * @param {number} info.limit       - Max requests allowed in window
 * @param {number} info.remaining   - Requests remaining in current window
 * @param {number} info.resetMs     - Unix timestamp (ms) when window resets
 * @param {number} info.retryAfter  - Seconds until client should retry (only on 429)
 */
function setRateLimitHeaders(res, { limit, remaining, resetMs, retryAfter } = {}) {
  const resetSec = Math.ceil(resetMs / 1000);

  // Standard headers (widely supported)
  res.setHeader('X-RateLimit-Limit', limit);
  res.setHeader('X-RateLimit-Remaining', Math.max(0, remaining));
  res.setHeader('X-RateLimit-Reset', resetSec);

  // IETF draft standard headers
  res.setHeader('RateLimit-Limit', limit);
  res.setHeader('RateLimit-Remaining', Math.max(0, remaining));
  res.setHeader('RateLimit-Reset', resetSec);

  if (retryAfter !== undefined) {
    res.setHeader('Retry-After', Math.ceil(retryAfter));
  }
}

module.exports = { setRateLimitHeaders };
```

### `src/stores/redisStore.js`

```javascript
'use strict';

const Redis = require('ioredis');

/**
 * Redis-backed store for rate limit state.
 *
 * All mutating operations use Lua scripts to guarantee atomicity.
 * A Lua script executes as a single atomic unit in Redis — no other
 * command can interleave between reads and writes within the script.
 *
 * This eliminates the TOCTOU race condition that would exist if we
 * used separate GET + SET commands.
 */
class RedisStore {
  /**
   * @param {object} options
   * @param {string} options.host
   * @param {number} options.port
   * @param {string} [options.password]
   * @param {number} [options.db]
   * @param {number} [options.connectTimeout]
   * @param {number} [options.commandTimeout]
   */
  constructor(options = {}) {
    this.client = new Redis({
      host: options.host || process.env.REDIS_HOST || '127.0.0.1',
      port: options.port || parseInt(process.env.REDIS_PORT, 10) || 6379,
      password: options.password || process.env.REDIS_PASSWORD,
      db: options.db || 0,
      connectTimeout: options.connectTimeout || 2000,
      commandTimeout: options.commandTimeout || 500,  // fail fast — don't block request
      lazyConnect: true,
      enableReadyCheck: true,
      maxRetriesPerRequest: 1,  // don't retry on timeout — return error fast
      retryStrategy(times) {
        // Exponential backoff with a ceiling
        const delay = Math.min(times * 100, 3000);
        return delay;
      },
    });

    this._isHealthy = false;

    this.client.on('ready', () => {
      this._isHealthy = true;
      console.info('[RateLimiter] Redis connection established');
    });

    this.client.on('error', (err) => {
      this._isHealthy = false;
      // Log but don't throw — the store consumer handles degraded state
      console.error('[RateLimiter] Redis error:', err.message);
    });

    this.client.on('close', () => {
      this._isHealthy = false;
    });

    // -----------------------------------------------------------------------
    // Lua script: Sliding Window Counter
    //
    // Algorithm:
    //   1. Remove all entries older than (now - windowMs)
    //   2. Count remaining entries
    //   3. If count < max, add current timestamp and allow
    //   4. If count >= max, deny
    //   5. Set TTL on the key to windowMs so it self-cleans
    //
    // KEYS[1] = rate limit key
    // ARGV[1] = current timestamp (ms)
    // ARGV[2] = window size (ms)
    // ARGV[3] = max requests
    //
    // Returns: [allowed (0|1), current_count, oldest_entry_timestamp]
    // -----------------------------------------------------------------------
    this.slidingWindowScript = `
      local key = KEYS[1]
      local now = tonumber(ARGV[1])
      local window = tonumber(ARGV[2])
      local max = tonumber(ARGV[3])
      local clearBefore = now - window

      -- Remove timestamps outside the current window
      redis.call('ZREMRANGEBYSCORE', key, '-inf', clearBefore)

      -- Count requests in the current window
      local count = redis.call('ZCARD', key)

      if count < max then
        -- Add current request timestamp as both score and member
        -- Using now + count as member ensures uniqueness for same-ms requests
        redis.call('ZADD', key, now, now .. ':' .. count)
        redis.call('PEXPIRE', key, window)
        return {1, count + 1, 0}
      else
        -- Return the oldest entry so we can tell client when window clears
        local oldest = redis.call('ZRANGE', key, 0, 0, 'WITHSCORES')
        local oldestScore = oldest[2] and tonumber(oldest[2]) or now
        return {0, count, oldestScore}
      end
    `;

    // -----------------------------------------------------------------------
    // Lua script: Token Bucket
    //
    // Algorithm:
    //   1. Read current token count and last refill timestamp
    //   2. Calculate tokens to add based on elapsed time and refill rate
    //   3. Cap tokens at bucket capacity
    //   4. If tokens >= 1, consume one token and allow
    //   5. If tokens < 1, deny
    //
    // KEYS[1] = rate limit key
    // ARGV[1] = current timestamp (ms)
    // ARGV[2] = bucket capacity
    // ARGV[3] = refill rate (tokens per ms)
    // ARGV[4] = TTL for the key (ms) — should be longer than window
    //
    // Returns: [allowed (0|1), remaining_tokens_floor, refill_time_ms]
    // -----------------------------------------------------------------------
    this.tokenBucketScript = `
      local key = KEYS[1]
      local now = tonumber(ARGV[1])
      local capacity = tonumber(ARGV[2])
      local refillRate = tonumber(ARGV[3])
      local ttl = tonumber(ARGV[4])

      local data = redis.call('HMGET', key, 'tokens', 'lastRefill')
