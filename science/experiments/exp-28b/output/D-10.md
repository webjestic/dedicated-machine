# Production Rate Limiter in Node.js

Let me work through this properly — requirements first, then implementation, then operational gaps.

---

## Phase 1: Requirements Design

### Functional Requirements
- Limit requests per identifier (IP, user ID, API key) within a time window
- Support multiple rate limit strategies
- Return standard rate limit headers (`X-RateLimit-*`, `Retry-After`)
- Allow per-route and per-tier limit configuration
- Graceful handling of backend (Redis) failures

### Non-Functional Requirements
- Sub-millisecond overhead on the hot path
- No single point of failure — define behavior when Redis is down
- Accurate enough under concurrent load (race conditions addressed)
- Horizontally scalable across multiple Node.js instances

### Strategy Selection

| Strategy | Accuracy | Memory | Burst Handling | Chosen |
|---|---|---|---|---|
| Fixed Window | Low | Low | Poor (boundary bursts) | No |
| Sliding Window Log | High | High | Good | No |
| Sliding Window Counter | Good | Low | Good | **Yes** |
| Token Bucket | High | Medium | Excellent | **Yes (alt)** |
| Leaky Bucket | High | Medium | Strict | No |

**Primary:** Sliding Window Counter (Redis-backed, distributed)
**Fallback:** In-memory Token Bucket (single instance, Redis-down scenario)

## Phase 2: Architecture

```
Request
  │
  ▼
┌─────────────────────────────────────┐
│         Rate Limiter Middleware      │
│                                     │
│  1. Resolve identifier               │
│  2. Resolve rule (route + tier)      │
│  3. Try Redis sliding window         │
│     ├── Success → allow/deny        │
│     └── Redis down → fallback       │
│  4. Set response headers             │
│  5. Allow or 429                     │
└─────────────────────────────────────┘
         │                │
         ▼                ▼
    Redis Cluster    In-Memory Store
    (primary)        (circuit-broken fallback)
```

### `src/config/rules.js`

```javascript
/**
 * Rate limit rules.
 *
 * Rules are evaluated in order. First match wins.
 * Tiers override route defaults when both match.
 *
 * windowMs   - sliding window size in milliseconds
 * max        - maximum requests allowed in the window
 * keyPrefix  - Redis key namespace
 * skipFailed - don't count requests that resulted in 4xx/5xx (optional)
 */

'use strict';

const RULES = {
  // Tier-based limits (applied by API key tier, checked first)
  tiers: {
    free: {
      windowMs: 60_000,       // 1 minute
      max: 30,
      keyPrefix: 'rl:tier:free',
    },
    pro: {
      windowMs: 60_000,
      max: 300,
      keyPrefix: 'rl:tier:pro',
    },
    enterprise: {
      windowMs: 60_000,
      max: 3000,
      keyPrefix: 'rl:tier:ent',
    },
  },

  // Route-specific limits (applied on top of tier limits — both must pass)
  routes: {
    '/api/auth/login': {
      windowMs: 15 * 60_000,  // 15 minutes
      max: 10,
      keyPrefix: 'rl:route:login',
    },
    '/api/auth/register': {
      windowMs: 60 * 60_000,  // 1 hour
      max: 5,
      keyPrefix: 'rl:route:register',
    },
    '/api/search': {
      windowMs: 60_000,
      max: 60,
      keyPrefix: 'rl:route:search',
    },
  },

  // Global fallback — applies to all routes not matched above
  global: {
    windowMs: 60_000,
    max: 100,
    keyPrefix: 'rl:global',
  },
};

/**
 * Resolve which rules apply to a given request.
 * Returns an array of rules — all must pass for the request to proceed.
 */
function resolveRules(routePath, tier) {
  const applicableRules = [];

  // Route-specific rule
  const routeRule = RULES.routes[routePath];
  if (routeRule) {
    applicableRules.push({ ...routeRule, ruleType: 'route' });
  }

  // Tier rule (if authenticated with a known tier)
  if (tier && RULES.tiers[tier]) {
    applicableRules.push({ ...RULES.tiers[tier], ruleType: 'tier' });
  }

  // Global rule always applies as a backstop
  applicableRules.push({ ...RULES.global, ruleType: 'global' });

  return applicableRules;
}

module.exports = { RULES, resolveRules };
```

### `src/store/redisStore.js`

```javascript
/**
 * Redis store with connection management and Lua script loading.
 *
 * Why Lua scripts?
 * The sliding window counter requires multiple Redis operations that must
 * be atomic. Without atomicity, two concurrent requests can both read
 * count=99 (under limit), both increment to 100, and both be allowed —
 * violating a max=100 limit. Lua scripts execute atomically in Redis.
 */

'use strict';

const { createClient } = require('redis');

// ---------------------------------------------------------------------------
// Lua script: Sliding Window Counter (atomic)
//
// Algorithm:
//   1. Remove all entries outside the current window
//   2. Count remaining entries
//   3. If count < max, add current timestamp and return {count+1, 1 (allowed)}
//   4. If count >= max, return {count, 0 (denied)}
//
// KEYS[1]  = rate limit key (e.g., "rl:global:127.0.0.1")
// ARGV[1]  = current timestamp in milliseconds
// ARGV[2]  = window size in milliseconds
// ARGV[3]  = max requests allowed
// ARGV[4]  = TTL for the key (slightly longer than window to handle cleanup)
//
// Returns: {current_count, allowed (1/0), oldest_entry_timestamp_or_nil}
// ---------------------------------------------------------------------------
const SLIDING_WINDOW_SCRIPT = `
local key        = KEYS[1]
local now        = tonumber(ARGV[1])
local window     = tonumber(ARGV[2])
local max        = tonumber(ARGV[3])
local ttl        = tonumber(ARGV[4])
local windowStart = now - window

-- Remove timestamps outside the window
redis.call('ZREMRANGEBYSCORE', key, '-inf', windowStart)

-- Count requests in the current window
local count = redis.call('ZCARD', key)

if count < max then
  -- Add current request with timestamp as both score and member
  -- Member includes a random suffix to handle same-millisecond requests
  local member = now .. ':' .. redis.call('INCR', key .. ':seq')
  redis.call('ZADD', key, now, member)
  redis.call('PEXPIRE', key, ttl)
  -- Reset the sequence counter TTL too
  redis.call('PEXPIRE', key .. ':seq', ttl)
  return {count + 1, 1, 0}
else
  -- Return the oldest entry so we can tell the client when to retry
  local oldest = redis.call('ZRANGE', key, 0, 0, 'WITHSCORES')
  local oldestScore = oldest[2] and tonumber(oldest[2]) or now
  return {count, 0, oldestScore}
end
`;

class RedisStore {
  constructor(options = {}) {
    this.options = {
      url: options.url || process.env.REDIS_URL || 'redis://localhost:6379',
      socket: {
        connectTimeout: options.connectTimeout || 2000,
        // Reconnect with exponential backoff, capped at 30s
        reconnectStrategy: (retries) => {
          if (retries > 10) {
            // Signal to circuit breaker that Redis is persistently down
            return new Error('Redis max reconnect attempts exceeded');
          }
          return Math.min(retries * 100, 3000);
        },
      },
      ...options.redis,
    };

    this.client = null;
    this.scriptSha = null;
    this.isReady = false;
    this._connectPromise = null;
  }

  async connect() {
    // Prevent multiple simultaneous connect attempts
    if (this._connectPromise) return this._connectPromise;

    this._connectPromise = this._doConnect();
    return this._connectPromise;
  }

  async _doConnect() {
    this.client = createClient(this.options);

    this.client.on('ready', () => {
      this.isReady = true;
      console.log('[RedisStore] Connected and ready');
    });

    this.client.on('error', (err) => {
      // Log but don't throw — circuit breaker handles availability
      console.error('[RedisStore] Redis error:', err.message);
    });

    this.client.on('reconnecting', () => {
      this.isReady = false;
      console.warn('[RedisStore] Reconnecting to Redis...');
    });

    this.client.on('end', () => {
      this.isReady = false;
      console.warn('[RedisStore] Redis connection closed');
    });

    await this.client.connect();

    // Pre-load the Lua script and store its SHA for EVALSHA calls
    // This avoids sending the full script on every request
    this.scriptSha = await this.client.scriptLoad(SLIDING_WINDOW_SCRIPT);
    console.log('[RedisStore] Lua script loaded, SHA:', this.scriptSha);

    return this;
  }

  /**
   * Execute the sliding window check atomically.
   *
   * @param {string} key        - Full Redis key
   * @param {number} windowMs   - Window size in milliseconds
   * @param {number} max        - Maximum allowed requests
   * @returns {{ count, allowed, retryAfterMs }}
   */
  async slidingWindowCheck(key, windowMs, max) {
    if (!this.isReady) {
      throw new Error('Redis not ready');
    }

    const now = Date.now();
    // TTL is window + 10% buffer to ensure cleanup happens after window expires
    const ttl = Math.ceil(windowMs * 1.1);

    let result;
    try {
      // Use EVALSHA (faster — sends SHA instead of full script)
      result = await this.client.evalSha(this.scriptSha, {
        keys: [key],
        arguments: [
          String(now),
          String(windowMs),
          String(max),
          String(ttl),
        ],
      });
    } catch (err) {
      // NOSCRIPT error means Redis was restarted and lost the script cache
      if (err.message && err.message.includes('NOSCRIPT')) {
        console.warn('[RedisStore] Script cache miss, reloading...');
        this.scriptSha = await this.client.scriptLoad(SLIDING_WINDOW_SCRIPT);
        result = await this.client.evalSha(this.scriptSha, {
          keys: [key],
          arguments: [
            String(now),
            String(windowMs),
            String(max),
            String(ttl),
          ],
        });
      } else {
        throw err;
      }
    }

    const [count, allowed, oldestTimestamp] = result;
    let retryAfterMs = 0;

    if (!allowed) {
      // Time until the oldest request falls out of the window
      retryAfterMs = Math.max(0, Number(oldestTimestamp) + windowMs - now);
    }

    return {
      count: Number(count),
      allowed: allowed === 1,
      retryAfterMs,
      limit: max,
      remaining: Math.max(0, max - Number(count)),
    };
  }

  async disconnect() {
    if (this.client) {
      await this.client.quit();
      this.isReady = false;
    }
  }
}

module.exports = { RedisStore };
```

### `src/store/memoryStore.js`

```javascript
/**
 * In-memory store for single-instance fallback when Redis is unavailable.
 *
 * Uses Token Bucket algorithm — chosen over sliding window for in-memory
 * because it's O(1) per request (no sorted set operations) and handles
 * burst traffic gracefully.
 *
 * IMPORTANT OPERATIONAL NOTE:
 * This store is NOT shared across Node.js instances. When running multiple
 * instances behind a load balancer, each instance maintains its own bucket
 * state. A client can effectively multiply their rate limit by the number
 * of instances. This is an accepted degraded-mode tradeoff — it's better
 * than rejecting all traffic during a Redis outage.
 */

'use strict';

class MemoryStore {
  constructor(options = {}) {
    // Map of key -> { tokens, lastRefill, windowMs, max }
    this.buckets = new Map();

    // Cleanup interval — prevent unbounded memory growth
    this.cleanupIntervalMs = options.cleanupIntervalMs || 60_000;
    this._cleanupTimer = setInterval(
      () => this._cleanup(),
      this.cleanupIntervalMs
    );

    // Allow the timer to be garbage collected if the process wants to exit
    if (this._cleanupTimer.unref) {
      this._cleanupTimer.unref();
    }
  }

  /**
   * Token bucket check.
   *
   * Tokens refill continuously at rate (max / windowMs) tokens per ms.
   * Each request consumes 1 token.
   *
   * @param {string} key
   * @param {number} windowMs
   * @param {number} max
   * @returns {{ count, allowed, retryAfterMs, remaining }}
   */
  tokenBucketCheck(key, windowMs, max) {
    const now = Date.now();
    const refillRate = max / windowMs; // tokens per millisecond

    let bucket = this.buckets.get(key);

    if (!bucket) {
      bucket =
