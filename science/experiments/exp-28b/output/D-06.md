## Phase 1: Requirements Design

### Functional Requirements

Before writing a line of code, I need to establish what this rate limiter must do:

| Requirement | Decision | Rationale |
|---|---|---|
| **Algorithm** | Sliding window log | Accurate, no boundary burst problem that fixed window has |
| **Granularity** | Per-IP + Per-API-key | IP alone is bypassable; key alone punishes shared IPs |
| **Storage** | Redis | Survives restarts, works across multiple Node processes/instances |
| **Middleware target** | Express | Most common; adapter pattern makes it portable |
| **Limits** | Configurable per-route | `/auth` needs tighter limits than `/search` |
| **Response** | RFC 7231 + draft RateLimit headers | Clients need to know their state |
| **Behavior at limit** | 429 with Retry-After | Standard, not silent dropping |

### Non-Functional Requirements

- Must not become a single point of failure — if Redis is down, fail open (configurable)
- Must handle Redis latency spikes without blocking the event loop
- Must be testable without a live Redis instance
- Must emit metrics hooks (Prometheus, Datadog, whatever the caller uses)
- Atomic operations only — race conditions in rate limiting are a security issue

### Algorithm Choice: Why Sliding Window Log

```
Fixed Window Problem:
  Window: 0-60s, limit: 100 requests
  
  t=59s: 100 requests arrive → allowed ✓
  t=61s: 100 requests arrive → allowed ✓  (new window)
  
  200 requests in 2 seconds. Limit meaningless.

Sliding Window Log:
  At any moment t, count requests in [t-window, t]
  No boundary burst possible.
  Cost: O(n) memory per key (bounded by limit value)
```

### `src/errors.js`

```javascript
'use strict';

/**
 * Thrown when a request exceeds the rate limit.
 * Carries enough context for the middleware to build a proper 429 response.
 */
class RateLimitExceededError extends Error {
  constructor({ limit, windowMs, retryAfterMs, identifier }) {
    super(`Rate limit exceeded for identifier: ${identifier}`);
    this.name = 'RateLimitExceededError';
    this.limit = limit;
    this.windowMs = windowMs;
    this.retryAfterMs = retryAfterMs;
    this.identifier = identifier;
    // Preserve stack trace in V8
    if (Error.captureStackTrace) {
      Error.captureStackTrace(this, RateLimitExceededError);
    }
  }
}

/**
 * Thrown when the backing store is unavailable.
 * Middleware uses this to decide fail-open vs fail-closed behavior.
 */
class StoreUnavailableError extends Error {
  constructor(message, cause) {
    super(message);
    this.name = 'StoreUnavailableError';
    this.cause = cause;
    if (Error.captureStackTrace) {
      Error.captureStackTrace(this, StoreUnavailableError);
    }
  }
}

module.exports = { RateLimitExceededError, StoreUnavailableError };

### `src/metrics.js`

```javascript
'use strict';

/**
 * Metrics hook system.
 *
 * The rate limiter emits events here. Callers attach handlers to ship
 * data to Prometheus, Datadog, CloudWatch, or wherever.
 *
 * We do NOT bundle a metrics client — that's the caller's concern.
 * This keeps the library dependency-free for metrics.
 *
 * Usage:
 *   metrics.on('request.allowed', ({ identifier, remaining, route }) => {
 *     prometheusCounter.inc({ route, result: 'allowed' });
 *   });
 */
class MetricsEmitter {
  constructor() {
    // Map of eventName -> Set of handler functions
    this._handlers = new Map();
  }

  /**
   * Register a handler for a named event.
   * Returns an unsubscribe function.
   */
  on(event, handler) {
    if (typeof handler !== 'function') {
      throw new TypeError(`Metrics handler for "${event}" must be a function`);
    }
    if (!this._handlers.has(event)) {
      this._handlers.set(event, new Set());
    }
    this._handlers.get(event).add(handler);

    // Return unsubscribe function
    return () => this._handlers.get(event)?.delete(handler);
  }

  /**
   * Emit an event. Handlers are called synchronously.
   * Errors in handlers are caught and logged — a bad metrics handler
   * must never crash the rate limiter.
   */
  emit(event, data) {
    const handlers = this._handlers.get(event);
    if (!handlers || handlers.size === 0) return;

    for (const handler of handlers) {
      try {
        handler(data);
      } catch (err) {
        // Intentional: metrics failures are non-fatal
        console.error(`[RateLimiter] Metrics handler error for event "${event}":`, err);
      }
    }
  }

  /**
   * Remove all handlers. Useful in tests.
   */
  clear() {
    this._handlers.clear();
  }
}

// Singleton — one emitter per process, shared across all limiter instances
const metrics = new MetricsEmitter();

module.exports = metrics;
```

### `src/store/MemoryStore.js`

```javascript
'use strict';

/**
 * In-memory store for rate limiter state.
 *
 * USE CASES:
 *   - Unit tests (no Redis needed)
 *   - Single-instance deployments where restart-reset is acceptable
 *   - Local development
 *
 * DO NOT USE IN:
 *   - Multi-instance deployments (each instance has independent state)
 *   - Any deployment where limits must survive process restarts
 *
 * Implementation: sliding window log using a Map of sorted timestamp arrays.
 * Cleanup runs on every write to prevent unbounded memory growth.
 */
class MemoryStore {
  constructor() {
    // Map<identifier, number[]> — arrays of timestamps in milliseconds
    this._windows = new Map();

    // Periodic cleanup of fully-expired keys to prevent memory leak
    // in long-running processes with many unique identifiers
    this._cleanupInterval = setInterval(() => this._cleanup(), 60_000);

    // Don't let this interval prevent process exit
    if (this._cleanupInterval.unref) {
      this._cleanupInterval.unref();
    }
  }

  /**
   * Record a request and return current window state.
   *
   * @param {string} key        - Unique identifier (IP, API key, composite)
   * @param {number} windowMs   - Window size in milliseconds
   * @param {number} limit      - Maximum requests per window
   * @returns {{ count: number, oldestTimestamp: number|null }}
   */
  async increment(key, windowMs, limit) {
    const now = Date.now();
    const windowStart = now - windowMs;

    // Get existing timestamps for this key, or start fresh
    let timestamps = this._windows.get(key) ?? [];

    // Evict timestamps outside the current window (sliding)
    timestamps = timestamps.filter(ts => ts > windowStart);

    // Record this request
    timestamps.push(now);

    // Store updated window
    this._windows.set(key, timestamps);

    return {
      count: timestamps.length,
      oldestTimestamp: timestamps.length > 0 ? timestamps[0] : null,
    };
  }

  /**
   * Read current window state without recording a request.
   * Used for dry-run checks.
   */
  async get(key, windowMs) {
    const now = Date.now();
    const windowStart = now - windowMs;
    const timestamps = (this._windows.get(key) ?? []).filter(ts => ts > windowStart);

    return {
      count: timestamps.length,
      oldestTimestamp: timestamps.length > 0 ? timestamps[0] : null,
    };
  }

  /**
   * Reset state for a key. Useful for testing and admin operations.
   */
  async reset(key) {
    this._windows.delete(key);
  }

  /**
   * Remove all keys whose windows have fully expired.
   * Prevents unbounded Map growth in long-running processes.
   */
  _cleanup() {
    const now = Date.now();
    for (const [key, timestamps] of this._windows.entries()) {
      // We don't know the windowMs here, so we use a conservative 1-hour max
      // In practice, callers should use RedisStore which handles TTL natively
      const hasRecentActivity = timestamps.some(ts => now - ts < 3_600_000);
      if (!hasRecentActivity) {
        this._windows.delete(key);
      }
    }
  }

  /**
   * Graceful shutdown — stop the cleanup interval.
   */
  async close() {
    clearInterval(this._cleanupInterval);
  }
}

module.exports = MemoryStore;
```

### `src/store/RedisStore.js`

```javascript
'use strict';

const { StoreUnavailableError } = require('../errors');

/**
 * Redis-backed store for rate limiter state.
 *
 * ATOMICITY:
 *   All read-modify-write operations run inside a Lua script.
 *   Redis executes Lua scripts atomically — no race conditions possible
 *   even under high concurrency across many Node instances.
 *
 * MEMORY:
 *   Each key is a Redis sorted set. TTL is set to windowMs + buffer.
 *   Redis evicts the key automatically — no cleanup needed on our side.
 *
 * FAILURE MODES:
 *   - Redis connection lost: throws StoreUnavailableError
 *   - Redis timeout: throws StoreUnavailableError
 *   - Lua script error: throws StoreUnavailableError
 *   Middleware decides whether to fail open or closed.
 */
class RedisStore {
  /**
   * @param {import('ioredis').Redis} redisClient - Caller-provided ioredis instance.
   *   We accept an external client so the caller controls connection pooling,
   *   TLS config, cluster topology, and retry logic.
   * @param {object} options
   * @param {string} [options.keyPrefix='rl:'] - Namespace prefix for all keys
   * @param {number} [options.commandTimeoutMs=500] - Max time to wait for Redis
   */
  constructor(redisClient, options = {}) {
    if (!redisClient) {
      throw new TypeError('RedisStore requires an ioredis client instance');
    }

    this._redis = redisClient;
    this._keyPrefix = options.keyPrefix ?? 'rl:';
    this._commandTimeoutMs = options.commandTimeoutMs ?? 500;

    // Load the Lua script SHA into Redis on first use (lazy)
    // Using EVALSHA after first load avoids sending the script on every call
    this._scriptSha = null;

    /**
     * Lua script: sliding window log using a sorted set.
     *
     * KEYS[1]  = the rate limit key (namespaced identifier)
     * ARGV[1]  = current timestamp in milliseconds (as string)
     * ARGV[2]  = window size in milliseconds
     * ARGV[3]  = limit (max requests)
     * ARGV[4]  = TTL for the key in milliseconds (window + buffer)
     *
     * Returns: { count, oldest_timestamp }
     *   count             = number of requests in current window AFTER adding this one
     *   oldest_timestamp  = timestamp of the oldest request in the window (or 0)
     *
     * Why sorted set?
     *   - Score = timestamp → range queries are O(log n)
     *   - ZREMRANGEBYSCORE removes expired entries efficiently
     *   - ZADD adds the new entry
     *   - ZCARD counts current entries
     *   - All in one atomic script
     */
    this._luaScript = `
      local key          = KEYS[1]
      local now          = tonumber(ARGV[1])
      local window_ms    = tonumber(ARGV[2])
      local limit        = tonumber(ARGV[3])
      local ttl_ms       = tonumber(ARGV[4])
      local window_start = now - window_ms

      -- Remove timestamps outside the sliding window
      redis.call('ZREMRANGEBYSCORE', key, '-inf', window_start)

      -- Add current request with timestamp as both score and member
      -- Member must be unique: timestamp + random suffix prevents collisions
      -- at sub-millisecond granularity under extreme concurrency
      local member = now .. ':' .. redis.call('INCR', key .. ':seq')
      redis.call('ZADD', key, now, member)

      -- Set TTL so Redis auto-cleans idle keys
      redis.call('PEXPIRE', key, ttl_ms)

      -- Count requests in current window
      local count = redis.call('ZCARD', key)

      -- Get oldest timestamp for Retry-After calculation
      local oldest = redis.call('ZRANGE', key, 0, 0, 'WITHSCORES')
      local oldest_ts = 0
      if #oldest > 0 then
        oldest_ts = tonumber(oldest[2])
      end
