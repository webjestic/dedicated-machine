# Production Rate Limiter in Node.js

Let me design this fully before writing a line of code.

---

## Phase 1: Requirements Design

### Functional Requirements
- Limit requests per identifier (IP, user ID, API key)
- Multiple rate limit windows (per-second, per-minute, per-hour)
- Multiple algorithms available (token bucket, sliding window, fixed window)
- Per-route and per-tier limit configuration
- Meaningful error responses with retry information

### Non-Functional Requirements
- Sub-millisecond overhead on the hot path
- Survive backing store failure gracefully (fail open or fail closed — must choose explicitly)
- Horizontally scalable across multiple Node.js instances
- No single point of failure
- Accurate under concurrent load (race condition safe)

### Explicit Design Decisions (must make these before coding)

| Decision | Choice | Reason |
|---|---|---|
| Algorithm | Sliding window log + token bucket | Sliding window for accuracy, token bucket for burst control |
| Backing store | Redis with Lua scripts | Atomic operations, no race conditions |
| Failure mode | **Fail open** with logging | Availability over strict limiting; flag for review |
| Identifier strategy | API key → User ID → IP (priority order) | Authenticated requests tracked more precisely |
| Clock sync | Redis server time only | Eliminates client clock skew across nodes |
| Burst allowance | Configurable per tier | Different products have different burst needs |

```
Request
   │
   ▼
┌─────────────────────────────────────┐
│         Rate Limiter Middleware      │
│                                      │
│  1. Extract identifier               │
│  2. Resolve tier/config              │
│  3. Execute Lua script (atomic)      │
│  4. Attach headers                   │
│  5. Allow or reject                  │
└─────────────────────────────────────┘
         │                    │
         ▼                    ▼
    Redis Primary         Redis Replica
    (writes/reads)        (failover)
         │
         ▼
  ┌─────────────┐
  │  Lua Script  │  ← single atomic operation
  │  (no races)  │
  └─────────────┘
```

### `src/store/redisClient.js`

```javascript
'use strict';

const Redis = require('ioredis');

/**
 * Redis client with:
 * - Automatic reconnection
 * - Sentinel support for HA
 * - Explicit failure tracking for fail-open logic
 */

let client = null;
let isHealthy = true;
let unhealthyAt = null;
const HEALTH_CHECK_INTERVAL_MS = 5000;
const UNHEALTHY_THRESHOLD_MS = 10000;

function createClient(options = {}) {
  const config = {
    host: process.env.REDIS_HOST || '127.0.0.1',
    port: parseInt(process.env.REDIS_PORT, 10) || 6379,
    password: process.env.REDIS_PASSWORD || undefined,
    db: parseInt(process.env.REDIS_DB, 10) || 0,

    // Connection pool behavior
    maxRetriesPerRequest: 1,       // Fail fast — don't queue up requests
    enableReadyCheck: true,
    connectTimeout: 2000,          // 2s connect timeout
    commandTimeout: 500,           // 500ms per command — rate limiter must be fast

    // Reconnection strategy — exponential backoff capped at 2s
    retryStrategy(times) {
      if (times > 10) {
        // Stop retrying after 10 attempts; let health check recover
        return null;
      }
      return Math.min(times * 100, 2000);
    },

    lazyConnect: false,
    ...options,
  };

  const redis = new Redis(config);

  redis.on('connect', () => {
    isHealthy = true;
    unhealthyAt = null;
    console.info('[RateLimiter] Redis connected');
  });

  redis.on('error', (err) => {
    if (isHealthy) {
      unhealthyAt = Date.now();
      isHealthy = false;
    }
    // Log but don't throw — middleware will handle fail-open
    console.error('[RateLimiter] Redis error:', err.message);
  });

  redis.on('close', () => {
    isHealthy = false;
    if (!unhealthyAt) unhealthyAt = Date.now();
    console.warn('[RateLimiter] Redis connection closed');
  });

  redis.on('reconnecting', () => {
    console.info('[RateLimiter] Redis reconnecting...');
  });

  // Periodic health check — marks recovered after sustained connectivity
  setInterval(() => {
    if (!isHealthy) {
      redis.ping().then(() => {
        isHealthy = true;
        unhealthyAt = null;
      }).catch(() => {
        // Still unhealthy
      });
    }
  }, HEALTH_CHECK_INTERVAL_MS);

  return redis;
}

function getClient() {
  if (!client) {
    client = createClient();
  }
  return client;
}

function getHealthStatus() {
  return {
    healthy: isHealthy,
    unhealthyAt,
    unhealthyDurationMs: unhealthyAt ? Date.now() - unhealthyAt : 0,
  };
}

module.exports = { getClient, getHealthStatus, createClient };
```

### `scripts/sliding_window.lua`

```lua
--[[
  Sliding Window Rate Limiter
  
  Atomic Lua script — executes as a single Redis transaction.
  No race conditions possible between check and increment.
  
  KEYS[1]: rate limit key (e.g., "rl:user:123:minute")
  ARGV[1]: current timestamp in milliseconds
  ARGV[2]: window size in milliseconds
  ARGV[3]: max requests allowed in window
  ARGV[4]: unique request ID (prevents duplicate counting on retry)
  
  Returns: { allowed (0|1), current_count, reset_at_ms, retry_after_ms }
--]]

local key = KEYS[1]
local now = tonumber(ARGV[1])
local window_ms = tonumber(ARGV[2])
local max_requests = tonumber(ARGV[3])
local request_id = ARGV[4]

local window_start = now - window_ms

-- Remove entries outside the current window
redis.call('ZREMRANGEBYSCORE', key, '-inf', window_start)

-- Count requests in current window
local current_count = redis.call('ZCARD', key)

-- Determine reset time (when oldest entry in window expires)
local oldest = redis.call('ZRANGE', key, 0, 0, 'WITHSCORES')
local reset_at = now + window_ms
if #oldest > 0 then
  reset_at = tonumber(oldest[2]) + window_ms
end

if current_count < max_requests then
  -- Add this request to the window
  redis.call('ZADD', key, now, request_id)
  -- Set TTL slightly beyond window to allow cleanup
  redis.call('PEXPIRE', key, window_ms + 1000)
  
  return { 1, current_count + 1, reset_at, 0 }
else
  -- Rejected — calculate retry_after
  local retry_after = reset_at - now
  return { 0, current_count, reset_at, retry_after }
end
```

### `scripts/token_bucket.lua`

```lua
--[[
  Token Bucket Rate Limiter
  
  Provides burst tolerance. Tokens refill at a steady rate.
  State stored as: { tokens, last_refill_timestamp }
  
  KEYS[1]: token bucket key
  ARGV[1]: current timestamp in milliseconds  
  ARGV[2]: bucket capacity (max burst)
  ARGV[3]: refill rate (tokens per millisecond)
  ARGV[4]: tokens required for this request (default 1)
  ARGV[5]: TTL for the key in milliseconds
  
  Returns: { allowed (0|1), tokens_remaining, retry_after_ms }
--]]

local key = KEYS[1]
local now = tonumber(ARGV[1])
local capacity = tonumber(ARGV[2])
local refill_rate = tonumber(ARGV[3])   -- tokens per ms
local tokens_requested = tonumber(ARGV[4]) or 1
local ttl_ms = tonumber(ARGV[5])

-- Load existing state
local bucket = redis.call('HMGET', key, 'tokens', 'last_refill')
local tokens = tonumber(bucket[1])
local last_refill = tonumber(bucket[2])

if tokens == nil then
  -- New bucket — start full
  tokens = capacity
  last_refill = now
end

-- Refill tokens based on elapsed time
local elapsed = now - last_refill
local refill_amount = elapsed * refill_rate
tokens = math.min(capacity, tokens + refill_amount)
last_refill = now

if tokens >= tokens_requested then
  -- Consume tokens
  tokens = tokens - tokens_requested
  
  redis.call('HMSET', key, 'tokens', tokens, 'last_refill', last_refill)
  redis.call('PEXPIRE', key, ttl_ms)
  
  return { 1, math.floor(tokens), 0 }
else
  -- Not enough tokens — calculate wait time
  local tokens_needed = tokens_requested - tokens
  local retry_after_ms = math.ceil(tokens_needed / refill_rate)
  
  redis.call('HMSET', key, 'tokens', tokens, 'last_refill', last_refill)
  redis.call('PEXPIRE', key, ttl_ms)
  
  return { 0, math.floor(tokens), retry_after_ms }
end
```
