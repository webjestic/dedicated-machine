**Decision:** Request Changes

**Summary:** This PR introduces a critical event loop violation: `requests.post()` is a synchronous blocking call inside an `async def` route. Under checkout load, every invocation of this endpoint will block the event loop for up to the full 5-second timeout, freezing every concurrent request across the entire application for that duration — not just this one. Replace `requests` with `httpx.AsyncClient` (or `aiohttp`) and `await` the call; this is a non-negotiable fix before merge. The tests also provide false confidence here — `TestClient` runs synchronously and will never surface event loop starvation, so the blocking behavior passes all tests cleanly and ships invisibly to production.
