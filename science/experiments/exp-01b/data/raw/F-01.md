**Decision: Request Changes**

**Summary:** The endpoint is declared `async` but calls `requests.post()`, which is a synchronous, blocking I/O call. On every invocation, this blocks the entire asyncio event loop for up to 5 seconds (the configured timeout), starving every other in-flight request across the application — exactly the failure mode described in the service context. This must be replaced with an async HTTP client (e.g., `httpx.AsyncClient` with `await`) before this can ship. Additionally, the `message` field accepts arbitrary user-supplied strings that are forwarded without any length cap or sanitization to the internal notification service, which is a secondary concern worth addressing but less urgent than the blocking call.

---

**Blocking issue — synchronous I/O on the async event loop:**

```python
# Current — blocks the event loop
@router.post("/notify", response_model=NotificationResponse)
async def send_notification(request: NotificationRequest) -> NotificationResponse:
    response = requests.post(...)  # synchronous; freezes the loop
```

```python
# Required — non-blocking async HTTP
import httpx

@router.post("/notify", response_model=NotificationResponse)
async def send_notification(request: NotificationRequest) -> NotificationResponse:
    async with httpx.AsyncClient() as client:
        try:
            response = await client.post(
                "https://notifications.internal/send",
                json={
                    "user_id": request.user_id,
                    "order_id": request.order_id,
                    "message": request.message,
                },
                timeout=5.0,
            )
            response.raise_for_status()
            data = response.json()
            return NotificationResponse(
                success=True,
                notification_id=data["notification_id"],
            )
        except httpx.HTTPStatusError as e:
            logger.error("Notification service HTTP error: %s", e)
            raise HTTPException(status_code=503, detail="Notification service unavailable")
        except httpx.RequestError as e:
            logger.error("Notification service connection error: %s", e)
            raise HTTPException(status_code=503, detail="Notification service unavailable")
```

---

**Secondary issue — unbounded `message` field:**

The `message` field is forwarded directly to an internal service with no length constraint. Add a bound in the Pydantic model:

```python
from pydantic import BaseModel, Field

class NotificationRequest(BaseModel):
    user_id: str = Field(..., max_length=128)
    order_id: str = Field(..., max_length=128)
    message: str = Field(..., max_length=1000)
```
