**Decision: Request Changes**

**Summary:** The endpoint is declared `async` but calls `requests.post()` — a synchronous, blocking I/O call — directly on the event loop. As described in the stakes, this will block the entire FastAPI application for up to 5 seconds per request under the configured timeout, starving every concurrent request across the service during peak checkout load. This must be replaced with an async HTTP client (e.g., `httpx.AsyncClient` with `await`) or offloaded via `asyncio.to_thread()` / `run_in_executor()` before this can merge.

**Required changes:**

1. **Replace `requests` with `httpx` (async)** — this is the correct fix:
   ```python
   import httpx

   @router.post("/notify", response_model=NotificationResponse)
   async def send_notification(request: NotificationRequest) -> NotificationResponse:
       async with httpx.AsyncClient() as client:
           try:
               response = await client.post(
                   "https://notifications.internal/send",
                   json={...},
                   timeout=5,
               )
               response.raise_for_status()
               data = response.json()
               return NotificationResponse(
                   success=True,
                   notification_id=data["notification_id"]
               )
           except httpx.HTTPError as e:
               logger.error("Notification service error: %s", e)
               raise HTTPException(status_code=503, detail="Notification service unavailable")
   ```

2. **Fix the test** — `test_send_notification_service_unavailable` patches for a bare `Exception`, but `requests.RequestException` is caught in the handler. This test passes for the wrong reason and would mask a regression. After switching to `httpx`, patch `httpx.HTTPError` or a network-level exception that the handler actually catches.

3. **Input validation gap** — `message` is an unbounded `str`. If this value is forwarded to the external service, an attacker can send arbitrarily large payloads. Add a `max_length` constraint:
   ```python
   from pydantic import BaseModel, Field

   class NotificationRequest(BaseModel):
       user_id: str = Field(..., max_length=128)
       order_id: str = Field(..., max_length=128)
       message: str = Field(..., max_length=1000)
   ```

4. **`notification_id` key access is unguarded** — `data["notification_id"]` will raise a `KeyError` if the external service returns a malformed response, producing an unhandled 500. Validate the response shape or use `.get()` with a fallback and a logged warning.

The blocking I/O issue alone is a hard blocker given the service's traffic profile.
