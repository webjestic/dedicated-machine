**Decision:** Request Changes

**Summary:** The `async def` route calls `requests.post()` — a synchronous, blocking HTTP call — directly on the event loop. Under checkout load, every concurrent request stalls while this call waits on the external service, which means a slow or unresponsive notification endpoint doesn't degrade one request: it freezes the entire application. Replace `requests` with `httpx.AsyncClient` and `await` the call, or offload to a thread pool via `asyncio.run_in_executor` as a stopgap. The tests pass because `TestClient` runs synchronously and will never surface this failure mode — the bug is invisible until production load exposes it.
