**Decision: Request Changes**

**Summary:** This PR introduces a critical event loop violation: `requests.post()` is a synchronous blocking call inside an `async def` route. Under real checkout load, every concurrent request will stall while this call waits on the network — a single slow response from the notification service will freeze the entire application. Replace `requests` with `httpx.AsyncClient` and `await` the call, or offload to a thread pool via `asyncio.run_in_executor` as a stopgap. The tests pass precisely because `TestClient` runs synchronously and will never surface this failure mode — the bug is invisible until production load exposes it.
