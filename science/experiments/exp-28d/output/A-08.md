# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** Middleware library interface (not standalone service — see Open Items)
- [ ] **R-02** Per-identifier rate limiting
- [ ] **R-03** Redis-backed (does not own the connection)
- [ ] **R-04** Identifier is passed in via `identifierFn` (no auth, no IP extraction built-in)

### §2 Algorithm
- [ ] **R-05** Approximated sliding window counter algorithm
- [ ] **R-06** `current_window = floor(now / window_size)`
- [ ] **R-07** `previous_window = current_window - 1`
- [ ] **R-08** `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-09** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-10** Reject if `weighted_count >= limit`
- [ ] **R-11** Increment current count and allow if `weighted_count < limit`
- [ ] **R-12** Lua script executed atomically via `EVALSHA` with `EVAL` fallback
- [ ] **R-13** Lua script body must match normative script exactly
- [ ] **R-14** Return value structure: `[allowed, current_count, previous_count, weighted]`

### §3 Storage
- [ ] **R-15** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-16** `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-17** TTL = `window_size * 2`, set only on first write (`INCR` returns 1)
- [ ] **R-18** Redis string type (integer counter)

### §4 Failure Modes
- [ ] **R-19** F-01: Redis connection unavailable — fail-open or fail-closed per `failOpen` flag
- [ ] **R-20** F-01: Log WARN per request (rate-limited to 1/sec) during Redis unavailability
- [ ] **R-21** F-01: Emit `ratelimit.redis.unavailable` metric
- [ ] **R-22** F-01: `ioredis` with `lazyConnect: true`, `enableOfflineQueue: false`, connection timeout ≤ 200ms
- [ ] **R-23** F-02: Redis command timeout enforced client-side at 100ms
- [ ] **R-24** F-02: Emit `ratelimit.redis.timeout` metric (distinct from unavailable)
- [ ] **R-25** F-03: Log ERROR with raw Redis response on unexpected data
- [ ] **R-26** F-03: Apply fail-open/fail-closed per D-01; do not crash; do not surface internals to client
- [ ] **R-27** F-06: If `identifierFn` returns null/undefined at request time — log ERROR, fail-open for that request, do not use fallback identifier
- [ ] **R-28** F-07: Catch `NOSCRIPT` errors, fall back to `EVAL`, re-cache SHA
- [ ] **R-29** F-07: Emit `ratelimit.redis.script_miss` metric

### §5 API Contract
- [ ] **R-30** `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] **R-31** Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] **R-32** Allowed request: call `next()`, set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-33** `X-RateLimit-Remaining` floors at 0
- [ ] **R-34** `X-RateLimit-Reset = (current_window + 1) * window_size`
- [ ] **R-35** Rejected request: HTTP 429, `Retry-After`, `X-RateLimit-*` headers, `Content-Type: application/json`
- [ ] **R-36** `Retry-After = ceil((current_window + 1) * window_size - now)`, minimum value 1
- [ ] **R-37** `X-RateLimit-Remaining = 0` on rejection
- [ ] **R-38** Response body: `{ error, message, retryAfter, limit, window }` — `retryAfter` matches header
- [ ] **R-39** Response body must not include identifier, internal keys, Redis state, or stack traces
- [ ] **R-40** `onLimitExceeded` override hook
- [ ] **R-41** Redis unavailable + fail-closed: HTTP 503, `Retry-After: 10`, structured body
- [ ] **R-42** `failOpen` is required with no default — operator must declare it explicitly

### §6.1 Observability — Metrics
- [ ] **R-43** `ratelimit.request.total` counter with `namespace`, `result` tags
- [ ] **R-44** `ratelimit.request.allowed` counter with `namespace` tag
- [ ] **R-45** `ratelimit.request.rejected` counter with `namespace` tag
- [ ] **R-46** `ratelimit.redis.latency_ms` histogram with `namespace` tag
- [ ] **R-47** `ratelimit.redis.error` counter with `namespace`, `error_type` tags
- [ ] **R-48** `ratelimit.redis.script_miss` counter with `namespace` tag
- [ ] **R-49** `ratelimit.failopen.request` counter with `namespace` tag
- [ ] **R-50** `ratelimit.identifier.missing` counter with `namespace` tag
- [ ] **R-51** Metrics emitted on every request (not only rejection)

### §6.1 Observability — Structured Logging
- [ ] **R-52** All log events are structured JSON
- [ ] **R-53** Required fields: `timestamp`, `level`, `component`, `namespace`, `identifier`, `result`, `weighted_count`, `limit`, `window_id`, `redis_latency_ms`, `request_id`
- [ ] **R-54** Identifier logged (with operator note about PII)

### §6.2 Graceful Degradation
- [ ] **R-55** Rate limiter must never crash the Node.js process
- [ ] **R-56** All Redis errors caught within middleware
- [ ] **R-57** No synchronous computation that scales with request volume
- [ ] **R-58** Fail-open must be explicit and logged (WARN); silent fail-open is not acceptable
- [ ] **R-59** Recovery from Redis outage must be automatic (no cached "Redis is down" state)

### §6.4 Health Check
- [ ] **R-60** Export `isAlive()` — returns `true` if middleware is initialized
- [ ] **R-61** Export async `isReady()` — Redis PING with 200ms timeout
- [ ] **R-62** `isReady()`: if PING succeeds → `{ status: "ready", redis: "ok" }`
- [ ] **R-63** `isReady()`: if PING fails + `failOpen: true` → `{ status: "ready", redis: "degraded", mode: "fail-open" }`
- [ ] **R-64** `isReady()`: if PING fails + `failOpen: false` → `{ status: "not_ready", redis: "unavailable" }` (503 equivalent)

### `src/luaScript.js` — Normative Lua Script and EVALSHA Management

```javascript
// src/luaScript.js
// Implements: R-12, R-13, R-14, R-28, R-29

'use strict';

/**
 * NORMATIVE LUA SCRIPT — §2, "Lua script (normative)"
 *
 * ⚠️  DO NOT ALTER THIS SCRIPT without re-running the full load test suite
 *     defined in §6.5. The design document states this requirement explicitly.
 *
 * KEYS[1] = current window key
 * KEYS[2] = previous window key
 * ARGV[1] = limit
 * ARGV[2] = window_size (seconds)
 * ARGV[3] = now (unix seconds)
 * ARGV[4] = elapsed_fraction
 *
 * Return: array of 4 values
 *   [0] = 1 (allowed) or 0 (rejected)
 *   [1] = current window count after increment (or before, if rejected)
 *   [2] = previous window count
 *   [3] = weighted count used for decision
 */
const LUA_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

/**
 * Loads the Lua script into Redis and returns the SHA1 hash.
 * Must be called once at initialization. The SHA is used for all
 * subsequent EVALSHA calls (R-12).
 *
 * @param {import('ioredis').Redis} redisClient
 * @returns {Promise<string>} SHA1 hash of the loaded script
 */
async function loadScript(redisClient) {
  const sha = await redisClient.script('load', LUA_SCRIPT);
  return sha;
}

/**
 * Executes the rate limit check atomically via EVALSHA.
 * Falls back to EVAL on NOSCRIPT error and re-caches the SHA (F-07, R-28).
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string} scriptSha - Current cached SHA. May be null on first call.
 * @param {string[]} keys    - [currentKey, previousKey]
 * @param {(string|number)[]} args - [limit, windowSize, now, elapsedFrac]
 * @param {{ onScriptMiss: () => void }} callbacks
 * @returns {Promise<{ sha: string, result: number[] }>}
 *   sha: updated SHA (may have changed on NOSCRIPT recovery)
 *   result: raw array from Lua [allowed, currentCount, previousCount, weighted]
 */
async function evalScript(redisClient, scriptSha, keys, args, callbacks) {
  // Attempt EVALSHA first (R-12: use EVALSHA for performance)
  if (scriptSha) {
    try {
      const result = await redisClient.evalsha(scriptSha, keys.length, ...keys, ...args);
      return { sha: scriptSha, result };
    } catch (err) {
      if (!isNoscriptError(err)) {
        // Not a NOSCRIPT error — propagate to caller for F-01/F-02/F-03 handling
        throw err;
      }
      // NOSCRIPT: fall through to EVAL recovery below (F-07, R-28)
      callbacks.onScriptMiss(); // R-29: emit ratelimit.redis.script_miss
    }
  }

  // EVAL with full script body — also used when scriptSha is null (first call
  // before loadScript completes, or after NOSCRIPT recovery).
  // R-28: re-cache SHA after successful EVAL.
  const result = await redisClient.eval(LUA_SCRIPT, keys.length, ...keys, ...args);
  const newSha = await redisClient.script('load', LUA_SCRIPT);
  return { sha: newSha, result };
}

/**
 * Returns true if the error is a Redis NOSCRIPT error (F-07).
 * ioredis surfaces this as an error whose message starts with "NOSCRIPT".
 */
function isNoscriptError(err) {
  return err && typeof err.message === 'string' && err.message.startsWith('NOSCRIPT');
}

/**
 * Validates that the Lua script return value matches the expected structure.
 * Returns true if valid, false if the response is unexpected (F-03, R-25, R-26).
 *
 * Expected: array of 4 numeric values where index 0 is 0 or 1.
 */
function isValidScriptResult(result) {
  return (
    Array.isArray(result) &&
    result.length === 4 &&
    (result[0] === 0 || result[0] === 1) &&
    result.every((v) => typeof v === 'number' || typeof v === 'string')
  );
}

module.exports = { LUA_SCRIPT, loadScript, evalScript, isValidScriptResult };
```

**Compliance note:** Satisfies R-12, R-13, R-14, R-28, R-29.

### `src/logger.js` — Structured JSON Logger

```javascript
// src/logger.js
// Implements: R-52, R-53, R-54

'use strict';

/**
 * Emits a structured JSON log event to stdout.
 *
 * All fields specified in §6.1 are required. This function enforces the
 * schema — callers must provide all fields. Missing fields are set to null
 * rather than omitted, so the schema is always consistent for log parsers.
 *
 * ⚠️  PII NOTICE (R-54, §6.1):
 * The `identifier` field is logged as-is. If identifiers contain PII
 * (e.g., email addresses used as API keys), the logging pipeline MUST
 * hash or mask this field before storage. This is an infrastructure concern
 * per the design document, but the implementing engineer must flag it to
 * the operator before production deployment.
 *
 * §6.1 Log volume note: INFO-level logging of every allowed request will be
 * high-volume. The design document states the implementing engineer should
 * confirm with the operator whether allowed-request logs are required.
 * DEFAULT: allowed requests are logged at INFO. Set LOG_ALLOWED=false in
 * environment to suppress allowed-request INFO logs. Rejections and errors
 * are always logged regardless of this setting.
 */

const LOG_ALLOWED = process.env.LOG_ALLOWED !== 'false';

/**
 * @typedef {Object} LogEvent
 * @property {'INFO'|'WARN'|'ERROR'} level
 * @property {string} namespace
 * @property {string|null} identifier
 * @property {'allowed'|'rejected'|'error'|'failopen'} result
 * @property {number|null} weighted_count
 * @property {number} limit
 * @property {number|null} window_id
 * @property {number|null} redis_latency_ms
 * @property {string|null} request_id
 * @property {string} [message]       - Human-readable context, not in schema but useful
 * @property {string} [error_type]    - Present on error events
 * @property {*}      [raw_response]  - Present on F-03 events only; stripped before emit
 *                                      if it contains internal details. See note below.
 */

/**
 * Emits a structured log event. All §6.1 required fields are present.
 *
 * @param {LogEvent} event
 */
function log(event) {
  // Suppress INFO logs for allowed requests if operator has configured LOG_ALLOWED=false.
  // Rejections (result=rejected) and errors (result=error|failopen) are always logged.
  if (
    event.level === 'INFO' &&
    event.result === 'allowed' &&
    !LOG_ALLOWED
  ) {
    return;
  }

  const entry = {
    timestamp:        new Date().toISOString(),       // ISO8601, R-53
    level:            event.level,                    // R-53
    component:        'ratelimit',                    // R-53 (literal per design)
    namespace:        event.namespace        ?? null, // R-53
    identifier:       event.identifier       ?? null, // R-53, R-54
    result:           event.result           ?? null, // R-53
    weighted_count:   event.weighted_count   ?? null, // R-53
    limit:            event.limit            ?? null, // R-53
    window_id:        event.window_id        ?? null, // R-53
    redis_latency_ms: event.redis_latency_ms ?? null, // R-53
    request_id:       event.request_id       ?? null, // R-53
  };

  // Optional fields — included only when present to avoid polluting every log line
  if (event.message)    entry.message    = event.message;
  if (event.error_type) entry.error_type = event.error_type;

  // F-03 (R-25): log raw Redis response on unexpected data.
  // ⚠️  The raw_response field may contain internal Redis state.
  // It is logged at ERROR level only, never surfaced to clients (R-39).
  if (event.raw_response !== undefined) entry.raw_response = event.raw_response;

  process.stdout.write(JSON.stringify(entry) + '\n');
}

module.exports = { log, LOG_ALLOWED };
```

**Compliance note:** Satisfies R-52, R-53, R-54.

### `src/rateLimiter.js` — Main Middleware Factory

This is the primary implementation file. It is organized by design section.

```javascript
// src/rateLimiter.js
// Implements: R-01 through R-59 (primary implementation)

'use strict';

const { loadScript, evalScript, isValidScriptResult } = require('./luaScript');
const { createMetrics } = require('./metrics');
const { log } = require('./logger');
const { RedisErrorType } = require('./errors');

// ---------------------------------------------------------------------------
// §4, F-01: Log rate limiter — one WARN log per second during Redis outage.
// This prevents log flooding as specified in §4, F-01.
// ---------------------------------------------------------------------------
function createWarnRateLimiter() {
  let lastWarnTime = 0;
  return function shouldEmitWarn() {
    const now = Date.now();
    if (now - lastWarnTime >= 1000) {
      lastWarnTime = now;
      return true;
    }
    return false;
  };
}

// ---------------------------------------------------------------------------
// §5: Options validation
// Runs at initialization time, not at request time (R-27 partial: config errors
// at startup where detectable).
// ---------------------------------------------------------------------------

/**
 * Validates the options object. Throws synchronously on misconfiguration.
 * This surfaces configuration errors at startup, not at request time.
 *
 * @param {object} options
 */
function validateOptions(options) {
  if (!options || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  // namespace: required, string, no spaces, max 64 chars (§5)
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must be 64 characters or fewer');
  }

  // limit: required, positive integer (§5)
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: required, positive integer (§5)
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: required, function (§5)
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: required, object (§5 — the rate limiter does not own the connection)
  if (!options.redisClient || typeof options.redisClient !== 'object') {
    throw new Error('[ratelimit] options.redisClient must be a pre-constructed ioredis client');
  }

  // failOpen: required, boolean, no default (§5, R-42)
  // DESIGN REQUIREMENT: "No default. Forces the operator to make a conscious choice."
  // We enforce this by rejecting anything that is not an explicit boolean.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'There is no default. See §7 D-01 in the design document.'
    );
  }

  // onLimitExceeded: optional, function if provided (§5)
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }
}

// ---------------------------------------------------------------------------
// §3: Key construction
// ---------------------------------------------------------------------------

/**
 * Constructs the Redis key for a given window.
 * Format: ratelimit:{namespace}:{identifier}:{window_id}
 * R-15
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Computes the current window ID.
 * R-16: window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)
 *
 * ⚠️  §3 NOTE N-02: The design requires this arithmetic to be verified in a
 * unit test with fixed timestamps before shipping. See compliance matrix.
 *
 * @param {number} nowSeconds - Current Unix time in seconds
 * @param {number} windowSizeSeconds
 * @returns {number}
 */
function computeWindowId(nowSeconds, windowSizeSeconds) {
  return Math.floor(nowSeconds / windowSizeSeconds);
}

// ---------------------------------------------------------------------------
// §5: Response helpers
// ---------------------------------------------------------------------------

/**
 * Sends the 429 Too Many Requests response.
 * Implements the full response contract from §5.
 * R-35, R-36, R-37, R-38, R-39
 *
 * @param {object} res - Express response object
 * @param {object} limitInfo
 * @param {number} limitInfo.limit
 * @param {number} limitInfo.windowSizeSeconds
 * @param {number} limitInfo.currentWindow
 * @param {number} limitInfo.nowSeconds
 */
function sendRejectedResponse(res, limitInfo) {
  const { limit, windowSizeSeconds, currentWindow, nowSeconds } = limitInfo;

  const resetTimestamp = (currentWindow + 1) * windowSizeSeconds;

  // R-36: Retry-After = ceil((current_window + 1) * window_size - now)
  // R-36: minimum value is 1 (design: "A Retry-After value of 0 is a defect")
  const retryAfterRaw = Math.ceil(resetTimestamp - nowSeconds);
  const retryAfter = Math.max(1, retryAfterRaw);

  res.set({
    'Retry-After':          String(retryAfter),  // R-35, R-36
    'X-RateLimit-Limit':    String(limit),        // R-35
    'X-RateLimit-Remaining': '0',                 // R-37
    'X-RateLimit-Reset':    String(resetTimestamp), // R-35, R-34
    'X-RateLimit-Window':   String(windowSizeSeconds), // R-35
    'Content-Type':         'application/json',   // R-35
  });

  // R-38: body fields. retryAfter matches header exactly.
