# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** Middleware library interface (not standalone service — design covers both; implementing middleware)
- [ ] **R-02** Per-identifier rate limiting
- [ ] **R-03** Redis-backed (does not own the connection)
- [ ] **R-04** Sliding window algorithm
- [ ] **R-05** Structured 429 response on limit exceeded
- [ ] **R-06** Identifier extracted via `identifierFn`, not `req.socket.remoteAddress`

### §2 Algorithm
- [ ] **R-07** Approximated sliding window counter (two fixed windows)
- [ ] **R-08** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-09** Reject if `weighted_count >= limit`; increment and allow if `weighted_count < limit`
- [ ] **R-10** Atomicity via Lua script (`EVAL`/`EVALSHA`) — exact script as specified
- [ ] **R-11** Lua script return values: `[allowed, current_count, previous_count, weighted]`

### §3 Storage
- [ ] **R-12** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-13** `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-14** TTL = `window_size * 2`, set only on first write (`new_count === 1`)
- [ ] **R-15** Redis config requirements documented (not enforced in code — infrastructure concern)

### §4 Failure Modes
- [ ] **R-16** F-01: Redis connection unavailable → fail-open or fail-closed per `failOpen` flag
- [ ] **R-17** F-01: Log WARN per failure (rate-limited to 1/sec); emit `ratelimit.redis.unavailable`
- [ ] **R-18** F-01: `ioredis` with `lazyConnect: true`, `enableOfflineQueue: false`, connection timeout ≤ 200ms
- [ ] **R-19** F-02: Redis command timeout (100ms); log `ratelimit.redis.timeout` separately
- [ ] **R-20** F-03: Unexpected Redis data → log ERROR with raw response; fail-open/closed; no crash; no internal details to client
- [ ] **R-21** F-04: No active defense; clock skew detected via monitoring (window_id in logs)
- [ ] **R-22** F-05: Handled by `allkeys-lru` eviction (infrastructure); evicted key treated as zero count
- [ ] **R-23** F-06: `identifierFn` returns null → log ERROR, fail-open for that request, no silent fallback, emit `ratelimit.identifier.missing`
- [ ] **R-24** F-07: `EVALSHA` with `NOSCRIPT` fallback to `EVAL`; re-cache SHA; emit `ratelimit.redis.script_miss`
- [ ] **R-25** F-08: No special handling required (state in Redis)

### §5 API Contract
- [ ] **R-26** `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] **R-27** Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] **R-28** Allowed request: call `next()`, set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-29** `X-RateLimit-Remaining` = `floor(limit - weighted_count)`, minimum 0
- [ ] **R-30** `X-RateLimit-Reset` = `(current_window + 1) * windowSizeSeconds`
- [ ] **R-31** Rejected: HTTP 429, `Retry-After`, all X-RateLimit headers, `Content-Type: application/json`
- [ ] **R-32** `Retry-After` = `ceil((current_window + 1) * windowSizeSeconds - now)`, minimum 1
- [ ] **R-33** Response body: `error`, `message`, `retryAfter`, `limit`, `window` — no identifier, no internal keys, no stack traces
- [ ] **R-34** `retryAfter` in body matches `Retry-After` header exactly
- [ ] **R-35** `onLimitExceeded` override hook
- [ ] **R-36** Fail-closed Redis unavailable: HTTP 503, `Retry-After: 10`, structured body
- [ ] **R-37** `Retry-After` minimum value is 1 (never 0)

### §6 Operational Requirements
- [ ] **R-38** Metrics: `ratelimit.request.total` (counter, tags: namespace, result)
- [ ] **R-39** Metrics: `ratelimit.request.allowed` (counter, tag: namespace)
- [ ] **R-40** Metrics: `ratelimit.request.rejected` (counter, tag: namespace)
- [ ] **R-41** Metrics: `ratelimit.redis.latency_ms` (histogram, tag: namespace, buckets: 1,5,10,25,50,100,250,500)
- [ ] **R-42** Metrics: `ratelimit.redis.error` (counter, tags: namespace, error_type)
- [ ] **R-43** Metrics: `ratelimit.redis.script_miss` (counter, tag: namespace)
- [ ] **R-44** Metrics: `ratelimit.failopen.request` (counter, tag: namespace)
- [ ] **R-45** Metrics: `ratelimit.identifier.missing` (counter, tag: namespace)
- [ ] **R-46** Structured JSON logging: all required fields on every log event
- [ ] **R-47** Identifier logged (PII flag to operator)
- [ ] **R-48** Fail-open log at WARN, rate-limited to 1 log/sec to prevent flooding
- [ ] **R-49** Never crash the Node.js process; all Redis errors caught
- [ ] **R-50** Never block the event loop
- [ ] **R-51** Recovery from Redis outage automatic (no cached "down" state)
- [ ] **R-52** Health check: `isAlive()` synchronous function
- [ ] **R-53** Health check: `isReady()` async function with Redis PING, 200ms timeout
- [ ] **R-54** Readiness: fail-open → 200 with degraded status; fail-closed → 503 if Redis down
- [ ] **R-55** Health check not on hot path

'use strict';

// §6.1: All log events must be structured JSON with required fields.
// §6.1: Identifier must be logged. If identifier contains PII (e.g., email
// addresses used as API keys), the logging pipeline must hash or mask it
// before storage. This is flagged to the operator here as a comment; it is
// an infrastructure concern, not a rate limiter concern.

// §4 F-01: WARN logs during Redis unavailability must be rate-limited to
// one log line per second to prevent log flooding.
// This map tracks the last emission time per namespace+event combination.
const warnRateLimitState = new Map();
const WARN_RATE_LIMIT_MS = 1000; // 1 second, as specified in §4 F-01

/**
 * Emit a structured JSON log event.
 *
 * @param {string} level - 'INFO' | 'WARN' | 'ERROR'
 * @param {object} fields - Required and optional fields per §6.1 schema.
 */
function log(level, fields) {
  const entry = {
    timestamp: new Date().toISOString(),
    level,
    component: 'ratelimit',
    // Spread caller-supplied fields after fixed fields so fixed fields
    // cannot be overridden by the caller.
    ...fields,
  };
  // §6.1: Structured JSON. Write to stdout; log aggregation is an
  // infrastructure concern.
  process.stdout.write(JSON.stringify(entry) + '\n');
}

/**
 * Emit a WARN log that is rate-limited to one emission per second per key.
 * Used for F-01 and F-02 to prevent log flooding during Redis outages.
 *
 * @param {string} rateLimitKey - Deduplication key (e.g., namespace + event name).
 * @param {object} fields - Log fields per §6.1 schema.
 */
function warnRateLimited(rateLimitKey, fields) {
  const now = Date.now();
  const last = warnRateLimitState.get(rateLimitKey) || 0;
  if (now - last >= WARN_RATE_LIMIT_MS) {
    warnRateLimitState.set(rateLimitKey, now);
    log('WARN', fields);
  }
}

module.exports = { log, warnRateLimited };
```

**Compliance note (R-46, R-47, R-48):** ✓ Structured JSON. ✓ Identifier in caller-supplied fields. ✓ WARN rate-limiting implemented.

### `src/luaScript.js` — Lua Script and EVALSHA Management

Satisfies: **R-10**, **R-11**, **R-24** (F-07 NOSCRIPT recovery)

```javascript
'use strict';

const crypto = require('crypto');
const metrics = require('./metrics');
const { REDIS_ERROR_TYPE } = require('./errors');

// ============================================================
// §2 (normative): The Lua script below is reproduced verbatim from the
// design document. The design states: "The implementing engineer must not
// alter the logic of this script without re-running the load test suite
// defined in §6.5."
//
// DO NOT MODIFY THIS SCRIPT without running all LT-01 through LT-07
// load test scenarios and documenting the results.
// ============================================================
const LUA_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

// Pre-compute the SHA1 of the script for EVALSHA.
// This is the SHA1 Redis will use after SCRIPT LOAD or the first EVAL.
// We compute it locally so we can attempt EVALSHA before the first EVAL.
// Note: Redis computes SHA1 of the script body; we must hash the exact
// same bytes that Redis will receive.
const SCRIPT_SHA = crypto.createHash('sha1').update(LUA_SCRIPT).digest('hex');

/**
 * Execute the rate limit Lua script against Redis.
 * Implements F-07: attempts EVALSHA first; on NOSCRIPT error, falls back
 * to EVAL with the full script body and re-caches the SHA.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string[]} keys - [currentKey, previousKey]
 * @param {(string|number)[]} args - [limit, windowSize, now, elapsedFrac]
 * @param {string} namespace - For metrics emission.
 * @returns {Promise<[number, number, number, number]>} - [allowed, currentCount, previousCount, weighted]
 */
async function evalScript(redisClient, keys, args, namespace) {
  try {
    // Attempt EVALSHA first (F-07: performance path).
    const result = await redisClient.evalsha(SCRIPT_SHA, keys.length, ...keys, ...args);
    return parseScriptResult(result);
  } catch (err) {
    if (err.message && err.message.startsWith('NOSCRIPT')) {
      // F-07: Script was evicted from Redis script cache.
      // Fall back to EVAL with full script body. Redis will re-cache it.
      metrics.scriptMiss(namespace);
      const result = await redisClient.eval(LUA_SCRIPT, keys.length, ...keys, ...args);
      return parseScriptResult(result);
    }
    // Re-throw non-NOSCRIPT errors to be handled by the caller.
    throw err;
  }
}

/**
 * Parse and validate the Lua script return value.
 * §2 specifies: [allowed(0|1), current_count, previous_count, weighted]
 *
 * @param {any} raw - Raw Redis response.
 * @returns {[number, number, number, number]}
 * @throws {Error} If the response does not match the expected structure (F-03).
 */
function parseScriptResult(raw) {
  // F-03: Validate the response structure before use.
  if (
    !Array.isArray(raw) ||
    raw.length !== 4 ||
    (raw[0] !== 0 && raw[0] !== 1)
  ) {
    const err = new Error('Unexpected Lua script response structure');
    err.code = 'UNEXPECTED_RESPONSE';
    err.rawResponse = raw;
    throw err;
  }
  return [
    Number(raw[0]), // allowed: 1 = allowed, 0 = rejected
    Number(raw[1]), // current_count after increment (or before, if rejected)
    Number(raw[2]), // previous_count
    Number(raw[3]), // weighted count used for decision
  ];
}

module.exports = { evalScript, LUA_SCRIPT, SCRIPT_SHA };
```

**Compliance note (R-10, R-11, R-24):** ✓ Exact Lua script from design. ✓ Return value structure matches spec. ✓ EVALSHA → NOSCRIPT → EVAL fallback. ✓ `script_miss` metric emitted on NOSCRIPT.

### `src/rateLimiter.js` — Main Middleware Factory

Satisfies: **R-01–R-09**, **R-12–R-14**, **R-16–R-23**, **R-25–R-37**, **R-46–R-51**

```javascript
'use strict';

const { evalScript } = require('./luaScript');
const metrics = require('./metrics');
const { log, warnRateLimited } = require('./logger');
const { REDIS_ERROR_TYPE } = require('./errors');

// ============================================================
// §5: Options validation constants
// ============================================================
const NAMESPACE_MAX_LENGTH = 64;
const NAMESPACE_PATTERN = /^[^\s]+$/; // No spaces, as specified in §5

/**
 * Validate options at initialization time.
 * §5 specifies all fields are required unless marked optional.
 * §4 F-06 specifies that identifier extraction failure must be a hard
 * configuration error at startup, not at request time (where possible).
 *
 * @param {object} options
 * @throws {Error} If any required option is missing or invalid.
 */
function validateOptions(options) {
  if (options === null || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  // namespace: string, no spaces, max 64 chars
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (options.namespace.length > NAMESPACE_MAX_LENGTH) {
    throw new Error(`[ratelimit] options.namespace must not exceed ${NAMESPACE_MAX_LENGTH} characters`);
  }
  if (!NAMESPACE_PATTERN.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }

  // limit: positive integer
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: positive integer
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: synchronous function
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: must be present (ioredis instance)
  if (options.redisClient === null || options.redisClient === undefined) {
    throw new Error('[ratelimit] options.redisClient must be provided');
  }

  // failOpen: required boolean, no default.
  // §5: "Required. Explicit declaration of fail-open (true) or fail-closed
  // (false) behavior. No default. Forces the operator to make a conscious
  // choice."
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true (fail-open) or false (fail-closed). ' +
      'There is no default. See §7 D-01 of the design document.'
    );
  }

  // onLimitExceeded: optional, but if provided must be a function
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }
}

/**
 * Build the Redis key for a given namespace, identifier, and window ID.
 * §3: Key format: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Compute the current and previous window IDs and elapsed fraction.
 * §3: window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)
 * §2: elapsed_fraction = (now % window_size) / window_size
 *
 * @param {number} windowSizeSeconds
 * @returns {{ nowSeconds: number, currentWindow: number, previousWindow: number, elapsedFraction: number }}
 */
function computeWindow(windowSizeSeconds) {
  // §3 note: "compute window_id as Math.floor(Date.now() / 1000 / window_size_in_seconds)
  // where Date.now() returns milliseconds."
  const nowMs = Date.now();
  const nowSeconds = nowMs / 1000; // Keep fractional seconds for elapsed_fraction precision
  const currentWindow = Math.floor(nowSeconds / windowSizeSeconds);
  const previousWindow = currentWindow - 1;
  // §2: elapsed_fraction = (now % window_size) / window_size
  // 'now' in the algorithm is Unix timestamp in seconds (integer in the spec,
  // but we use fractional for accuracy in elapsed_fraction).
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;

  return { nowSeconds, currentWindow, previousWindow, elapsedFraction };
}

/**
 * Compute the Retry-After value in seconds.
 * §5: ceil((current_window + 1) * window_size - now), minimum 1.
 * §6.3: "A Retry-After value of 0 is a defect; minimum value is 1."
 *
 * @param {number} currentWindow
 * @param {number} windowSizeSeconds
 * @param {number} nowSeconds
 * @returns {number}
 */
function computeRetryAfter(currentWindow, windowSizeSeconds, nowSeconds) {
  const resetTime = (currentWindow + 1) * windowSizeSeconds;
  const retryAfter = Math.ceil(resetTime - nowSeconds);
  // §6.3: minimum value is 1.
  return Math.max(1, retryAfter);
}

/**
 * Set rate limit response headers on the response object.
 * Used for both allowed (§5 allowed request headers) and rejected
 * (§5 rejected request headers) responses.
 *
 * @param {object} res - Node.js/Express response object.
 * @param {object} params
 */
function setRateLimitHeaders(res, { limit, remaining, reset, windowSizeSeconds }) {
  res.setHeader('X-RateLimit-Limit', String(limit));
  // §5: X-RateLimit-Remaining: floor at 0.
  res.setHeader('X-RateLimit-Remaining', String(Math.max(0, Math.floor(remaining))));
  res.setHeader('X-RateLimit-Reset', String(reset));
  res.setHeader('X-RateLimit-Window', String(windowSizeSeconds));
}

/**
 * Send the default 429 Too Many Requests response.
 * §5: Status 429, Retry-After header, all X-RateLimit headers,
 * Content-Type: application/json, structured body.
 *
 * @param {object} res
 * @param {object} limitInfo
 */
function sendDefaultLimitExceededResponse(res, limitInfo) {
  const { limit, retryAfter, reset, windowSizeSeconds } = limitInfo;

  setRateLimitHeaders(res, {
    limit,
    remaining: 0, // §5: Always 0 on rejection.
    reset,
    windowSizeSeconds,
  });

  res.setHeader('Retry-After', String(retryAfter));
  res.setHeader('Content-Type', 'application/json');

  // §5: Body must not include identifier, internal key names, Redis state,
  // or stack traces.
  // §5: retryAfter in body must match Retry-After header exactly.
  const body = {
    error: 'rate_limit_exceeded',
    message: 'Too many requests. Please retry after the indicated delay.',
    retryAfter,
    limit,
    window: windowSizeSeconds,
  };

  res.statusCode = 429;
  res.end(JSON.stringify(body));
}

/**
 * Send the 503 Service Unavailable response for fail-closed Redis failure.
 * §5: Status 503, Retry-After: 10, structured body.
 *
 * @param {object} res
 */
function sendServiceUnavailableResponse(res) {
  res.setHeader('Retry-After', '10');
  res.setHeader('Content-Type', 'application/json');

  const body = {
    error: 'service_unavailable',
    message: 'Rate limiting is temporarily unavailable. Please retry shortly.',
    retryAfter: 10,
  };

  res.statusCode = 503;
  res.end(JSON.stringify(body));
}

/**
 * Handle a Redis operation error according to the fail-open/fail-closed
 * configuration. Covers F-01, F-02, F-03.
 *
 * @param {Error} err - The Redis error.
 * @param {string} namespace
 * @param {string} identifier
 * @param {boolean} failOpen
 * @param {object} req
 * @param {object} res
 * @param {Function} next
 * @param {object} windowInfo - { nowSeconds, currentWindow, windowSizeSeconds }
 */
function handleRedisError(err, namespace, identifier, failOpen, req, res, next, windowInfo) {
  // Classify the error type for metrics (§6.1).
  let errorType;
  if (err.code === 'UNEXPECTED_RESPONSE') {
    errorType = REDIS_ERROR_TYPE.UNEXPECTED;
  } else if (
    err.message &&
    (err.message.includes('ETIMEDOUT') || err.message.includes('Command timed out'))
  ) {
    errorType = REDIS_ERROR_TYPE.TIMEOUT;
  } else {
    errorType = REDIS_ERROR_TYPE.UNAVAILABLE;
  }

  metrics.redisError(namespace,
