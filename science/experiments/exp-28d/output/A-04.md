# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** Middleware library interface (not standalone service — see Open Items)
- [ ] **R-02** Per-identifier rate limiting
- [ ] **R-03** Redis-backed (does not own the connection)
- [ ] **R-04** Identifier extraction via `identifierFn` (not `req.socket.remoteAddress`)

### §2 Algorithm
- [ ] **R-05** Approximated sliding window counter algorithm
- [ ] **R-06** `current_window = floor(now / window_size)`
- [ ] **R-07** `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-08** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-09** Reject if `weighted_count >= limit`
- [ ] **R-10** Increment `current_count` only if allowed
- [ ] **R-11** Atomicity via Lua script (normative script must not be altered)
- [ ] **R-12** Lua script return values: `[allowed, new_count, previous_count, weighted]`

### §3 Storage
- [ ] **R-13** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-14** `window_id = Math.floor(Date.now() / 1000 / window_size_in_seconds)`
- [ ] **R-15** TTL = `window_size * 2`, set only when `INCR` returns 1
- [ ] **R-16** Redis version ≥ 3.2 (documented requirement, not enforceable in code — flagged)

### §4 Failure Modes
- [ ] **R-17** F-01: Redis connection unavailable → fail-open or fail-closed per `failOpen` option
- [ ] **R-18** F-01: WARN log, rate-limited to 1 per second, emit `ratelimit.redis.unavailable`
- [ ] **R-19** F-01: `ioredis` with `lazyConnect: true`, `enableOfflineQueue: false`, connection timeout ≤ 200ms
- [ ] **R-20** F-02: Redis command timeout (100ms), enforced client-side; emit `ratelimit.redis.timeout`
- [ ] **R-21** F-03: Unexpected Redis response → ERROR log, fail-open/closed, no crash, no internal details to client
- [ ] **R-22** F-04: No active defense; clock skew detected via monitoring only
- [ ] **R-23** F-05: Handled by `allkeys-lru` eviction policy (documented requirement)
- [ ] **R-24** F-06: `identifierFn` returns null → ERROR log, fail-open for that request, no silent fallback, emit `ratelimit.identifier.missing`
- [ ] **R-25** F-07: `EVALSHA` with `NOSCRIPT` fallback to `EVAL`; re-cache SHA; emit `ratelimit.redis.script_miss`
- [ ] **R-26** F-08: No special handling required (state in Redis)

### §5 API Contract
- [ ] **R-27** `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] **R-28** Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] **R-29** Allowed: call `next()`, set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-30** `X-RateLimit-Remaining` floored at 0
- [ ] **R-31** Rejected: HTTP 429, `Retry-After`, all X-RateLimit headers, `Content-Type: application/json`
- [ ] **R-32** Rejected body: `{ error, message, retryAfter, limit, window }` — no identifier, no internals, no stack traces
- [ ] **R-33** `retryAfter` in body matches `Retry-After` header exactly
- [ ] **R-34** `Retry-After` minimum value is 1 (not 0)
- [ ] **R-35** `Retry-After = ceil((current_window + 1) * window_size - now)`
- [ ] **R-36** `X-RateLimit-Reset = (current_window + 1) * window_size`
- [ ] **R-37** Fail-closed Redis error: HTTP 503, `Retry-After: 10`, structured body
- [ ] **R-38** `onLimitExceeded` override hook

### §6 Operational Requirements
- [ ] **R-39** Metrics: `ratelimit.request.total` (counter, tags: namespace, result)
- [ ] **R-40** Metrics: `ratelimit.request.allowed` (counter, tag: namespace)
- [ ] **R-41** Metrics: `ratelimit.request.rejected` (counter, tag: namespace)
- [ ] **R-42** Metrics: `ratelimit.redis.latency_ms` (histogram, tag: namespace, buckets: 1,5,10,25,50,100,250,500)
- [ ] **R-43** Metrics: `ratelimit.redis.error` (counter, tags: namespace, error_type)
- [ ] **R-44** Metrics: `ratelimit.redis.script_miss` (counter, tag: namespace)
- [ ] **R-45** Metrics: `ratelimit.failopen.request` (counter, tag: namespace)
- [ ] **R-46** Metrics: `ratelimit.identifier.missing` (counter, tag: namespace)
- [ ] **R-47** Structured JSON logging on every rate limiter event (fields: timestamp, level, component, namespace, identifier, result, weighted_count, limit, window_id, redis_latency_ms, request_id)
- [ ] **R-48** Identifier must be logged (PII flag to operator documented)
- [ ] **R-49** Middleware must never crash the Node.js process
- [ ] **R-50** Middleware must never block the event loop
- [ ] **R-51** Fail-open must be explicit and logged (no silent fail-open)
- [ ] **R-52** Recovery from Redis outage must be automatic (no cached "down" state)
- [ ] **R-53** Health check: `isAlive()` function exported
- [ ] **R-54** Health check: `isReady()` async function exported; Redis PING with 200ms timeout
- [ ] **R-55** Readiness: fail-open → 200 with degraded status; fail-closed → 503
- [ ] **R-56** Metrics backend is a decision point (D-05) — implement stub interface

### §6.5 Load Tests
- [ ] **R-57** Load test scenarios LT-01 through LT-07 are specified (not implemented in code — test infrastructure is out of scope for the library, but the design requires they be run; flagged in Open Items)

### §6.6 Alerting
- [ ] **R-58** Alert thresholds documented (operational config, not code — flagged in Open Items)

### `src/validate-options.js`

Implements: **R-28** (options validation), **R-24** (fail-open has no default — forces explicit declaration)

```javascript
'use strict';

/**
 * Validates the options object passed to rateLimiter().
 *
 * Design ref: §5 API Contract — Options object.
 * Every field marked "required" in the design is enforced here.
 * Missing or malformed options are a hard startup error, not a runtime error.
 * Design ref §4 F-06: identifier extraction failure must be surfaced at
 * initialization, not at request time (for static config errors).
 */
function validateOptions(options) {
  if (options === null || typeof options !== 'object') {
    throw new TypeError('[ratelimit] options must be a plain object');
  }

  // namespace: string, no spaces, max 64 chars
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new TypeError('[ratelimit] options.namespace must be a non-empty string');
  }
  if (/\s/.test(options.namespace)) {
    throw new TypeError('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new TypeError('[ratelimit] options.namespace must be 64 characters or fewer');
  }

  // limit: positive integer
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new TypeError('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: positive integer
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new TypeError('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: synchronous function
  if (typeof options.identifierFn !== 'function') {
    throw new TypeError('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: must be provided; the limiter does not own the connection
  if (options.redisClient === null || options.redisClient === undefined) {
    throw new TypeError('[ratelimit] options.redisClient must be a pre-constructed ioredis client');
  }

  // failOpen: REQUIRED boolean — no default.
  // Design §5: "Forces the operator to make a conscious choice."
  // Design §7 D-01: fail-open vs fail-closed is an explicit decision point.
  // We do not supply a default here. If this throws, the operator must
  // explicitly set failOpen: true or failOpen: false.
  if (typeof options.failOpen !== 'boolean') {
    throw new TypeError(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'There is no default. See design §7 D-01 for rationale.'
    );
  }

  // onLimitExceeded: optional, but if provided must be a function
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new TypeError('[ratelimit] options.onLimitExceeded must be a function if provided');
  }
}

module.exports = { validateOptions };
```

**Compliance note:** R-28 satisfied. `failOpen` has no default — R-28 and D-01 satisfied.

### `src/lua-script.js`

Implements: **R-11** (atomicity via Lua), **R-12** (return values), **R-25** (EVALSHA + NOSCRIPT fallback)

```javascript
'use strict';

const crypto = require('crypto');

/**
 * NORMATIVE LUA SCRIPT — DO NOT ALTER WITHOUT RE-RUNNING LOAD TEST SUITE (§6.5).
 *
 * Design ref: §2 Algorithm — "The implementing engineer must not alter the
 * logic of this script without re-running the load test suite defined in §6.5."
 *
 * Script inputs:
 *   KEYS[1] = current window key
 *   KEYS[2] = previous window key
 *   ARGV[1] = limit (number)
 *   ARGV[2] = window_size (seconds)
 *   ARGV[3] = now (Unix seconds, float)
 *   ARGV[4] = elapsed_fraction (float)
 *
 * Return value (array):
 *   [0] = 1 (allowed) or 0 (rejected)
 *   [1] = current window count after increment (or before, if rejected)
 *   [2] = previous window count
 *   [3] = weighted count used for decision
 */
const LUA_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

/**
 * SHA1 of the script, computed once at module load time.
 * Used for EVALSHA. If Redis does not have the script cached (NOSCRIPT error),
 * we fall back to EVAL and re-cache the SHA.
 * Design ref: §4 F-07.
 */
const LUA_SCRIPT_SHA = crypto
  .createHash('sha1')
  .update(LUA_SCRIPT)
  .digest('hex');

/**
 * Execute the rate limit Lua script against Redis.
 *
 * Attempts EVALSHA first. On NOSCRIPT error, falls back to EVAL.
 * Emits ratelimit.redis.script_miss metric on NOSCRIPT (F-07).
 *
 * @param {object} params
 * @param {import('ioredis').Redis} params.redisClient
 * @param {string} params.currentKey
 * @param {string} params.previousKey
 * @param {number} params.limit
 * @param {number} params.windowSizeSeconds
 * @param {number} params.nowSeconds  - Unix timestamp in seconds (float)
 * @param {number} params.elapsedFraction
 * @param {object} params.metrics     - metrics interface (§6.1)
 * @param {string} params.namespace
 * @returns {Promise<{allowed: boolean, newCount: number, previousCount: number, weightedCount: number}>}
 */
async function executeLuaScript({
  redisClient,
  currentKey,
  previousKey,
  limit,
  windowSizeSeconds,
  nowSeconds,
  elapsedFraction,
  metrics,
  namespace,
}) {
  const argv = [
    String(limit),
    String(windowSizeSeconds),
    String(nowSeconds),
    String(elapsedFraction),
  ];

  let rawResult;

  try {
    // Primary path: EVALSHA (avoids re-sending script body on every call)
    rawResult = await redisClient.evalsha(
      LUA_SCRIPT_SHA,
      2,             // number of keys
      currentKey,
      previousKey,
      ...argv
    );
  } catch (err) {
    if (err.message && err.message.startsWith('NOSCRIPT')) {
      // F-07: Script was evicted from Redis cache. Fall back to EVAL.
      // Design: "Re-cache the script SHA after successful EVAL."
      // ioredis does not automatically cache the SHA on EVAL; we track it
      // in module scope (LUA_SCRIPT_SHA is already computed). The next call
      // will attempt EVALSHA again — if Redis now has it, it will succeed.
      // If not, we will fall back again. This is correct behavior.
      metrics.increment('ratelimit.redis.script_miss', { namespace });

      rawResult = await redisClient.eval(
        LUA_SCRIPT,
        2,
        currentKey,
        previousKey,
        ...argv
      );
    } else {
      // Re-throw for the caller (middleware.js) to handle as a Redis error
      throw err;
    }
  }

  // Validate the return structure (F-03: unexpected data)
  if (
    !Array.isArray(rawResult) ||
    rawResult.length < 4 ||
    rawResult[0] === null ||
    rawResult[1] === null
  ) {
    const parseError = new Error('UNEXPECTED_REDIS_RESPONSE');
    parseError.rawResult = rawResult;
    throw parseError;
  }

  return {
    allowed: rawResult[0] === 1,
    newCount: Number(rawResult[1]),
    previousCount: Number(rawResult[2]),
    weightedCount: Number(rawResult[3]),
  };
}

module.exports = { executeLuaScript, LUA_SCRIPT, LUA_SCRIPT_SHA };
```

**Compliance note:** R-11, R-12, R-25 satisfied.

### `src/keys.js`

Implements: **R-13**, **R-14** (key format and window_id calculation)

```javascript
'use strict';

/**
 * Constructs Redis keys for the rate limiter.
 *
 * Design ref: §3 Storage — Key Strategy.
 * Format: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * Design note (§3): "compute window_id as Math.floor(Date.now() / 1000 / window_size_in_seconds)
 * where Date.now() returns milliseconds. Verify this arithmetic in a unit test
 * with fixed timestamps before shipping."
 *
 * This function accepts nowMs (milliseconds) explicitly rather than calling
 * Date.now() internally, so that unit tests can inject fixed timestamps.
 */

/**
 * @param {object} params
 * @param {string} params.namespace
 * @param {string} params.identifier
 * @param {number} params.windowSizeSeconds
 * @param {number} params.nowMs - Current time in milliseconds (e.g. Date.now())
 * @returns {{ currentKey: string, previousKey: string, currentWindow: number, nowSeconds: number, elapsedFraction: number }}
 */
function buildKeys({ namespace, identifier, windowSizeSeconds, nowMs }) {
  // Convert milliseconds to seconds (float preserved for elapsed_fraction)
  const nowSeconds = nowMs / 1000;

  // Design §2: current_window = floor(now / window_size)
  const currentWindow = Math.floor(nowSeconds / windowSizeSeconds);
  const previousWindow = currentWindow - 1;

  // Design §2: elapsed_fraction = (now % window_size) / window_size
  // Using nowSeconds (float) for sub-second precision in the approximation.
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;

  // Design §3: key format ratelimit:{namespace}:{identifier}:{window_id}
  const currentKey  = `ratelimit:${namespace}:${identifier}:${currentWindow}`;
  const previousKey = `ratelimit:${namespace}:${identifier}:${previousWindow}`;

  return {
    currentKey,
    previousKey,
    currentWindow,
    nowSeconds,
    elapsedFraction,
  };
}

module.exports = { buildKeys };
```

**Compliance note:** R-13, R-14 satisfied. `nowMs` is injected rather than calling `Date.now()` internally — this is required for the unit test verifiability the design calls out in §3.

### `src/logger.js`

Implements: **R-47**, **R-48** (structured JSON logging)

```javascript
'use strict';

/**
 * Structured JSON logger for the rate limiter.
 *
 * Design ref: §6.1 Observability — Structured Logging.
 * All log events must be structured JSON with the required fields.
 *
 * This module writes to process.stdout (INFO/WARN) and process.stderr (ERROR).
 * It does not depend on a logging framework so that it can be embedded in any
 * application. If the host application uses a logging framework (Winston, Pino,
 * etc.), the implementing engineer should replace the write() calls below with
 * calls to that framework's logger, preserving the field structure.
 *
 * PII WARNING (Design §6.1):
 * "If the identifier contains PII (e.g., email addresses used as API keys),
 * the logging pipeline must hash or mask it before storage."
 * The rate limiter logs the raw identifier as the design requires.
 * The operator must configure the logging pipeline to hash/mask PII identifiers
 * before log storage. This is flagged here as required by the design.
 *
 * Log volume (Design §6.1):
 * "The implementing engineer should confirm with the operator whether
 * allowed-request logs are required or whether only rejections and errors
 * should be logged at INFO."
 * Default: allowed requests are logged at INFO. See DECISION POINT D-06 below.
 */

/**
 * DECISION POINT D-06 — Log volume for allowed requests.
 * Design §6.1 flags this as a question for the operator.
 * DEFAULT CHOSEN: Log allowed requests at INFO level.
 * This is the safer default for incident investigation (R-47 requires fields
 * on "every rate limiter log event"). To suppress allowed-request logs,
 * set LOG_ALLOWED_REQUESTS=false in the environment or change the constant below.
 *
 * Changing this: set the exported LOG_ALLOWED_REQUESTS to false, or wire it
 * to an environment variable.
 */
const LOG_ALLOWED_REQUESTS = true; // D-06: change to false to suppress allowed-request INFO logs

/**
 * Rate-limited warn logger for Redis unavailability.
 * Design §4 F-01: "Log a WARN-level event per request (rate-limited to one
 * log line per second to prevent log flooding)."
 */
let lastRedisWarnTs = 0;

/**
 * @param {object} fields - All required log fields per §6.1
 */
function log(fields) {
  const entry = {
    timestamp: new Date().toISOString(),
    level: fields.level,
    component: 'ratelimit',
    namespace: fields.namespace,
    identifier: fields.identifier,
    result: fields.result,
    weighted_count: fields.weighted_count,
    limit: fields.limit,
    window_id: fields.window_id,
    redis_latency_ms: fields.redis_latency_ms,
    request_id: fields.request_id,
  };

  // Include error details at ERROR level (but never surface to client — R-32)
  if (fields.level === 'ERROR' && fields.error_detail) {
    entry.error_detail = fields.error_detail;
  }

  const line = JSON.stringify(entry);

  if (fields.level === 'ERROR') {
    process.stderr.write(line + '\n');
  } else {
    process.stdout.write(line + '\n');
  }
}

/**
 * Log an allowed request.
 * Respects D-06 (log volume decision).
 */
function logAllowed(fields) {
  if (!LOG_ALLOWED_REQUESTS) return;
  log({ ...fields, level: 'INFO', result: 'allowed' });
}

/**
 * Log a rejected request (429). Always logged.
 */
function logRejected(fields) {
  log({ ...fields, level: 'INFO', result: 'rejected' });
}

/**
 * Log a fail-open bypass. Always logged at WARN.
 * Design §6.2: "Every request that bypasses rate limiting due to a Redis
 * failure must... log at WARN level. Silent fail-open is not acceptable."
 *
 * Rate-limited to one log line per second for Redis unavailability (F-01).
 * Other fail-open causes (F-03, F-06) are not rate-limited.
 */
function logFailOpen(fields, rateLimit = false) {
  if (rateLimit) {
    const now = Date.now();
    if (now - lastRedisWarnTs < 1000) return;
    lastRedisWarnTs = now;
  }
  log({ ...fields, level: 'WARN', result: 'failopen' });
}

/**
 * Log an error condition. Always logged.
 */
function logError(fields) {
  log({ ...fields, level: 'ERROR', result: 'error' });
}

module.exports = { logAllowed, logRejected, logFailOpen, logError };
```

**Compliance note:** R-47, R-48 satisfied. PII warning documented. D-06 (log volume) flagged and defaulted.

### `src/middleware.js`

Implements: **R-05–R-12** (algorithm), **R-17–R-26** (failure modes), **R-27–R-38** (API contract), **R-39–R-51** (observability and degradation)

```javascript
'use strict';

const { buildKeys }        = require('./keys');
const { executeLuaScript } = require('./lua-script');
const { logAllowed, logRejected, logFailOpen, logError } = require('./logger');

/**
 * Core rate limiter middleware factory.
 *
 * Design ref: §5 API Contract, §2 Algorithm, §4 Failure Modes, §6 Operational.
 *
 * @param {object} options - Validated options (see validate-options.js)
 * @param {object} metrics - Metrics client (see metrics.js)
 * @returns {function(req, res, next): void} Express-compatible middleware
 */
function createMiddleware(options, metrics) {
  const {
    namespace,
    limit,
    windowSizeSeconds,
    identifierFn,
    redisClient,
    onLimitExceeded,
    failOpen,
  } = options;

  /**
   * Handles Redis failure (F-01, F-02, F-03).
   * Applies fail-open or fail-closed behavior per the explicit `failOpen` option.
   * Design §6.2: "Every request that bypasses rate limiting due to a Redis
   * failure must emit ratelimit.failopen.request and log at WARN level."
   *
   * @param {object} req
   * @param {object} res
   * @param {function} next
   * @param {string} identifier
   * @param {string} errorType - 'timeout' | 'unavailable' | 'unexpected'
   * @param {Error} err
   * @param {boolean} rateLimitLog - whether to rate-limit the log output (F-01 only)
   */
  function handleRedisFailure(req, res, next, identifier, errorType, err, rateLimitLog) {
    // Emit error metric (R-43)
    metrics.increment('ratelimit.redis.error', { namespace, error_type: errorType });

    const logFields = {
      namespace,
      identifier,
      weighted_count: null,
      limit,
      window_id: null,
      redis_latency_ms: null,
      request_id: req.id || req.headers['x-request-id'] || null,
      error_detail: err ? err.message : errorType,
    };

    if (failOpen) {
      // Fail-open: allow the request through, log at WARN (R-17, R-18, R-51)
