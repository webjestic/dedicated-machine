# Production Rate Limiter in Node.js

Let me design this fully before writing a line of code.

---

## Phase 1: Requirements Design

### What algorithm?

| Algorithm | Pros | Cons | Best For |
|---|---|---|---|
| Token Bucket | Allows bursting, smooth | Complex distributed sync | APIs with burst tolerance |
| Fixed Window | Simple, fast | Boundary spike problem | Internal tools |
| Sliding Window Log | Precise | High memory (stores timestamps) | Low-volume, high-accuracy |
| Sliding Window Counter | Precise + memory efficient | Slight approximation | **Production APIs** ✓ |
| Leaky Bucket | Perfectly smooth output | No burst tolerance | Payment processing |

**Decision: Sliding Window Counter** — precise, memory-efficient, handles the boundary spike problem that kills fixed window implementations in production.

### What storage backend?

- **In-memory only**: Dies on restart, can't share across instances. Not production.
- **Redis**: Atomic operations via Lua scripts, TTL management, shared across all instances. **This is the answer.**
- **Redis + in-memory fallback**: What we'll actually build.

### Axes of rate limiting needed

```
Per IP          → abuse prevention
Per User ID     → authenticated user fairness  
Per API Key     → partner/tenant isolation
Per Route       → protect expensive endpoints differently
Composite       → IP + Route, User + Route
```

### Failure modes to design for upfront

```
1. Redis is down          → fail open or fail closed? (configurable)
2. Redis is slow          → timeout threshold needed
3. Clock skew             → Redis server time, not app server time
4. Thundering herd        → Lua atomicity handles this
5. Memory leak            → TTLs on every key, mandatory
6. Bypass via headers     → X-Forwarded-For spoofing
7. Key collision          → namespace all keys
8. Config hot-reload      → rules change without restart
```

### `src/limiter/slidingWindow.lua`

```lua
-- Sliding Window Counter Algorithm
-- Atomic execution in Redis — no race conditions possible
--
-- KEYS[1] = current window key   e.g. "rl:user:42:route:/api/data:1703001600"
-- KEYS[2] = previous window key  e.g. "rl:user:42:route:/api/data:1703001540"
-- ARGV[1] = current timestamp (ms)
-- ARGV[2] = window size (ms)
-- ARGV[3] = limit (max requests)
-- ARGV[4] = TTL for keys (seconds)
--
-- Returns: { current_count, limit, remaining, reset_time_ms, allowed(0|1) }

local currentKey  = KEYS[1]
local previousKey = KEYS[2]
local now         = tonumber(ARGV[1])
local windowMs    = tonumber(ARGV[2])
local limit       = tonumber(ARGV[3])
local ttl         = tonumber(ARGV[4])

-- How far into the current window are we? (0.0 to 1.0)
local windowStart    = math.floor(now / windowMs) * windowMs
local elapsedInWindow = now - windowStart
local previousWeight = 1.0 - (elapsedInWindow / windowMs)

-- Get counts from both windows
local currentCount  = tonumber(redis.call('GET', currentKey) or 0)
local previousCount = tonumber(redis.call('GET', previousKey) or 0)

-- Weighted count: previous window contributes less as current window progresses
local weightedCount = math.floor(previousCount * previousWeight) + currentCount

local allowed = 0
local newCount = currentCount

if weightedCount < limit then
    -- Increment current window
    newCount = redis.call('INCR', currentKey)
    -- Set TTL only on new keys (INCR returns 1 for new key)
    if newCount == 1 then
        redis.call('PEXPIRE', currentKey, windowMs * 2)
    end
    allowed = 1
end

-- Reset time = end of current window
local resetTime = windowStart + windowMs

return {
    weightedCount + (allowed == 1 and 1 or 0),  -- count after this request
    limit,
    math.max(0, limit - weightedCount - (allowed == 1 and 1 or 0)),
    resetTime,
    allowed
}
```

### `src/limiter/RateLimiter.js`

```javascript
'use strict';

const fs   = require('fs');
const path = require('path');

// Lua script loaded once at startup — not on every request
const LUA_SCRIPT = fs.readFileSync(
  path.join(__dirname, 'slidingWindow.lua'),
  'utf8'
);

/**
 * Redis-backed sliding window rate limiter.
 *
 * Design decisions documented here because they matter:
 *
 * 1. Lua script atomicity: INCR + EXPIRE must be atomic. A non-atomic
 *    implementation has a race where two requests both read 0, both
 *    increment, and both set TTL — the second TTL resets the expiry.
 *    The Lua script eliminates this entirely.
 *
 * 2. Redis server time: We use Date.now() here but pass it to Redis.
 *    In a multi-region setup, you'd use Redis TIME command instead.
 *    Flagged in operational notes below.
 *
 * 3. Key namespace: All keys prefixed with 'rl:' to avoid collision
 *    with application keys in the same Redis instance.
 */
class RateLimiter {
  /**
   * @param {Object} redisClient - ioredis client instance
   * @param {Object} options
   * @param {number} options.windowMs      - Window size in milliseconds
   * @param {number} options.limit         - Max requests per window
   * @param {string} options.keyPrefix     - Namespace prefix for Redis keys
   * @param {number} options.redisTimeout  - Max ms to wait for Redis (default: 100)
   */
  constructor(redisClient, options = {}) {
    if (!redisClient) {
      throw new Error('RateLimiter requires a Redis client');
    }

    this.redis        = redisClient;
    this.windowMs     = options.windowMs    || 60_000;   // 1 minute default
    this.limit        = options.limit       || 100;
    this.keyPrefix    = options.keyPrefix   || 'rl';
    this.redisTimeout = options.redisTimeout || 100;     // 100ms — fail fast

    // Pre-load the Lua script SHA into Redis for efficient execution.
    // EVALSHA is faster than EVAL — Redis caches the compiled script.
    this._scriptSha = null;
    this._loadScript();
  }

  /**
   * Load Lua script into Redis and cache the SHA.
   * Called once on construction. Retried automatically if Redis
   * flushes its script cache (SCRIPT FLUSH or restart).
   */
  async _loadScript() {
    try {
      this._scriptSha = await this.redis.script('LOAD', LUA_SCRIPT);
    } catch (err) {
      // Non-fatal at construction time — will fall back to EVAL
      console.error('[RateLimiter] Failed to preload Lua script:', err.message);
    }
  }

  /**
   * Build Redis keys for current and previous windows.
   * Key format: rl:{prefix}:{windowTimestamp}
   *
   * Example: rl:ip:192.168.1.1:route:/api/data:1703001600000
   *
   * @param {string} identifier - Unique key for this rate limit rule
   * @param {number} now        - Current timestamp in ms
   * @returns {{ currentKey: string, previousKey: string }}
   */
  _buildKeys(identifier, now) {
    const windowSize    = this.windowMs;
    const currentWindow = Math.floor(now / windowSize) * windowSize;
    const prevWindow    = currentWindow - windowSize;

    // Sanitize identifier to prevent key injection
    const safeId = identifier.replace(/[^a-zA-Z0-9:._\-\/]/g, '_');

    return {
      currentKey:  `${this.keyPrefix}:${safeId}:${currentWindow}`,
      previousKey: `${this.keyPrefix}:${safeId}:${prevWindow}`,
    };
  }

  /**
   * Execute the sliding window check against Redis.
   * Uses EVALSHA with fallback to EVAL if script cache was flushed.
   *
   * @param {string} identifier - Rate limit key (e.g., "ip:1.2.3.4")
   * @returns {Promise<RateLimitResult>}
   */
  async check(identifier) {
    const now = Date.now();
    const { currentKey, previousKey } = this._buildKeys(identifier, now);

    // TTL = 2 full windows to ensure previous window data survives
    const ttlSeconds = Math.ceil((this.windowMs * 2) / 1000);

    const args = [
      now,
      this.windowMs,
      this.limit,
      ttlSeconds,
    ];

    let result;

    try {
      result = await this._execWithTimeout(
        () => this._evalScript([currentKey, previousKey], args),
        this.redisTimeout
      );
    } catch (err) {
      // Surface the error type for the caller to decide: fail open or closed
      const rateLimitError = new Error(`Redis rate limit check failed: ${err.message}`);
      rateLimitError.code  = 'REDIS_FAILURE';
      rateLimitError.cause = err;
      throw rateLimitError;
    }

    // Destructure Lua return values
    const [count, limit, remaining, resetTime, allowed] = result;

    return {
      allowed:    allowed === 1,
      count:      Number(count),
      limit:      Number(limit),
      remaining:  Math.max(0, Number(remaining)),
      resetTime:  Number(resetTime),      // Unix ms timestamp
      retryAfter: allowed === 1
        ? null
        : Math.ceil((Number(resetTime) - now) / 1000),  // seconds
    };
  }

  /**
   * Execute Lua script, falling back from EVALSHA to EVAL
   * if the script cache was flushed (NOSCRIPT error).
   */
  async _evalScript(keys, args) {
    if (this._scriptSha) {
      try {
        return await this.redis.evalsha(
          this._scriptSha,
          keys.length,
          ...keys,
          ...args
        );
      } catch (err) {
        // NOSCRIPT: Redis flushed its script cache — reload and retry
        if (err.message && err.message.includes('NOSCRIPT')) {
          await this._loadScript();
          // Fall through to EVAL below
        } else {
          throw err;
        }
      }
    }

    // Direct EVAL fallback (slightly slower but always works)
    return this.redis.eval(
      LUA_SCRIPT,
      keys.length,
      ...keys,
      ...args
    );
  }

  /**
   * Wrap a Redis call with a hard timeout.
   * Redis should never be the reason your API goes down.
   * If Redis is slow, we fail fast and let the caller decide what to do.
   *
   * @param {Function} fn      - Async function to execute
   * @param {number}   timeout - Timeout in milliseconds
   */
  _execWithTimeout(fn, timeout) {
    return new Promise((resolve, reject) => {
      const timer = setTimeout(() => {
        reject(new Error(`Redis timeout after ${timeout}ms`));
      }, timeout);

      fn()
        .then((result) => {
          clearTimeout(timer);
          resolve(result);
        })
        .catch((err) => {
          clearTimeout(timer);
          reject(err);
        });
    });
  }
}

module.exports = RateLimiter;
```

### `src/limiter/FallbackLimiter.js`

```javascript
'use strict';

/**
 * In-memory fallback rate limiter.
 *
 * Used ONLY when Redis is unavailable.
 *
 * Critical limitations to understand:
 *   - State is NOT shared across Node.js processes/instances
 *   - State is lost on process restart
 *   - Memory grows with unique identifiers — cleanup is mandatory
 *
 * This is a degraded mode, not a replacement for Redis.
 * Operational behavior (fail open vs closed) is controlled by config.
 */
class FallbackLimiter {
  /**
   * @param {Object} options
   * @param {number} options.windowMs - Window size in ms
   * @param {number} options.limit    - Max requests per window
   */
  constructor(options = {}) {
    this.windowMs = options.windowMs || 60_000;
    this.limit    = options.limit    || 100;

    // Map of identifier -> { count, windowStart }
    this._store = new Map();

    // Cleanup interval — prevents unbounded memory growth
    // Runs every 5 minutes, removes expired windows
    this._cleanupInterval = setInterval(
      () => this._cleanup(),
      5 * 60 * 1000
    ).unref(); // unref() prevents this timer from keeping the process alive
  }

  /**
   * Check and increment the rate limit counter for an identifier.
   * @param {string} identifier
   * @returns {RateLimitResult}
   */
  check(identifier) {
    const now         = Date.now();
    const windowStart = Math.floor(now / this.windowMs) * this.windowMs;
    const resetTime   = windowStart + this.windowMs;

    let entry = this._store.get(identifier);

    // New identifier or window has rolled over
    if (!entry || entry.windowStart !== windowStart) {
      entry = { count: 0, windowStart };
    }

    const allowed = entry.count < this.limit;

    if (allowed) {
      entry.count += 1;
      this._store.set(identifier, entry);
    }

    return {
      allowed,
      count:      entry.count,
      limit:      this.limit,
      remaining:  Math.max(0, this.limit - entry.count),
      resetTime,
      retryAfter: allowed ? null : Math.ceil((resetTime - now) / 1000),
    };
  }

  /**
   * Remove entries from expired windows.
   * Called on interval — not on every request (that would be O(n) per request).
   */
  _cleanup() {
    const now = Date.now();
    for (const [key, entry] of this._store.entries()) {
      const windowEnd = entry.windowStart + this.windowMs;
      if (now > windowEnd) {
        this._store.delete(key);
      }
    }
  }

  /**
   * Graceful shutdown — clear the cleanup interval.
   */
  destroy() {
    clearInterval(this._cleanupInterval);
    this._store.clear();
  }
}

module.exports = FallbackLimiter;
```

### `src/config/rules.js`

```javascript
'use strict';

/**
 * Rate limit rule definitions.
 *
 * Rules are evaluated in order. ALL matching rules must pass.
 * This means a request can be blocked by any single rule.
 *
 * Rule anatomy:
 *   name        - Human-readable, used in logs and headers
 *   keyFn       - Function that derives the Redis key from (req)
 *                 Returns null to skip this rule for this request
 *   windowMs    - Window duration
 *   limit       - Max requests in window
 *   failOpen    - If Redis fails, allow the request (true) or block (false)
 *                 Default: true (availability > security for most APIs)
 *                 Set false for sensitive endpoints (auth, payments)
 */

const MINUTE = 60 * 1000;
const HOUR   = 60 * MINUTE;

const rules = [
  // ─── Global IP limit ────────────────────────────────────────────────────
  // Catches unauthenticated abuse. Generous limit — real users rarely hit this.
  {
    name:     'ip-global',
    keyFn:    (req) => `ip:${req.ip}`,
    windowMs: MINUTE,
    limit:    200,
    failOpen: true,
  },

  // ─── Per-user limit (authenticated requests only) ───────────────────────
  // Skipped entirely if no user is authenticated (keyFn returns null).
  {
    name:     'user-global',
    keyFn:    (req) => req.user?.id ? `user:${req.user.id}` : null,
    windowMs: MINUTE,
    limit:    100,
    failOpen: true,
  },

  // ─── Expensive endpoint protection ──────────────────────────────────────
  // Applied only to routes matching the pattern. Tight limit.
  {
    name:     'user-search-endpoint',
    keyFn:    (req) => {
      if (!req.path.startsWith('/api/search')) return null;
      const id = req.user?.id || req.ip;
      return `search:${id}`;
    },
    windowMs: MINUTE,
    limit:    10,
    failOpen: false,  // Search is expensive — fail closed if Redis is down
  },

  // ─── Auth endpoint — brute force protection ──────────────────────────────
  {
    name:     'auth-endpoint',
    keyFn:    (req) => {
      if (!req.path.startsWith('/auth')) return null;
      return `auth:${req.ip}`;
    },
    windowMs: 15 * MINUTE,
    limit:    20,
    failOpen: false,  // Never fail open on auth endpoints
  },

  // ─── API key tier limits ─────────────────────────────────────────────────
  // Partners get higher limits than anonymous users.
  {
    name:     'api-key',
    keyFn:    (req) => {
      const apiKey = req.headers['x-api-key'];
      return apiKey ? `apikey:${apiKey}` : null;
    },
    windowMs: HOUR,
    limit:    10_000,
    failOpen: true,
  },
];

module.exports = rules;
```

### `src/utils/identityExtractor.js`

```javascript
'use strict';

/**
 * Safe IP extraction from requests.
 *
 * This is a security-critical function. Getting it wrong means:
 *   - Attackers can bypass rate limiting by spoofing X-Forwarded-For
 *   - You rate-limit your load balancer's IP instead of the real client
 *
 * Rules:
 *   1. Trust X-Forwarded-For ONLY if you control the proxy that sets it
 *   2. Always take the LEFTMOST IP in X-Forwarded-For (client IP)
 *   3. Validate the IP format — don't trust raw header values
 *   4. Fall back to req.socket.remoteAddress if no proxy headers
 *
 * If you're behind AWS ALB, Cloudflare, or nginx, configure
 * express 'trust proxy' setting appropriately instead of
 * parsing headers manually here.
 */

// IPv4 and IPv6 validation regex
const IPV4_REGEX = /^(\d{1,3}\.){3}\d{1,3}$/;
const IPV6_REGEX = /^[0-9a-fA-F:]+$/;

/**
 * Validate that a string looks like an IP address.
 * Not a full RFC-compliant validator — just sanity checking
 * to prevent header injection attacks.
 *
 * @param {string} ip
 * @returns {boolean}
 */
function isValidIp(ip) {
  if (!ip || typeof ip !== 'string') return false;
  const trimmed = ip.trim();
  return IPV4_REGEX.test(trimmed) || IPV6_REGEX.test(trimmed);
}

/**
 * Extract the real client IP from a request.
 *
 * @param {Object}  req
 * @param {boolean} trustProxy - Whether to read X-Forwarded-For
 * @returns {string} IP address or 'unknown'
 */
function extractIp(req, trustProxy = false) {
  if (trustProxy) {
    const forwarded = req.headers['x-forwarded-for'];
    if (forwarded) {
      // X-Forwarded-For: client, proxy1, proxy2
      // We want the leftmost (original client) IP
      const ips = forwarded.split(',').map(ip => ip.trim());
      const clientIp = ips[0];
      if (isValidIp(clientIp)) {
        return clientIp;
      }
    }

    // Cloudflare sets this header — more reliable than X-Forwarded-For
    const cfIp = req.headers['cf-connecting-ip'];
    if (cfIp && isValidIp(cfIp)) {
      return cfIp;
    }
  }

  // Direct connection — use socket remote address
  const socketIp = req.socket?.remoteAddress || req.connection?.remoteAddress;
  if (socketIp && isValidIp(socketIp)) {
    return socketIp;
  }

  // Express sets req.ip when trust proxy is configured
  if (req.ip && isValidIp(req.ip)) {
    return req.ip;
  }

  return 'unknown';
}

module.exports = { extractIp, isValidIp };
```

### `src/middleware/rateLimitMiddleware.js`

```javascript
'use strict';

const RateLimiter    = require('../limiter/RateLimiter');
const FallbackLimiter = require('../limiter/FallbackLimiter');
const { extractIp }  = require('../utils/identityExtractor');
const logger         = require('../utils/logger');

/**
 * Rate limit middleware factory.
 *
 * Returns an Express middleware that enforces all configured rules.
 * All applicable rules are checked in parallel — the request is
 * rejected if ANY rule is exceeded.
 *
 * Headers set on every response (successful or rejected):
 *   X-RateLimit-Limit     - The rule's limit
 *   X-RateLimit-Remaining - Requests remaining in current window
 *   X-RateLimit-Reset     - Unix timestamp when window resets
 *   Retry-After           - Seconds until retry (only on 429)
 *
 * @param {Object}   redisClient - ioredis client
 * @param {Array}    rules       - Rule definitions from config/rules.js
 * @param {Object}   options
 * @param {boolean}  options.trustProxy    - Trust X-Forwarded-For header
 * @param {boolean}  options.defaultFailOpen - Global fail-open default
 */
function createRateLimitMiddleware(redisClient, rules, options = {}) {
  const {
    trustProxy     = false,
    defaultFailOpen = true,
  } = options;

  // Create one limiter instance per rule — each has its own window/limit config
  const limiters = rules.map(rule => ({
    rule,
    limiter: new RateLimiter(redisClient, {
      windowMs:  rule.windowMs,
      limit:     rule.limit,
      keyPrefix: `rl:${rule.name}`,
    }),
    fallback: new FallbackLimiter({
      windowMs: rule.windowMs,
      limit:    rule.limit,
    }),
  }));

  return async function rateLimitMiddleware(req, res, next) {
    // Attach real IP to request object for rule keyFns to use
    req.ip = extractIp(req, trustProxy);

    // Collect all rules that apply to this request
    const applicableRules = limiters
      .map(({ rule, limiter, fallback }) => {
        const key = rule.keyFn(req);
        if (!key) return null;  // Rule opted out for this request
        return { rule, limiter, fallback, key };
      })
      .filter(Boolean);

    if (applicableRules.length === 0) {
      return next();
    }

    // Check all applicable rules in parallel — don't serialize Redis calls
    const checkPromises = applicableRules.map(async ({ rule, limiter, fallback, key }) => {
      let result;
      let usedFallback = false;

      try {
        result = await limiter.check(key);
      } catch (err) {
        // Redis failed — decide based on rule's failOpen setting
        const failOpen = rule.failOpen ?? defaultFailOpen;

        logger.warn('rate_limit_redis_failure', {
          rule:      rule.name,
          key,
          error:     err.message,
          failOpen,
          requestId: req.id,
          ip:        req.ip,
        });

        if (failOpen) {
          // Allow the request but record that we're in degraded mode
          usedFallback = true;
          result = fallback.check(key);
        } else {
          // Fail closed — return a synthetic "blocked" result
          result = {
            allowed:    false,
            count:      0,
            limit:      rule.limit,
            remaining:  0,
            resetTime:  Date.now() + rule.windowMs,
            retryAfter: Math.ceil(rule.windowMs / 1000),
          };
        }
      }

      return { rule, result, usedFallback };
    });

    let ruleResults;
    try {
      ruleResults = await Promise.all(checkPromises);
    } catch (err) {
      // Unexpected error in the rate limiter itself — don't crash the app
      logger.error('rate_limit_unexpected_error', {
        error:     err.message,
        stack:     err.stack,
        requestId: req.id,
      });
      return next(); // Fail open for unexpected errors
    }

    // Find the most restrictive result across all rules
    const blocked = ruleResults.filter(r => !r.result.allowed);
    const mostRestrictive = blocked.length > 0
      ? blocked.reduce((a, b) =>
          a.result.remaining <= b.result.remaining ? a : b
        )
      : ruleResults.reduce((a, b) =>
          a.result.remaining <= b.result.remaining ? a : b
        );

    // Set rate limit headers from the most restrictive rule
    // Using the standard headers from the IETF draft spec
    const { result } = mostRestrictive;
    res.setHeader('X-RateLimit-Limit',     result.limit);
    res.setHeader('X-RateLimit-Remaining', result.remaining);
    res.setHeader('X-RateLimit-Reset',     Math.ceil(result.resetTime / 1000));

    if (blocked.length > 0) {
      const blockingRule = blocked[0];

      res.setHeader('Retry-After', blockingRule.result.retryAfter);

      logger.info('rate_limit_exceeded', {
        rule:      blockingRule.rule.name,
        ip:        req.ip,
        userId:    req.user?.id,
        path:      req.path,
        method:    req.method,
        requestId: req.id,
      });

      return res.status(429).json({
        error:      'Too Many Requests',
        message:    `Rate limit exceeded for rule: ${blockingRule.rule.name}`,
        retryAfter: blockingRule.result.retryAfter,
        resetTime:  blockingRule.result.resetTime,
      });
    }

    // Log degraded mode usage for monitoring
    const fallbacksUsed = ruleResults.filter(r => r.usedFallback);
    if (fallbacksUsed.length > 0) {
      logger.warn('rate_limit_fallback_active', {
        rules:     fallbacksUsed.map(r => r.rule.name),
        requestId: req.id,
      });
    }

    next();
  };
}

module.exports = createRateLimitMiddleware;
```

### `src/utils/logger.js`

```javascript
'use strict';

/**
 * Structured logger.
 *
 * Outputs JSON lines — compatible with Datadog, Splunk, CloudWatch.
 * Replace with winston/pino in production for:
 *   - Log levels controlled by environment
 *   - Log sampling for high-volume events
 *   - Async writes (pino is significantly faster)

const LOG_LEVEL = process.env.LOG_LEVEL || 'info';

const LEVELS = { error: 0, warn: 1, info: 2, debug: 3 };
const currentLevel = LEVELS[LOG_LEVEL] ?? LEVELS.info;

function log(level, event, data = {}) {
  if (LEVELS[level] > currentLevel) return;

  const entry = {
    timestamp: new Date().toISOString(),
    level,
    event,
    service:   'rate-limiter',
    ...data,
  };

  // Use stderr for errors/warnings, stdout for info/debug
  const output = level === 'error' || level === 'warn'
    ? process.stderr
    : process.stdout;

  output.write(JSON.stringify(entry) + '\n');
}
