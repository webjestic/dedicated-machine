# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** — Consumes (does not own) a Redis connection
- [ ] **R-02** — Identifier is passed in or extracted; no authentication logic
- [ ] **R-03** — No per-route differentiation in the limiter itself
- [ ] **R-04** — Limits supplied at initialization time only

### §2 Algorithm
- [ ] **R-05** — Approximated sliding window counter algorithm
- [ ] **R-06** — `current_window = floor(now / window_size)`, `previous_window = current_window - 1`
- [ ] **R-07** — `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-08** — `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-09** — Reject if `weighted_count >= limit`; allow and increment if `weighted_count < limit`
- [ ] **R-10** — Atomicity via Lua script (`EVAL`/`EVALSHA`); non-atomic read-then-write is explicitly prohibited
- [ ] **R-11** — Lua script logic matches normative script exactly (no alterations)
- [ ] **R-12** — Return value index 0: `1` = allowed, `0` = rejected; indices 1–3 carry counts

### §3 Storage
- [ ] **R-13** — Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-14** — `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-15** — TTL = `window_size * 2` seconds, set only on first write (`new_count === 1`)
- [ ] **R-16** — Redis type: String (integer counter); operations: GET, INCR, EXPIRE via Lua

### §4 Failure Modes
- [ ] **R-17** — F-01: Redis connection unavailable → fail-open or fail-closed per `failOpen` flag; WARN log (rate-limited to 1/sec); emit `ratelimit.redis.unavailable` metric
- [ ] **R-18** — F-01: ioredis configured with `lazyConnect: true`, `enableOfflineQueue: false`, connection timeout ≤ 200ms
- [ ] **R-19** — F-02: Redis command timeout (100ms client-side); emit `ratelimit.redis.timeout` metric separately from unavailable
- [ ] **R-20** — F-03: Unexpected Redis response → ERROR log with raw response; fail-open/closed per flag; no process crash; no internal detail to client
- [ ] **R-21** — F-04: No active defense; clock skew detected via monitoring only (emit `window_id` in logs/metrics)
- [ ] **R-22** — F-05: Handled by `allkeys-lru` eviction policy (configuration requirement, documented)
- [ ] **R-23** — F-06: `identifierFn` returns null/undefined → ERROR log; fail-open for that request; no silent fallback identifier; emit `ratelimit.identifier.missing`
- [ ] **R-24** — F-07: `EVALSHA` with `NOSCRIPT` fallback to `EVAL`; re-cache SHA; emit `ratelimit.redis.script_miss`
- [ ] **R-25** — F-08: No special handling; Redis holds state; automatic on reconnect

### §5 API Contract
- [ ] **R-26** — `rateLimiter(options)` returns Express/Node.js middleware `(req, res, next)`
- [ ] **R-27** — Options: `namespace` (string, no spaces, max 64 chars), `limit` (positive int), `windowSizeSeconds` (positive int), `identifierFn` (sync fn), `redisClient` (ioredis), `onLimitExceeded` (optional), `failOpen` (boolean, required, no default)
- [ ] **R-28** — Allowed request: call `next()`; set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-29** — `X-RateLimit-Remaining` = `floor(limit - weighted_count)`, floored at 0
- [ ] **R-30** — `X-RateLimit-Reset` = `(current_window + 1) * window_size` (Unix seconds)
- [ ] **R-31** — Rejected request: HTTP 429; headers: `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining: 0`, `X-RateLimit-Reset`, `X-RateLimit-Window`, `Content-Type: application/json`
- [ ] **R-32** — `Retry-After` = `ceil((current_window + 1) * window_size - now)`; minimum value 1; must match body `retryAfter`
- [ ] **R-33** — 429 body: `{ error, message, retryAfter, limit, window }`; no identifier, no internal keys, no stack traces
- [ ] **R-34** — 503 body (fail-closed Redis failure): `{ error, message, retryAfter: 10 }`; `Retry-After: 10` header
- [ ] **R-35** — `onLimitExceeded` override: called instead of default 429 handler when provided

### §6.1 Observability — Metrics
- [ ] **R-36** — `ratelimit.request.total` counter with tags `namespace`, `result:[allowed|rejected|error]`
- [ ] **R-37** — `ratelimit.request.allowed` counter with tag `namespace`
- [ ] **R-38** — `ratelimit.request.rejected` counter with tag `namespace`
- [ ] **R-39** — `ratelimit.redis.latency_ms` histogram with tag `namespace`; buckets: 1, 5, 10, 25, 50, 100, 250, 500
- [ ] **R-40** — `ratelimit.redis.error` counter with tags `namespace`, `error_type:[timeout|unavailable|unexpected]`
- [ ] **R-41** — `ratelimit.redis.script_miss` counter with tag `namespace`
- [ ] **R-42** — `ratelimit.failopen.request` counter with tag `namespace`
- [ ] **R-43** — `ratelimit.identifier.missing` counter with tag `namespace`

### §6.1 Observability — Structured Logging
- [ ] **R-44** — All log events: structured JSON with fields: `timestamp`, `level`, `component`, `namespace`, `identifier`, `result`, `weighted_count`, `limit`, `window_id`, `redis_latency_ms`, `request_id`
- [ ] **R-45** — Identifier must be logged (PII flag documented)
- [ ] **R-46** — WARN log for fail-open events (rate-limited to 1/sec per F-01)
- [ ] **R-47** — Rejections and errors always logged; allowed-request log level is a decision point (D-05 adjacent)

### §6.2 Graceful Degradation
- [ ] **R-48** — Rate limiter must never crash the Node.js process; all Redis errors caught
- [ ] **R-49** — No synchronous computation scaling with request volume
- [ ] **R-50** — Fail-open must be explicit and logged; silent fail-open is not acceptable
- [ ] **R-51** — Recovery from Redis outage automatic; no cached "Redis is down" state in process memory

### §6.3 Client Error Guidance
- [ ] **R-52** — `Retry-After` minimum value 1 (value of 0 is a defect)
- [ ] **R-53** — 429 response must not include identifier, internal keys, stack traces

### §6.4 Health Check
- [ ] **R-54** — Export `isAlive()` → `true` if middleware initialized; never fails due to Redis
- [ ] **R-55** — Export async `isReady()` → Redis PING with 200ms timeout; returns structured response per fail-open/closed config
- [ ] **R-56** — Health check must not run on the hot path

### §7 Decision Points (referenced throughout)
- [ ] **R-57** — D-01: fail-open vs. fail-closed — `failOpen` option required, no default (operator must choose explicitly)
- [ ] **R-58** — D-03: identifier extraction (IP from `X-Forwarded-For`) — implement but flag for audit
- [ ] **R-59** — D-05: metrics backend — implement abstraction; default to no-op with clear swap point

### `src/luaScript.js`
**Satisfies: R-10, R-11, R-12, R-24**

```javascript
'use strict';

// §2 — Normative Lua script.
// CRITICAL: Do not alter the logic of this script without re-running the
// load test suite defined in §6.5. The script is reproduced verbatim from
// the design document. Any deviation is a defect.
const RATE_LIMIT_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

// Precomputed SHA1 of the script above for use with EVALSHA.
// This is computed at startup by calling scriptLoad(); the value here is
// the in-memory cache. It is NOT hardcoded — hardcoding would break on any
// whitespace change. The cache is populated by loadScript().
let cachedScriptSha = null;

/**
 * Load the Lua script into Redis and cache the SHA.
 * Must be called before the first request is served.
 * Called again automatically after a NOSCRIPT error (F-07).
 *
 * @param {import('ioredis').Redis} redisClient
 * @returns {Promise<string>} SHA1 of the loaded script
 */
async function loadScript(redisClient) {
  const sha = await redisClient.script('load', RATE_LIMIT_SCRIPT);
  cachedScriptSha = sha;
  return sha;
}

/**
 * Execute the rate limit Lua script using EVALSHA with NOSCRIPT fallback.
 *
 * §4, F-07: Use EVALSHA for performance. On NOSCRIPT error, fall back to
 * EVAL with the full script body and re-cache the SHA. An unhandled NOSCRIPT
 * error would cause every check to fail — this fallback is required.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string[]} keys  [currentKey, previousKey]
 * @param {(string|number)[]} args [limit, windowSize, now, elapsedFrac]
 * @param {{ emit: Function }} metrics - metrics emitter for script_miss
 * @param {string} namespace
 * @returns {Promise<[number, number, number, number]>} [allowed, currentCount, previousCount, weighted]
 */
async function evalScript(redisClient, keys, args, metrics, namespace) {
  // R-24: EVALSHA with NOSCRIPT fallback
  if (cachedScriptSha) {
    try {
      const result = await redisClient.evalsha(cachedScriptSha, keys.length, ...keys, ...args);
      return result;
    } catch (err) {
      if (err.message && err.message.startsWith('NOSCRIPT')) {
        // F-07: Script was evicted. Re-load and fall through to EVAL.
        // R-41: emit script_miss metric
        metrics.increment('ratelimit.redis.script_miss', { namespace });
        cachedScriptSha = null;
        // Fall through to EVAL below.
      } else {
        throw err;
      }
    }
  }

  // Either first call (no SHA cached) or NOSCRIPT recovery path.
  const result = await redisClient.eval(
    RATE_LIMIT_SCRIPT,
    keys.length,
    ...keys,
    ...args
  );
  // Re-cache SHA for subsequent calls (F-07 recovery).
  await loadScript(redisClient);
  return result;
}

module.exports = { RATE_LIMIT_SCRIPT, loadScript, evalScript };
```

**Compliance note:** R-10, R-11, R-12, R-24 satisfied. The script is reproduced verbatim. EVALSHA/NOSCRIPT fallback is implemented. SHA is never hardcoded.

### `src/logger.js`
**Satisfies: R-44, R-45, R-46, R-47**

```javascript
'use strict';

// §6.1 — Structured logging.
// All log events are structured JSON per the design specification.
// Required fields per §6.1:
//   timestamp, level, component, namespace, identifier, result,
//   weighted_count, limit, window_id, redis_latency_ms, request_id
//
// R-45 — OPERATOR FLAG: The identifier field is logged as-is. If identifiers
// contain PII (e.g., email addresses used as API keys), the logging pipeline
// MUST hash or mask this field before storage. This is an infrastructure
// concern per the design, but the implementing engineer must communicate it
// to the operator before go-live.

// §4, F-01 — WARN log for Redis unavailability must be rate-limited to
// one log line per second to prevent log flooding.
// This is a per-namespace rate limit on the warn log, not on request handling.
const warnLogLastEmittedAt = new Map(); // namespace -> timestamp ms
const WARN_LOG_RATE_LIMIT_MS = 1000;

/**
 * Emit a structured JSON log event to stdout.
 * All fields specified in §6.1 must be present.
 *
 * @param {object} params
 * @param {'INFO'|'WARN'|'ERROR'} params.level
 * @param {string} params.namespace
 * @param {string|null} params.identifier
 * @param {'allowed'|'rejected'|'error'|'failopen'} params.result
 * @param {number|null} params.weighted_count
 * @param {number} params.limit
 * @param {number} params.window_id
 * @param {number|null} params.redis_latency_ms
 * @param {string|null} params.request_id
 * @param {object} [params.extra] - Additional fields (e.g., raw error). Never surfaced to clients.
 */
function log({ level, namespace, identifier, result, weighted_count, limit, window_id, redis_latency_ms, request_id, extra }) {
  const event = {
    timestamp: new Date().toISOString(),
    level,
    component: 'ratelimit',
    namespace,
    identifier,
    result,
    weighted_count,
    limit,
    window_id,
    redis_latency_ms,
    request_id: request_id || null,
    ...extra,
  };
  // Write to stdout as newline-delimited JSON.
  // The logging pipeline is responsible for ingestion and PII masking (R-45).
  process.stdout.write(JSON.stringify(event) + '\n');
}

/**
 * Emit a WARN log for Redis failure, rate-limited to once per second per
 * namespace to prevent log flooding (§4, F-01).
 *
 * @param {object} params - Same as log()
 */
function warnRateLimited(params) {
  const now = Date.now();
  const last = warnLogLastEmittedAt.get(params.namespace) || 0;
  if (now - last >= WARN_LOG_RATE_LIMIT_MS) {
    warnLogLastEmittedAt.set(params.namespace, now);
    log({ ...params, level: 'WARN' });
  }
}

module.exports = { log, warnRateLimited };
```

**Compliance note:** R-44 (all required fields present), R-45 (identifier logged; PII flag documented), R-46 (WARN rate-limited to 1/sec per namespace), R-47 (log level for allowed requests is a decision point — see Open Items).

### `src/redisClient.js`
**Satisfies: R-18, R-01**

```javascript
'use strict';

// §4, F-01 / F-02 — Redis client configuration requirements.
// The rate limiter does not own the Redis connection (§1). This factory is
// provided as a reference implementation showing the REQUIRED configuration.
// Callers who construct their own ioredis client MUST apply these settings.
//
// Required settings per design:
//   - lazyConnect: true        — do not connect on construction; connect on first command
//   - enableOfflineQueue: false — do not queue commands while disconnected;
//                                 queued commands would execute in a burst on reconnect,
//                                 producing false "all clear" on stale data (§4, F-01)
//   - connectTimeout: 200      — connection timeout ≤ 200ms (§4, F-01)
//   - commandTimeout: 100      — client-side command timeout 100ms (§4, F-02)
//                                 enforced client-side, not server-side

const Redis = require('ioredis');

/**
 * Create an ioredis client with the configuration required by §4, F-01/F-02.
 * This is a reference factory. The rate limiter accepts any pre-constructed
 * ioredis client via options.redisClient — it does not require use of this factory.
 * However, any client passed in MUST have enableOfflineQueue: false.
 *
 * @param {object} redisOptions - ioredis connection options (host, port, password, etc.)
 * @returns {import('ioredis').Redis}
 */
function createRedisClient(redisOptions = {}) {
  return new Redis({
    ...redisOptions,
    // R-18: Required configuration per §4, F-01
    lazyConnect: true,
    enableOfflineQueue: false,
    connectTimeout: 200,
    // R-19: Client-side command timeout per §4, F-02
    commandTimeout: 100,
  });
}

module.exports = { createRedisClient };
```

**Compliance note:** R-18 (lazyConnect, enableOfflineQueue: false, connectTimeout ≤ 200ms), R-19 (commandTimeout: 100ms), R-01 (rate limiter does not own the connection).

### `src/identifiers.js`
**Satisfies: R-58 (D-03), R-02**

```javascript
'use strict';

// §7, Decision Point D-03 — Identifier extraction.
//
// DECISION POINT D-03: The design specifies that the real client IP must be
// extracted from X-Forwarded-For or a trusted header, NOT from
// req.socket.remoteAddress (which will be the proxy IP). The design explicitly
// states: "This extraction logic must be implemented and audited before
// deployment."
//
// DEFAULT IMPLEMENTED: Extract the leftmost IP from X-Forwarded-For.
//
// ⚠️  SECURITY WARNING: The leftmost IP in X-Forwarded-For is client-supplied
// and can be spoofed if your proxy does not strip or overwrite it. This
// implementation is a STARTING POINT, not a production-ready default.
//
// REQUIRED BEFORE DEPLOYMENT:
//   1. Confirm with your infrastructure team which header your reverse proxy
//      sets and whether it is trustworthy.
//   2. If using AWS ALB: use the last IP in X-Forwarded-For (ALB appends the
//      real IP; client cannot spoof it).
//   3. If using Cloudflare: use CF-Connecting-IP header.
//   4. If using nginx with real_ip_module: configure trusted proxy ranges and
//      use req.socket.remoteAddress only after real_ip resolution.
//   5. Audit this function with your security team before go-live.
//
// This function is provided as a convenience. The operator MUST supply their
// own identifierFn via options if this default is not appropriate.

/**
 * Extract the leftmost IP from X-Forwarded-For.
 *
 * ⚠️  See D-03 warning above. Audit before production use.
 *
 * @param {import('http').IncomingMessage} req
 * @returns {string|null}
 */
function extractIpFromXForwardedFor(req) {
  const header = req.headers['x-forwarded-for'];
  if (!header) {
    return null;
  }
  // X-Forwarded-For: client, proxy1, proxy2
  // Take the first (leftmost) value.
  // ⚠️  D-03: This is spoofable. See module-level warning.
  const firstIp = header.split(',')[0].trim();
  return firstIp || null;
}

module.exports = { extractIpFromXForwardedFor };
```

**Compliance note:** R-58 (D-03) — identifier extraction implemented with explicit audit flag. R-02 — no authentication logic.

### `src/rateLimiter.js`
**Satisfies: R-03–R-09, R-13–R-17, R-20–R-23, R-25–R-55 (primary implementation)**

```javascript
'use strict';

const { evalScript, loadScript } = require('./luaScript');
const { METRIC_NAMES, createMetricsEmitter } = require('./metrics');
const { log, warnRateLimited } = require('./logger');

// ---------------------------------------------------------------------------
// Input validation helpers
// ---------------------------------------------------------------------------

/**
 * Validate the options object at initialization time.
 * Hard errors here are preferable to silent misconfiguration at request time.
 * §5 — all required fields must be present and well-typed.
 *
 * @param {object} options
 * @throws {Error} on invalid configuration
 */
function validateOptions(options) {
  if (!options || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be a non-null object');
  }

  // namespace: string, no spaces, max 64 chars
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must not exceed 64 characters');
  }

  // limit: positive integer
  if (!Number.isInteger(options.limit) || options.limit <= 0) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: positive integer
  if (!Number.isInteger(options.windowSizeSeconds) || options.windowSizeSeconds <= 0) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: synchronous function
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: must be provided (rate limiter does not own the connection)
  if (!options.redisClient || typeof options.redisClient !== 'object') {
    throw new Error('[ratelimit] options.redisClient must be a pre-constructed ioredis client');
  }

  // onLimitExceeded: optional, but if provided must be a function
  if (options.onLimitExceeded !== undefined && typeof options.onLimitExceeded !== 'function') {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // failOpen: required boolean — no default.
  // §7, D-01: The operator must make a conscious choice. No default is provided.
  // Forcing an explicit declaration prevents silent misconfiguration.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true (fail-open) or false (fail-closed). ' +
      'There is no default. See §7, D-01 in the design document.'
    );
  }
}

// ---------------------------------------------------------------------------
// Key construction
// ---------------------------------------------------------------------------

/**
 * Build the Redis key for a given window.
 * §3 — Key format: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

// ---------------------------------------------------------------------------
// Window arithmetic
// ---------------------------------------------------------------------------

/**
 * Compute sliding window values from the current time.
 * §2 — Algorithm arithmetic.
 *
 * NOTE: Uses Date.now() internally. The implementing engineer must verify
 * this arithmetic in a unit test with fixed timestamps per §3 note.
 *
 * @param {number} windowSizeSeconds
 * @returns {{
 *   nowSeconds: number,
 *   currentWindow: number,
 *   previousWindow: number,
 *   elapsedFraction: number
 * }}
 */
function computeWindowValues(windowSizeSeconds) {
  // R-14: window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)
  const nowMs = Date.now();
  const nowSeconds = nowMs / 1000; // floating-point seconds
  const currentWindow = Math.floor(nowSeconds / windowSizeSeconds);
  const previousWindow = currentWindow - 1;
  // R-07: elapsed_fraction = (now % window_size) / window_size
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;

  return { nowSeconds, currentWindow, previousWindow, elapsedFraction };
}

// ---------------------------------------------------------------------------
// Response helpers
// ---------------------------------------------------------------------------

/**
 * Compute the Retry-After value in seconds.
 * §5 — Retry-After = ceil((current_window + 1) * window_size - now)
 * Minimum value is 1 (a value of 0 is a defect per §6.3).
 *
 * @param {number} currentWindow
 *
