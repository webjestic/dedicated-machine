# Production Rate Limiter in Node.js

Let me work through this properly — requirements first, then implementation, then operational gaps.

---

## Phase 1: Requirements Design

### Functional Requirements
- Limit requests per identifier (IP, user ID, API key) within a sliding window
- Support multiple rate limit tiers (different limits for different consumers)
- Return standard rate limit headers on every response
- Reject requests that exceed limits with `429 Too Many Requests`
- Allow limits to recover as the window slides

### Non-Functional Requirements
- **Correctness**: No race conditions allowing bursts past the limit
- **Performance**: < 5ms overhead per request
- **Availability**: Rate limiter failure should not take down the API (fail-open configurable)
- **Accuracy**: Sliding window, not fixed window (fixed windows allow 2x burst at window boundaries)
- **Distributed**: Works correctly across multiple API server instances

### Algorithm Selection

| Algorithm | Pros | Cons |
|---|---|---|
| Fixed Window | Simple, fast | 2x burst at boundary |
| Sliding Window Log | Perfectly accurate | High memory (stores every timestamp) |
| Sliding Window Counter | Accurate, memory efficient | Slight approximation |
| Token Bucket | Handles bursts well | Complex distributed sync |
| Leaky Bucket | Smooth output rate | Doesn't handle legitimate bursts |

**Decision: Sliding Window Counter** — best balance of accuracy, memory efficiency, and atomic Redis implementation.

### Sliding Window Counter — How It Works

```
Window size: 60 seconds
Current time: 75s
Current window: [60s-120s], count = 8
Previous window: [0s-60s], count = 15
Limit: 20

Overlap of previous window into current = (60 - 15) / 60 = 75%
Weighted count = (15 * 0.75) + 8 = 11.25 + 8 = 19.25

19.25 < 20 → ALLOW
```

### `src/utils/keyGenerator.js`

```javascript
const crypto = require('crypto');

/**
 * Key generation strategies.
 *
 * Keys must be:
 * - Consistent: same identifier always produces same key
 * - Scoped: different endpoints/tiers don't share counters
 * - Safe: no user-controlled data can manipulate key structure
 */

/**
 * Hash a value so user-controlled input can't inject
 * Redis key separators or exceed key length limits.
 */
function safeHash(value) {
  return crypto
    .createHash('sha256')
    .update(String(value))
    .digest('hex')
    .slice(0, 16); // 16 hex chars = 64 bits, collision-safe at this scale
}

/**
 * Extract real client IP, accounting for proxies.
 *
 * SECURITY NOTE: X-Forwarded-For is trivially spoofable unless your
 * infrastructure sets it. Only trust it if you control the proxy layer.
 * Set `trustProxy` based on your actual deployment topology.
 */
function getClientIp(req, { trustProxy = false } = {}) {
  if (trustProxy) {
    const forwarded = req.headers['x-forwarded-for'];
    if (forwarded) {
      // X-Forwarded-For: client, proxy1, proxy2
      // Take the first (leftmost) IP — the original client
      const clientIp = forwarded.split(',')[0].trim();
      if (isValidIp(clientIp)) return clientIp;
    }
  }
  return req.socket?.remoteAddress || 'unknown';
}

function isValidIp(ip) {
  // Basic validation — prevents header injection
  return /^[\d.:\[\]a-fA-F]+$/.test(ip) && ip.length < 50;
}

const KeyStrategies = {
  /**
   * Key by IP address.
   * Use for anonymous/unauthenticated traffic.
   */
  byIp(req, windowKey, options = {}) {
    const ip = getClientIp(req, options);
    return `rl:ip:${safeHash(ip)}:${windowKey}`;
  },

  /**
   * Key by authenticated user ID.
   * Use when you have a verified user identity.
   * Falls back to IP if no user present.
   */
  byUser(req, windowKey, options = {}) {
    if (req.user?.id) {
      return `rl:user:${safeHash(req.user.id)}:${windowKey}`;
    }
    return KeyStrategies.byIp(req, windowKey, options);
  },

  /**
   * Key by API key.
   * Use for machine-to-machine / developer API access.
   */
  byApiKey(req, windowKey) {
    const apiKey =
      req.headers['x-api-key'] ||
      req.query?.api_key;

    if (apiKey) {
      return `rl:apikey:${safeHash(apiKey)}:${windowKey}`;
    }
    return KeyStrategies.byIp(req, windowKey);
  },

  /**
   * Key by IP + endpoint.
   * Use for endpoint-specific limits (e.g., /auth/login).
   * Prevents one endpoint's limit from blocking all API access.
   */
  byIpAndEndpoint(req, windowKey, options = {}) {
    const ip = getClientIp(req, options);
    const endpoint = safeHash(req.route?.path || req.path);
    return `rl:ip+ep:${safeHash(ip)}:${endpoint}:${windowKey}`;
  },
};

module.exports = { KeyStrategies, getClientIp };
```

### `src/core/RedisStore.js`

```javascript
const { createClient } = require('redis');

/**
 * Lua script for atomic sliding window counter.
 *
 * Why Lua?
 * - Executes atomically on Redis — no race conditions
 * - Single round trip regardless of window count
 * - No WATCH/MULTI/EXEC retry loop needed
 *
 * Script receives per-window keys and config, returns
 * [allowed, currentCount, ttlSeconds] for the most restrictive window.
 *
 * KEYS: [currentKey, previousKey] per window
 * ARGV: [windowSizeSeconds, maxRequests, currentWindowStart, now] per window
 *       + [increment] as final arg
 */
const SLIDING_WINDOW_SCRIPT = `
local increment = tonumber(ARGV[#ARGV])
local now = tonumber(ARGV[#ARGV - 1])

-- We'll track the most restrictive window result
local min_remaining = math.huge
local total_windows = (#KEYS) / 2
local blocked = false
local most_restrictive_reset = 0
local most_restrictive_count = 0
local most_restrictive_limit = 0

-- First pass: check all windows without incrementing
-- to avoid incrementing if we're going to block
for i = 1, total_windows do
  local currentKey  = KEYS[(i - 1) * 2 + 1]
  local previousKey = KEYS[(i - 1) * 2 + 2]

  local base = (i - 1) * 4
  local windowSize    = tonumber(ARGV[base + 1])
  local maxRequests   = tonumber(ARGV[base + 2])
  local windowStart   = tonumber(ARGV[base + 3])
  -- ARGV[base + 4] is 'now', same for all

  local currentCount  = tonumber(redis.call('GET', currentKey)) or 0
  local previousCount = tonumber(redis.call('GET', previousKey)) or 0

  -- How far into the current window are we? (0.0 to 1.0)
  local elapsed   = now - windowStart
  local overlap   = (windowSize - elapsed) / windowSize

  -- Weight the previous window by how much it overlaps into current
  local weighted  = (previousCount * overlap) + currentCount
  local remaining = maxRequests - weighted

  if remaining < min_remaining then
    min_remaining = remaining
    most_restrictive_reset = windowStart + windowSize
    most_restrictive_count = math.ceil(weighted)
    most_restrictive_limit = maxRequests
  end

  if weighted + increment > maxRequests then
    blocked = true
  end
end

if blocked then
  return {
    0,
    most_restrictive_count,
    most_restrictive_limit,
    most_restrictive_reset,
    math.max(0, math.floor(min_remaining))
  }
end

-- Second pass: commit the increment to all windows
for i = 1, total_windows do
  local currentKey = KEYS[(i - 1) * 2 + 1]
  local base = (i - 1) * 4
  local windowSize = tonumber(ARGV[base + 1])

  redis.call('INCR', currentKey)
  -- TTL = 2x window so previous window data survives into next window
  redis.call('EXPIRE', currentKey, windowSize * 2)
end

return {
  1,
  most_restrictive_count + increment,
  most_restrictive_limit,
  most_restrictive_reset,
  math.max(0, math.floor(min_remaining) - increment)
}
`;

class RedisStore {
  constructor(options = {}) {
    this.options = {
      url: options.url || process.env.REDIS_URL || 'redis://localhost:6379',
      socket: {
        connectTimeout: options.connectTimeout || 2000,
        // Don't wait forever if Redis is unreachable
        reconnectStrategy: (retries) => {
          if (retries > 3) return new Error('Redis unreachable');
          return Math.min(retries * 100, 500);
        },
      },
      ...options.redisOptions,
    };

    this.client = null;
    this.scriptSha = null;
    this.connected = false;
    this._connecting = false;
  }

  async connect() {
    if (this.connected || this._connecting) return;
    this._connecting = true;

    try {
      this.client = createClient(this.options);

      this.client.on('error', (err) => {
        // Log but don't crash — handled at call sites
        console.error('[RedisStore] Redis error:', err.message);
        this.connected = false;
      });

      this.client.on('ready', () => {
        this.connected = true;
        console.log('[RedisStore] Redis connected');
      });

      this.client.on('reconnecting', () => {
        this.connected = false;
      });

      await this.client.connect();

      // Pre-load script into Redis for efficient execution
      this.scriptSha = await this.client.scriptLoad(SLIDING_WINDOW_SCRIPT);
      this.connected = true;
    } finally {
      this._connecting = false;
    }
  }

  async disconnect() {
    if (this.client) {
      await this.client.quit();
      this.connected = false;
    }
  }

  /**
   * Execute the sliding window check.
   *
   * @param {Array<{currentKey, previousKey, windowSizeSeconds, maxRequests, windowStart}>} windows
   * @param {number} now - Current timestamp in seconds
   * @param {number} increment - How much to increment (almost always 1)
   * @returns {Promise<{allowed, count, limit, resetTime, remaining}>}
   */
  async checkLimit(windows, now, increment = 1) {
    if (!this.connected) {
      throw new Error('Redis not connected');
    }

    const keys = [];
    const args = [];

    for (const w of windows) {
      keys.push(w.currentKey, w.previousKey);
      args.push(
        String(w.windowSizeSeconds),
        String(w.maxRequests),
        String(w.windowStart),
        String(now),
      );
    }
    args.push(String(increment));
    args.push(String(now)); // 'now' as second-to-last for script

    // Reorder: script expects [windowArgs..., now, increment]
    // Rebuild correctly:
    const finalArgs = [];
    for (const w of windows) {
      finalArgs.push(
        String(w.windowSizeSeconds),
        String(w.maxRequests),
        String(w.windowStart),
        String(now),
      );
    }
    finalArgs.push(String(now));    // ARGV[#ARGV - 1]
    finalArgs.push(String(increment)); // ARGV[#ARGV]

    let result;
    try {
      // Use EVALSHA for efficiency (script already loaded)
      result = await this.client.evalSha(this.scriptSha, {
        keys,
        arguments: finalArgs,
      });
    } catch (err) {
      if (err.message?.includes('NOSCRIPT')) {
        // Script was flushed (e.g., Redis restart) — reload and retry
        this.scriptSha = await this.client.scriptLoad(SLIDING_WINDOW_SCRIPT);
        result = await this.client.evalSha(this.scriptSha, {
          keys,
          arguments: finalArgs,
        });
      } else {
        throw err;
      }
    }

    const [allowed, count, limit, resetTime, remaining] = result.map(Number);

    return {
      allowed: allowed === 1,
      count,
      limit,
      resetTime,   // Unix timestamp seconds when window resets
      remaining: Math.max(0, remaining),
    };
  }

  isHealthy() {
    return this.connected;
  }
}

module.exports = { RedisStore };
```

### `src/core/MemoryStore.js`

```javascript
/**
 * In-memory store for:
 * - Local development without Redis
 * - Unit testing
 * - Single-instance deployments
 *
 * WARNING: Does NOT work correctly across multiple processes/servers.
 * Use RedisStore in any multi-instance deployment.
 */
class MemoryStore {
  constructor() {
    this.windows = new Map();
    // Periodic cleanup to prevent unbounded memory growth
    this._cleanupInterval = setInterval(() => this._cleanup(), 60_000);
    // Don't keep process alive just for cleanup
    this._cleanupInterval.unref();
  }

  async connect() {
    // No-op — nothing to connect to
  }

  async disconnect() {
    clearInterval(this._cleanupInterval);
  }

  async checkLimit(windows, now, increment = 1) {
    // Check all windows first before mutating
    let blocked = false;
    let minRemaining = Infinity;
    let mostRestrictiveReset = 0;
    let mostRestrictiveCount = 0;
    let mostRestrictiveLimit = 0;

    const snapshots = windows.map((w) => {
      const currentCount  = this._get(w.currentKey)  || 0;
      const previousCount = this._get(w.previousKey) || 0;

      const elapsed  = now - w.windowStart;
      const overlap  = (w.windowSizeSeconds - elapsed) / w.windowSizeSeconds;
      const weighted = (previousCount * overlap) + currentCount;
      const remaining = w.maxRequests - weighted;

      if (remaining < minRemaining) {
        minRemaining = remaining;
        mostRestrictiveReset = w.windowStart + w.windowSizeSeconds;
        mostRestrictiveCount = Math.ceil(weighted);
        mostRestrictiveLimit = w.maxRequests;
      }

      if (weighted + increment > w.maxRequests) {
        blocked = true;
      }

      return { w, currentCount };
    });

    if (blocked) {
      return {
        allowed: false,
        count: mostRestrictiveCount,
        limit: mostRestrictiveLimit,
        resetTime: mostRestrictiveReset,
        remaining: Math.max(0, Math.floor(minRemaining)),
      };
    }

    // Commit increments
    for (const { w } of snapshots) {
      this._increment(w.currentKey, w.windowSizeSeconds * 2);
    }

    return {
      allowed: true,
      count: mostRestrictiveCount + increment,
      limit: mostRestrictiveLimit,
      resetTime: mostRestrictiveReset,
      remaining: Math.max(0, Math.floor(minRemaining) - increment),
    };
  }

  _get(key) {
    const entry = this.windows.get(key);
    if (!entry) return null;
    if (Date.now() / 1000 > entry.expiresAt) {
      this.windows.delete(key);
      return null;
    }
    return entry.count;
  }

  _increment(key, ttlSeconds) {
    const now = Date.now() / 1000;
    const entry = this.windows.get(key);
    if (entry && now < entry.expiresAt) {
      entry.count += 1;
    } else {
      this.windows.set(key, { count: 1, expiresAt: now + ttlSeconds });
    }
  }

  _cleanup() {
    const now = Date.now() / 1000;
    for (const [key, entry] of this.windows.entries()) {
      if (now > entry.expiresAt) {
        this.windows.delete(key);
      }
    }
  }

  isHealthy() {
    return true;
  }
}

module.exports = { MemoryStore };
```

### `src/core/RateLimiter.js`

```javascript
const { TIERS } = require('../config/tiers');

class RateLimiter {
  /**
   * @param {object} store - RedisStore or MemoryStore instance
   * @param {object} options
   * @param {boolean} options.failOpen - Global fail-open override
   */
  constructor(store, options = {}) {
    this.store = store;
    this.globalFailOpen = options.failOpen ?? true;
  }

  /**
   * Check and record a request against rate limits.
   *
   * @param {string} tierName - Must exist in TIERS config
   * @param {Function} keyGenerator - (windowKey) => string
   * @returns {Promise<RateLimitResult>}
   */
  async check(tierName, keyGenerator) {
    const tier = TIERS[tierName];
    if (!tier) {
      throw new Error(`Unknown rate limit tier: "${tierName}"`);
    }

    const now = Math.floor(Date.now() / 1000);

    // Build window descriptors for all configured windows in this tier
    const windows = tier.windows.map((config) => {
      const windowStart = Math.floor(now / config.windowSizeSeconds)
        * config.windowSizeSeconds;

      const previousWindowStart = windowStart - config.windowSizeSeconds;

      // windowKey encodes the window size so different windows
      // don't accidentally share Redis keys
      const windowKey = `w${config.windowSizeSeconds}`;

      return {
        currentKey:        keyGenerator(`${windowKey}:cur:${windowStart}`),
        previousKey:       keyGenerator(`${windowKey}:prev:${previousWindowStart}`),
        windowSizeSeconds: config.windowSizeSeconds,
        maxRequests:       config.maxRequests,
        windowStart,
      };
    });

    let result;
    try {
      result = await this.store.checkLimit(windows, now);
    } catch (err) {
      return this._handleStoreFailure(err, tier, tierName);
    }

    return {
      ...result,
      tier: tierName,
      storeError: null,
    };
  }

  _handleStoreFailure(err, tier, tierName) {
    const failOpen = tier.failOpen ?? this.globalFailOpen;

    console.error(
      `[RateLimiter] Store failure for tier "${tierName}": ${err.message}`,
      { failOpen }
    );

    if (failOpen) {
      // Allow the request but signal that limits weren't enforced
      return {
        allowed: true,
        count: 0,
        limit: 0,
        resetTime: 0,
        remaining: 0,
        tier: tierName,
        storeError: err.message,
      };
    } else {
      // Fail closed — block the request
      return {
        allowed: false,
        count: 0,
        limit: 0,
        resetTime: 0,
        remaining: 0,
        tier: tierName,
        storeError: err.message,
      };
    }
  }
}

module.exports = { RateLimiter };
```

### `src/middleware/rateLimitMiddleware.js`

```javascript
const { KeyStrategies } = require('../utils/keyGenerator');
const { TIERS } = require('../config/tiers');

/**
 * Standard rate limit response headers.
 * Following the IETF draft standard:
 * https://datatracker.ietf.org/doc/html/draft-ietf-httpapi-ratelimit-headers
 */
function setRateLimitHeaders(res, result) {
  if (!result || result.limit === 0) return; // Store error with fail-open

  res.setHeader('RateLimit-Limit',     result.limit);
  res.setHeader('RateLimit-Remaining', Math.max(0, result.remaining));
  res.setHeader('RateLimit-Reset',     result.resetTime);

  // Retry-After only on rejection, in seconds
  if (!result.allowed) {
    const retryAfter = Math.max(1, result.resetTime - Math.floor(Date.now() / 1000));
    res.setHeader('Retry-After', retryAfter);
  }
}

/**
 * Determine which tier a request belongs to.
 * Override this function to implement your own tier resolution logic.
 */
function resolveTier(req) {
  // API key takes highest precedence
  if (req.headers['x-api-key']) {
    // In production: look up API key in DB to determine tier
    // For now: presence of key = authenticated tier
    return 'authenticated';
  }

  if (req.user?.tier && TIERS[req.user.tier]) {
    return req.user.tier;
  }

  if (req.user?.id) {
    return 'authenticated';
  }

  return 'anonymous';
}

/**
 * Determine which key strategy to use for a request.
 */
function resolveKeyStrategy(req, tier) {
  if (req.headers['x-api-key']) return KeyStrategies.byApiKey;
  if (req.user?.id)             return KeyStrategies.byUser;
  return KeyStrategies.byIp;
}

/**
 * Factory function — creates rate limit middleware.
 *
 * @param {RateLimiter} rateLimiter - Configured RateLimiter instance
 * @param {object} options
 * @param {string}   options.tier          - Force a specific tier (optional)
 * @param {Function} options.keyStrategy   - Override key strategy (optional)
 * @param {Function} options.onLimitReached - Custom handler for 429 (optional)
 * @param {object}   options.trustProxy    - Trust X-Forwarded-For header
 */
function createRateLimitMiddleware(rateLimiter, options = {}) {
  const {
    tier: forcedTier,
    keyStrategy: forcedKeyStrategy,
    onLimitReached,
    trustProxy = false,
  } = options;

  return async function rateLimitMiddleware(req, res, next) {
    const tier = forcedTier || resolveTier(req);
    const keyStrategy = forcedKeyStrategy || resolveKeyStrategy(req, tier);

    let result;
    try {
      result = await rateLimiter.check(
        tier,
        (windowKey) => keyStrategy(req, windowKey, { trustProxy }),
      );
    } catch (err) {
      // Unexpected error in rate limiter itself (not store failure)
      // Store failures are handled inside rateLimiter.check()
      console.error('[RateLimitMiddleware] Unexpected error:', err);
      // Fail open — don't block traffic due to our own bug
      return next();
    }

    // Always set headers, even on rejection
    setRateLimitHeaders(res, result);

    // Log store errors for alerting (but don't block the request if fail-open)
    if (result.storeError) {
      console.error('[RateLimitMiddleware] Proceeding without rate limiting:', {
        storeError: result.storeError,
        tier,
        path: req.path,
      });
    }

    if (!result.allowed) {
      if (onLimitReached) {
        return onLimitReached(req, res, next, result);
      }

      return res.status(429).json({
        error: 'Too Many Requests',
        message: 'Rate limit exceeded. Please slow down.',
        retryAfter: res.getHeader('Retry-After'),
      });
    }

    next();
  };
}

/**
 * Convenience: middleware for sensitive endpoints (auth, password reset, etc.)
 * Uses endpoint-scoped keys so limits don't bleed across routes.
 */
function createSensitiveEndpointMiddleware(rateLimiter, options = {}) {
  return createRateLimitMiddleware(rateLimiter, {
    tier: 'sensitive',
    keyStrategy: KeyStrategies.byIpAndEndpoint,
    onLimitReached: (req, res) => {
      res.status(429).json({
        error: 'Too Many Requests',
        message: 'Too many attempts. Please try again later.',
        retryAfter: res.getHeader('Retry-After'),
      });
    },
    ...options,
  });
}

module.exports = {
  createRateLimitMiddleware,
  createSensitiveEndpointMiddleware,
  setRateLimitHeaders,
  resolveTier,
};
```
