# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview
- [ ] R-01: Middleware library interface (not standalone service — see Open Items)
- [ ] R-02: Per-identifier rate limiting
- [ ] R-03: Sliding window algorithm
- [ ] R-04: Redis-backed
- [ ] R-05: Structured 429 response on limit exceeded
- [ ] R-06: Identifier is passed in or extracted — no auth logic

### §2 Algorithm
- [ ] R-07: Approximated sliding window counter (two fixed windows)
- [ ] R-08: `current_window = floor(now / window_size)`
- [ ] R-09: `previous_window = current_window - 1`
- [ ] R-10: `elapsed_fraction = (now % window_size) / window_size`
- [ ] R-11: `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] R-12: Reject if `weighted_count >= limit`
- [ ] R-13: Increment `current_count` only if allowed
- [ ] R-14: Atomicity via Lua script (`EVAL`/`EVALSHA`)
- [ ] R-15: Lua script body must match normative spec exactly
- [ ] R-16: Return value structure: `[allowed, current_count, previous_count, weighted]`

### §3 Storage
- [ ] R-17: Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] R-18: `window_id = Math.floor(Date.now() / 1000 / window_size_in_seconds)`
- [ ] R-19: TTL = `window_size * 2` seconds, set only on first write (`new_count == 1`)
- [ ] R-20: Redis string type (integer counter)
- [ ] R-21: `maxmemory-policy` must not be `noeviction` — documented requirement (config, not code)

### §4 Failure Modes
- [ ] R-22: F-01 — Redis connection unavailable: fail-open or fail-closed per `failOpen` flag
- [ ] R-23: F-01 — WARN log, rate-limited to 1/second during unavailability
- [ ] R-24: F-01 — Emit `ratelimit.redis.unavailable` metric
- [ ] R-25: F-01 — `ioredis` with `lazyConnect: true`, `enableOfflineQueue: false`
- [ ] R-26: F-02 — Redis command timeout (100ms), enforced client-side
- [ ] R-27: F-02 — Emit `ratelimit.redis.timeout` metric (distinct from unavailable)
- [ ] R-28: F-03 — Unexpected Redis data: ERROR log, fail-open/closed, no crash, no leak to client
- [ ] R-29: F-04 — No active defense; detection via metrics/logs (window_id in log)
- [ ] R-30: F-05 — `allkeys-lru` documented; `noeviction` prohibited (config, not code)
- [ ] R-31: F-06 — `identifierFn` returns null: ERROR log, fail-open for that request, no silent fallback, emit `ratelimit.identifier.missing`
- [ ] R-32: F-07 — `EVALSHA` with `NOSCRIPT` fallback to `EVAL`; re-cache SHA; emit `ratelimit.redis.script_miss`
- [ ] R-33: F-08 — No special handling; state in Redis

### §5 API Contract
- [ ] R-34: `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] R-35: Options: `namespace` (string, no spaces, max 64 chars)
- [ ] R-36: Options: `limit` (positive integer)
- [ ] R-37: Options: `windowSizeSeconds` (positive integer)
- [ ] R-38: Options: `identifierFn` (pure, synchronous `(req) => string | null`)
- [ ] R-39: Options: `redisClient` (pre-constructed `ioredis` instance)
- [ ] R-40: Options: `onLimitExceeded` (optional override for 429 handler)
- [ ] R-41: Options: `failOpen` (boolean, required, no default)
- [ ] R-42: Allowed request: call `next()`
- [ ] R-43: Allowed request headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window`
- [ ] R-44: `X-RateLimit-Remaining` floored at 0
- [ ] R-45: `X-RateLimit-Reset = (current_window + 1) * window_size`
- [ ] R-46: Rejected: HTTP 429
- [ ] R-47: Rejected headers: `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining: 0`, `X-RateLimit-Reset`, `X-RateLimit-Window`, `Content-Type: application/json`
- [ ] R-48: `Retry-After = ceil((current_window + 1) * window_size - now)`, minimum 1
- [ ] R-49: Rejected body: `{ error, message, retryAfter, limit, window }` — exact field names
- [ ] R-50: `retryAfter` in body matches `Retry-After` header exactly
- [ ] R-51: Response body must not include identifier, internal keys, Redis state, stack traces
- [ ] R-52: Fail-closed Redis error: HTTP 503, `Retry-After: 10`, structured body
- [ ] R-53: `isAlive()` function exported
- [ ] R-54: `isReady()` async function exported
- [ ] R-55: Readiness: Redis PING with 200ms timeout
- [ ] R-56: Readiness: fail-open → 200 with degraded status; fail-closed → 503

### §6 Operational Requirements
- [ ] R-57: Metrics: `ratelimit.request.total` (counter, tags: namespace, result)
- [ ] R-58: Metrics: `ratelimit.request.allowed` (counter, tag: namespace)
- [ ] R-59: Metrics: `ratelimit.request.rejected` (counter, tag: namespace)
- [ ] R-60: Metrics: `ratelimit.redis.latency_ms` (histogram, tag: namespace)
- [ ] R-61: Metrics: `ratelimit.redis.error` (counter, tags: namespace, error_type)
- [ ] R-62: Metrics: `ratelimit.redis.script_miss` (counter, tag: namespace)
- [ ] R-63: Metrics: `ratelimit.failopen.request` (counter, tag: namespace)
- [ ] R-64: Metrics: `ratelimit.identifier.missing` (counter, tag: namespace)
- [ ] R-65: Structured JSON logging on every rate limiter event
- [ ] R-66: Log fields: timestamp, level, component, namespace, identifier, result, weighted_count, limit, window_id, redis_latency_ms, request_id
- [ ] R-67: Middleware must never crash the Node.js process
- [ ] R-68: Middleware must never block the event loop
- [ ] R-69: Fail-open must be explicit and logged (WARN); silent fail-open is not acceptable
- [ ] R-70: Recovery from Redis outage must be automatic (no cached "down" state)

### `src/logger.js` — Structured JSON Logging

*Satisfies: R-65, R-66*

```javascript
'use strict';

/**
 * Structured JSON logger for the rate limiter.
 *
 * Design §6.1 requires every log event to be structured JSON with a fixed set
 * of fields. This module is the single emission point. It does NOT wrap a
 * third-party logger (winston, pino, etc.) because the design does not specify
 * one. It writes to process.stdout/stderr directly.
 *
 * DECISION POINT (D-05 adjacent): The design specifies structured JSON logging
 * but does not specify a logging library. This implementation writes raw JSON
 * to stdout (INFO/WARN) and stderr (ERROR). If the operator requires a specific
 * logging library, replace the `_emit` function below. The field contract is
 * normative and must not change.
 *
 * §6.1 NOTE TO OPERATOR: If `identifier` contains PII (email addresses used as
 * API keys, etc.), the logging pipeline must hash or mask it before storage.
 * The rate limiter logs the raw identifier as required by the design for
 * incident investigation. Masking is an infrastructure concern per §6.1.
 *
 * §6.1 LOG VOLUME NOTE: INFO-level logging of every allowed request will be
 * high-volume. Confirm with the operator whether allowed-request logs are
 * required. This implementation logs all events at INFO. To suppress allowed-
 * request logs, change the `result === 'allowed'` branch in middleware.js to
 * call `logger.debug` instead of `logger.info` — but only after operator
 * confirmation, as the design states rejections and errors must always be
 * logged.
 */

/**
 * @typedef {Object} LogEvent
 * @property {'INFO'|'WARN'|'ERROR'} level
 * @property {string} namespace
 * @property {string|null} identifier
 * @property {'allowed'|'rejected'|'error'|'failopen'} result
 * @property {number|null} weighted_count
 * @property {number} limit
 * @property {number|null} window_id
 * @property {number|null} redis_latency_ms
 * @property {string|null} request_id
 * @property {string} [message]
 * @property {string} [error_type]
 * @property {*} [raw_redis_response]
 */

/**
 * Emits a single structured log line.
 * @param {'INFO'|'WARN'|'ERROR'} level
 * @param {Partial<LogEvent>} fields
 */
function _emit(level, fields) {
  const entry = {
    timestamp: new Date().toISOString(),
    level,
    component: 'ratelimit',
    ...fields,
  };
  const line = JSON.stringify(entry);
  if (level === 'ERROR') {
    process.stderr.write(line + '\n');
  } else {
    process.stdout.write(line + '\n');
  }
}

const logger = {
  info:  (fields) => _emit('INFO',  fields),
  warn:  (fields) => _emit('WARN',  fields),
  error: (fields) => _emit('ERROR', fields),
};

module.exports = logger;
```

**Compliance note:** R-65, R-66 satisfied. The `component: 'ratelimit'` field is hardcoded per the design's example. The `timestamp` is ISO8601. All required fields are passed through from call sites in `middleware.js`.

### `src/algorithm.js` — Sliding Window Algorithm and Lua Script

*Satisfies: R-07–R-16, R-17–R-20, R-32*

```javascript
'use strict';

/**
 * Sliding window algorithm implementation.
 *
 * This module owns:
 *   - The normative Lua script (§2, must not be altered without re-running §6.5)
 *   - Key construction (§3)
 *   - Window arithmetic (§2)
 *   - EVALSHA with NOSCRIPT fallback (§4, F-07)
 *
 * The Lua script below is reproduced verbatim from §2 of the design document.
 * DO NOT ALTER THE LOGIC OF THIS SCRIPT without re-running the full load test
 * suite defined in §6.5. The comment is normative per the design.
 */

const crypto = require('crypto');

// ---------------------------------------------------------------------------
// Normative Lua script (§2 — must not be altered without re-running §6.5)
// ---------------------------------------------------------------------------
const LUA_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

// Pre-compute SHA1 of the script for EVALSHA calls.
// This is the SHA1 Redis will use after SCRIPT LOAD or first EVAL.
// We compute it locally to use in EVALSHA before the first EVAL.
// NOTE: Redis computes SHA1 of the exact script bytes. This must match.
const LUA_SCRIPT_SHA = crypto
  .createHash('sha1')
  .update(LUA_SCRIPT)
  .digest('hex');

// ---------------------------------------------------------------------------
// Window arithmetic (§2, §3)
// ---------------------------------------------------------------------------

/**
 * Compute all window-related values for a given moment in time.
 *
 * §3 NOTE: window_id = Math.floor(Date.now() / 1000 / window_size_in_seconds)
 * Date.now() returns milliseconds; divide by 1000 to get seconds first.
 * This arithmetic is verified in test/algorithm.test.js with fixed timestamps.
 *
 * @param {number} nowMs - Current time in milliseconds (Date.now())
 * @param {number} windowSizeSeconds - Window duration in seconds
 * @returns {{
 *   nowSeconds: number,
 *   currentWindow: number,
 *   previousWindow: number,
 *   elapsedFraction: number,
 *   windowResetTimestamp: number
 * }}
 */
function computeWindowValues(nowMs, windowSizeSeconds) {
  const nowSeconds = nowMs / 1000; // floating point seconds
  const currentWindow = Math.floor(nowSeconds / windowSizeSeconds);
  const previousWindow = currentWindow - 1;
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;
  // When the current window expires: (currentWindow + 1) * windowSizeSeconds
  const windowResetTimestamp = (currentWindow + 1) * windowSizeSeconds;

  return {
    nowSeconds,
    currentWindow,
    previousWindow,
    elapsedFraction,
    windowResetTimestamp,
  };
}

// ---------------------------------------------------------------------------
// Key construction (§3)
// ---------------------------------------------------------------------------

/**
 * Construct a Redis key for a given namespace, identifier, and window ID.
 *
 * Key format: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * §3 WARNING: An adversary who can forge arbitrary identifiers can cause
 * unbounded key creation. Identifier extraction must be hardened (§7, D-03).
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

// ---------------------------------------------------------------------------
// Redis script execution with EVALSHA / NOSCRIPT fallback (§4, F-07)
// ---------------------------------------------------------------------------

/**
 * Execute the rate limit Lua script against Redis.
 *
 * Implements F-07: uses EVALSHA for performance. On NOSCRIPT error (Redis
 * restarted, SCRIPT FLUSH executed), falls back to EVAL with the full script
 * body and re-caches the SHA. The caller is responsible for emitting the
 * ratelimit.redis.script_miss metric on NOSCRIPT recovery.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string} currentKey
 * @param {string} previousKey
 * @param {number} limit
 * @param {number} windowSizeSeconds
 * @param {number} nowSeconds
 * @param {number} elapsedFraction
 * @returns {Promise<[number, number, number, number]>}
 *   [allowed(0|1), currentCount, previousCount, weightedCount]
 * @throws Will throw if Redis returns an error other than NOSCRIPT, or if
 *   the EVAL fallback also fails. Caller must handle.
 */
async function executeRateLimitScript(
  redisClient,
  currentKey,
  previousKey,
  limit,
  windowSizeSeconds,
  nowSeconds,
  elapsedFraction,
) {
  const keys = [currentKey, previousKey];
  const args = [
    String(limit),
    String(windowSizeSeconds),
    String(nowSeconds),
    String(elapsedFraction),
  ];

  let result;
  let scriptMiss = false;

  try {
    // Attempt EVALSHA first (F-07: performance path)
    result = await redisClient.evalsha(LUA_SCRIPT_SHA, keys.length, ...keys, ...args);
  } catch (err) {
    if (err && err.message && err.message.startsWith('NOSCRIPT')) {
      // F-07: Script not in cache. Fall back to EVAL with full script body.
      // The script will be cached by Redis after this call.
      scriptMiss = true;
      result = await redisClient.eval(LUA_SCRIPT, keys.length, ...keys, ...args);
    } else {
      // Not a NOSCRIPT error — propagate to caller for failure mode handling.
      throw err;
    }
  }

  return { result, scriptMiss };
}

module.exports = {
  LUA_SCRIPT,
  LUA_SCRIPT_SHA,
  computeWindowValues,
  buildKey,
  executeRateLimitScript,
};
```

**Compliance note:** R-07–R-16 (algorithm), R-17–R-20 (key strategy, TTL via Lua), R-32 (F-07 NOSCRIPT fallback) satisfied. The Lua script is reproduced verbatim from §2. The `scriptMiss` flag is returned to the caller so it can emit the `ratelimit.redis.script_miss` metric (R-62).

### `src/redis-client.js` — ioredis Construction Helper

*Satisfies: R-25, R-26*

```javascript
'use strict';

/**
 * ioredis client construction helper.
 *
 * §4 F-01 REQUIREMENT: The Redis client must be configured with:
 *   - lazyConnect: true       — do not connect on construction
 *   - enableOfflineQueue: false — do not queue commands when Redis is down;
 *                                 queued commands executing on reconnect would
 *                                 produce false "all clear" on stale data
 *
 * §4 F-02 REQUIREMENT: Command timeout of 100ms enforced client-side.
 *   ioredis `commandTimeout` option enforces this per-command.
 *
 * NOTE: The rate limiter does NOT own the Redis connection (§5 API Contract).
 * The operator passes a pre-constructed redisClient in options. This helper
 * is provided for operators who want a correctly-configured client. It is NOT
 * called internally by the middleware — the middleware uses whatever client
 * is passed in options.redisClient.
 *
 * If the operator provides their own ioredis client, they are responsible for
 * ensuring it is configured with enableOfflineQueue: false and an appropriate
 * commandTimeout. The middleware cannot enforce this on an externally-provided
 * client.
 */

/**
 * Construct a correctly-configured ioredis client for use with this rate limiter.
 *
 * @param {Object} redisOptions - ioredis connection options (host, port, etc.)
 * @returns {import('ioredis').Redis}
 */
function createRedisClient(redisOptions = {}) {
  // ioredis is a peer dependency. The operator must install it.
  // We require it here rather than at module load to produce a clear error
  // message if it is missing.
  let Redis;
  try {
    Redis = require('ioredis');
  } catch (e) {
    throw new Error(
      '[ratelimit] ioredis is required but not installed. ' +
      'Run: npm install ioredis'
    );
  }

  return new Redis({
    ...redisOptions,

    // F-01: Do not connect on construction. Connect on first command.
    lazyConnect: true,

    // F-01: Disable offline queue. Commands issued while Redis is unavailable
    // must fail immediately (throw) rather than queue. This is required so
    // that the middleware can detect the failure and apply fail-open/closed
    // behavior. If the queue were enabled, commands would silently wait and
    // then execute in a burst when Redis reconnects, producing incorrect counts.
    enableOfflineQueue: false,

    // F-02: Enforce 100ms command timeout client-side.
    // This is the maximum time the middleware will wait for a Redis response
    // before treating it as a timeout failure (F-02 behavior).
    commandTimeout: 100,
  });
}

module.exports = { createRedisClient };
```

**Compliance note:** R-25 (`lazyConnect: true`, `enableOfflineQueue: false`) and R-26 (100ms command timeout) satisfied. The design states the rate limiter does not own the Redis connection (§5), so this helper is advisory. The gap between "operator may provide a misconfigured client" and "design requires these settings" is flagged in the Open Items section.

### `src/middleware.js` — Core Middleware Factory

*Satisfies: R-01–R-06, R-22–R-24, R-27–R-31, R-33–R-52, R-57–R-70*

```javascript
'use strict';

/**
 * rateLimiter(options) — Express/Node.js middleware factory.
 *
 * This is the primary implementation file. It wires together:
 *   - Options validation (§5)
 *   - Per-request rate limit evaluation (§2, §3)
 *   - All failure modes (§4)
 *   - Response contract (§5)
 *   - Metrics emission (§6.1)
 *   - Structured logging (§6.1)
 *
 * Read §5 of the design document before modifying this file.
 */

const { computeWindowValues, buildKey, executeRateLimitScript } = require('./algorithm');
const metrics = require('./metrics');
const logger = require('./logger');

// ---------------------------------------------------------------------------
// Log rate limiter for F-01/F-02 WARN logs (§4 F-01)
// "Log a WARN-level event per request (rate-limited to one log line per second
//  to prevent log flooding)"
// ---------------------------------------------------------------------------

/**
 * Creates a rate-limited logger that emits at most one WARN per `intervalMs`.
 * Used exclusively for Redis-unavailable and timeout warnings to prevent log
 * flooding during an outage (§4 F-01).
 *
 * @param {number} intervalMs
 * @returns {(fields: Object) => void}
 */
function createThrottledWarn(intervalMs) {
  let lastEmitTime = 0;
  return function throttledWarn(fields) {
    const now = Date.now();
    if (now - lastEmitTime >= intervalMs) {
      lastEmitTime = now;
      logger.warn(fields);
    }
  };
}

// ---------------------------------------------------------------------------
// Options validation (§5)
// ---------------------------------------------------------------------------

/**
 * Validate the options object passed to rateLimiter().
 * Throws synchronously on invalid configuration so the error surfaces at
 * middleware initialization time, not at request time (§4 F-06 principle).
 *
 * @param {Object} options
 */
function validateOptions(options) {
  if (options === null || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  // namespace: string, no spaces, max 64 chars (§5)
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must be 64 characters or fewer');
  }

  // limit: positive integer (§5)
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: positive integer (§5)
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: function (§5)
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: object (ioredis instance) (§5)
  if (options.redisClient === null || typeof options.redisClient !== 'object') {
    throw new Error('[ratelimit] options.redisClient must be an ioredis Redis instance');
  }

  // onLimitExceeded: optional function (§5)
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // failOpen: boolean, required, no default (§5)
  // DECISION POINT D-01: The design requires this to be explicitly set by the
  // operator. There is no default. This is intentional — the design states
  // "Forces the operator to make a conscious choice." We enforce this with a
  // hard error rather than silently choosing a default.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'There is no default. See §7 D-01 of the design document.'
    );
  }
}

// ---------------------------------------------------------------------------
// Response helpers (§5)
// ---------------------------------------------------------------------------

/**
 * Compute the Retry-After value in seconds.
 * §5: ceil((current_window + 1) * window_size - now), minimum 1.
 *
 * @param {number} windowResetTimestamp - Unix seconds when window resets
 * @param {number} nowSeconds - Current time in seconds (floating point)
 * @returns {number} Integer seconds, minimum 1
 */
function computeRetry
