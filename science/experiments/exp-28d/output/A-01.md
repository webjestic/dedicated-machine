# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 — Overview / Assumptions
- [ ] **R-01** Identifier is passed in / extracted; no auth logic in limiter
- [ ] **R-02** Redis connection is consumed, not owned (caller provides client)
- [ ] **R-03** `X-Forwarded-For` / trusted header extraction is the caller's responsibility via `identifierFn`; design flags this as D-03

### §2 — Algorithm
- [ ] **R-04** Approximated sliding window counter algorithm
- [ ] **R-05** `current_window = floor(now / window_size)`
- [ ] **R-06** `previous_window = current_window - 1`
- [ ] **R-07** `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-08** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-09** Reject if `weighted_count >= limit`
- [ ] **R-10** Increment `current_count` atomically only if allowed
- [ ] **R-11** Lua script executed via `EVALSHA` with `EVAL` fallback (F-07)
- [ ] **R-12** Lua script logic must not be altered
- [ ] **R-13** `EXPIRE` set to `window_size * 2` only on first write (`new_count == 1`)

### §3 — Storage
- [ ] **R-14** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-15** `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-16** TTL = `window_size * 2` seconds, set only on first write
- [ ] **R-17** Redis config requirements documented (not enforced in code — infrastructure concern)

### §4 — Failure Modes
- [ ] **R-18** F-01: Redis connection unavailable → fail-open or fail-closed per `failOpen` flag
- [ ] **R-19** F-01: Log WARN at most once per second during outage (rate-limited logging)
- [ ] **R-20** F-01: Emit `ratelimit.redis.unavailable` metric
- [ ] **R-21** F-01: `enableOfflineQueue: false` on ioredis client
- [ ] **R-22** F-02: Redis command timeout enforced client-side at 100ms
- [ ] **R-23** F-02: Emit `ratelimit.redis.timeout` metric (distinct from unavailable)
- [ ] **R-24** F-03: Log ERROR with raw Redis response on unexpected data
- [ ] **R-25** F-03: Apply fail-open/fail-closed; do not crash; do not surface internals to client
- [ ] **R-26** F-06: If `identifierFn` returns null/undefined → log ERROR, fail-open for that request, do not use fallback identifier
- [ ] **R-27** F-07: Catch `NOSCRIPT` error, fall back to `EVAL`, re-cache SHA
- [ ] **R-28** F-07: Emit `ratelimit.redis.script_miss` metric

### §5 — API Contract
- [ ] **R-29** `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] **R-30** Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] **R-31** Allowed request: call `next()`, set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-32** `X-RateLimit-Remaining` = `floor(limit - weighted_count)`, minimum 0
- [ ] **R-33** `X-RateLimit-Reset` = `(current_window + 1) * window_size`
- [ ] **R-34** Rejected request: HTTP 429, `Retry-After`, all `X-RateLimit-*` headers, `Content-Type: application/json`
- [ ] **R-35** `Retry-After` = `ceil((current_window + 1) * window_size - now)`, minimum 1
- [ ] **R-36** Response body: `error`, `message`, `retryAfter`, `limit`, `window` fields
- [ ] **R-37** `retryAfter` in body matches `Retry-After` header exactly
- [ ] **R-38** Response body must not include identifier, internal keys, Redis state, stack traces
- [ ] **R-39** Fail-closed Redis error: HTTP 503 with `Retry-After: 10` and specified body
- [ ] **R-40** `onLimitExceeded` override called instead of default 429 handler when provided

### §6.1 — Observability / Metrics
- [ ] **R-41** Emit `ratelimit.request.total` counter with `namespace`, `result` tags
- [ ] **R-42** Emit `ratelimit.request.allowed` counter with `namespace` tag
- [ ] **R-43** Emit `ratelimit.request.rejected` counter with `namespace` tag
- [ ] **R-44** Emit `ratelimit.redis.latency_ms` histogram with `namespace` tag
- [ ] **R-45** Emit `ratelimit.redis.error` counter with `namespace`, `error_type` tags
- [ ] **R-46** Emit `ratelimit.redis.script_miss` counter with `namespace` tag
- [ ] **R-47** Emit `ratelimit.failopen.request` counter with `namespace` tag
- [ ] **R-48** Emit `ratelimit.identifier.missing` counter with `namespace` tag
- [ ] **R-49** Metrics backend is D-05 (decision point)

### §6.1 — Observability / Logging
- [ ] **R-50** All log events are structured JSON
- [ ] **R-51** Required fields: `timestamp`, `level`, `component`, `namespace`, `identifier`, `result`, `weighted_count`, `limit`, `window_id`, `redis_latency_ms`, `request_id`
- [ ] **R-52** Identifier must be logged (PII flag to operator)
- [ ] **R-53** Fail-open log events at WARN level

### §6.2 — Graceful Degradation
- [ ] **R-54** Middleware must never crash the Node.js process
- [ ] **R-55** All Redis errors caught within middleware
- [ ] **R-56** No synchronous computation scaling with request volume
- [ ] **R-57** Recovery from Redis outage automatic (no cached "down" state)

### §6.4 — Health Check
- [ ] **R-58** Export `isAlive()` → synchronous, returns `true` if initialized
- [ ] **R-59** Export async `isReady()` → Redis PING with 200ms timeout
- [ ] **R-60** `isReady()` fail-open: 200 with `redis: "degraded"` body
- [ ] **R-61** `isReady()` fail-closed: 503 with `redis: "unavailable"` body

### `src/luaScript.js` — Lua Script and SHA Management

*Satisfies: R-11, R-12, R-13, R-27, R-28*

```javascript
'use strict';

const crypto = require('crypto');

/**
 * Normative Lua script from §2 of the design document.
 *
 * IMPLEMENTATION CONSTRAINT: This script must not be altered without re-running
 * the load test suite defined in §6.5. The logic is specified verbatim in the
 * design document and any change constitutes a deviation from the design.
 *
 * Return value layout (0-indexed):
 *   [0] — 1 = allowed, 0 = rejected
 *   [1] — current window count after increment (or before, if rejected)
 *   [2] — previous window count
 *   [3] — weighted count used for the allow/reject decision
 */
const RATE_LIMIT_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

/**
 * SHA1 of the script, computed once at module load time.
 * Used for EVALSHA calls. If Redis returns NOSCRIPT, the full script body
 * is re-submitted via EVAL and this SHA is re-confirmed (F-07).
 */
const SCRIPT_SHA = crypto
  .createHash('sha1')
  .update(RATE_LIMIT_SCRIPT)
  .digest('hex');

/**
 * Execute the rate limit Lua script against Redis.
 *
 * Implements F-07: catches NOSCRIPT errors, falls back to EVAL with the full
 * script body, and re-caches the SHA. The SHA is module-level state; after
 * a successful EVAL fallback, subsequent calls will use EVALSHA again.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string[]} keys   - [currentKey, previousKey]
 * @param {(string|number)[]} args - [limit, windowSize, now, elapsedFrac]
 * @param {{ emit: Function }} metrics - metrics emitter (for script_miss counter)
 * @param {string} namespace - for metric tagging
 * @returns {Promise<[number, number, number, number]>} - [allowed, currentCount, prevCount, weighted]
 */
async function evalScript(redisClient, keys, args, metrics, namespace) {
  try {
    const result = await redisClient.evalsha(SCRIPT_SHA, keys.length, ...keys, ...args);
    return result;
  } catch (err) {
    if (err.message && err.message.startsWith('NOSCRIPT')) {
      // F-07: Script was evicted from Redis script cache.
      // Fall back to EVAL with the full script body. Redis will re-cache it.
      // Emit script_miss metric so the operator is alerted (§6.5, LT-07; §6.1 metrics).
      metrics.emit('ratelimit.redis.script_miss', 1, { namespace });

      const result = await redisClient.eval(
        RATE_LIMIT_SCRIPT,
        keys.length,
        ...keys,
        ...args
      );
      // Script is now re-cached in Redis under the same SHA.
      // Future calls will use EVALSHA again without further action.
      return result;
    }
    // Not a NOSCRIPT error — re-throw for the caller to classify.
    throw err;
  }
}

module.exports = { RATE_LIMIT_SCRIPT, SCRIPT_SHA, evalScript };
```

**Compliance note:** R-11 (EVALSHA with EVAL fallback), R-12 (script not altered), R-13 (EXPIRE only on `new_count == 1`), R-27 (NOSCRIPT catch + EVAL fallback), R-28 (script_miss metric emitted). ✓

### `src/logger.js` — Structured JSON Logger

*Satisfies: R-50, R-51, R-52, R-53*

```javascript
'use strict';

/**
 * Structured JSON logger for the rate limiter component.
 *
 * All log events are structured JSON as required by §6.1.
 * Required fields per §6.1:
 *   timestamp, level, component, namespace, identifier, result,
 *   weighted_count, limit, window_id, redis_latency_ms, request_id
 *
 * OUTPUT: process.stdout (structured logs are expected to be collected by
 * the container/process supervisor and forwarded to the logging pipeline).
 *
 * PII FLAG (§6.1, R-52): The `identifier` field is logged as-is. If identifiers
 * contain PII (email addresses, phone numbers, etc. used as API keys), the
 * logging pipeline MUST hash or mask this field before storage. This is an
 * infrastructure concern per the design document, but the implementing engineer
 * must flag it to the operator before production deployment.
 */

/**
 * Rate-limited warn logger for Redis unavailability (F-01).
 * The design specifies: "Log a WARN-level event per request (rate-limited to
 * one log line per second to prevent log flooding)."
 *
 * This state is module-level. It is intentionally not per-namespace or
 * per-instance because the log flooding concern is about total log volume,
 * not per-namespace volume.
 */
let _lastWarnLoggedAt = 0;
const WARN_RATE_LIMIT_MS = 1000; // one log line per second per design spec

/**
 * Emit a structured log event.
 *
 * @param {string} level - 'INFO' | 'WARN' | 'ERROR'
 * @param {object} fields - must include all required fields from §6.1
 */
function log(level, fields) {
  const entry = {
    timestamp: new Date().toISOString(),
    level,
    component: 'ratelimit',
    ...fields,
  };
  process.stdout.write(JSON.stringify(entry) + '\n');
}

/**
 * Emit a WARN log for Redis unavailability, rate-limited to once per second.
 * Satisfies F-01 requirement: "rate-limited to one log line per second to
 * prevent log flooding."
 *
 * @param {object} fields
 */
function warnRateLimited(fields) {
  const now = Date.now();
  if (now - _lastWarnLoggedAt >= WARN_RATE_LIMIT_MS) {
    _lastWarnLoggedAt = now;
    log('WARN', fields);
  }
}

module.exports = { log, warnRateLimited };
```

**Compliance note:** R-50 (structured JSON), R-51 (required fields assembled at call sites), R-52 (PII flag documented), R-53 (WARN level for fail-open, enforced at call sites). ✓

### `src/rateLimiter.js` — Main Middleware Factory

*Satisfies: R-01 through R-57 (core implementation)*

```javascript
'use strict';

const { evalScript } = require('./luaScript');
const { defaultMetrics } = require('./metrics');
const { log, warnRateLimited } = require('./logger');
const { ErrorType } = require('./errors');

/**
 * Validate options at initialization time.
 * The design requires that configuration errors surface at startup, not at
 * request time (§4, F-06 — "must be a hard configuration error surfaced at
 * startup or middleware initialization").
 *
 * @param {object} options
 * @throws {Error} if any required option is missing or invalid
 */
function validateOptions(options) {
  if (!options || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  if (typeof options.namespace !== 'string' || options.namespace.trim() === '') {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must not exceed 64 characters');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }

  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  if (!options.redisClient || typeof options.redisClient.evalsha !== 'function') {
    throw new Error('[ratelimit] options.redisClient must be an ioredis client instance');
  }

  if (options.onLimitExceeded !== undefined && typeof options.onLimitExceeded !== 'function') {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // DECISION POINT D-01 (fail-open vs. fail-closed):
  // The design requires `failOpen` to be explicitly declared — no default.
  // This forces the operator to make a conscious choice (§5, options table).
  // We enforce this by throwing if the field is absent or not a boolean.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true (fail-open) or false ' +
      '(fail-closed). There is no default. See §7, Decision Point D-01 of the design document.'
    );
  }
}

/**
 * Build the Redis key for a given identifier and window ID.
 * Key format from §3: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Classify a Redis error into one of the three error types defined in §4.
 *
 * @param {Error} err
 * @returns {string} - one of ErrorType.UNAVAILABLE | ErrorType.TIMEOUT | ErrorType.UNEXPECTED
 */
function classifyRedisError(err) {
  if (!err || !err.message) return ErrorType.UNEXPECTED;

  const msg = err.message;

  // ioredis connection-level errors
  if (
    msg.includes('ECONNREFUSED') ||
    msg.includes('ECONNRESET') ||
    msg.includes('ETIMEDOUT') ||
    msg.includes('Connection is closed') ||
    msg.includes('Stream isn\'t writeable') ||
    msg.includes('connect ETIMEDOUT')
  ) {
    return ErrorType.UNAVAILABLE;
  }

  // ioredis command timeout (set via commandTimeout option)
  if (msg.includes('Command timed out') || msg.includes('Timeout')) {
    return ErrorType.TIMEOUT;
  }

  return ErrorType.UNEXPECTED;
}

/**
 * Parse and validate the Lua script return value.
 * Expected: array of 4 numbers [allowed, currentCount, prevCount, weighted]
 *
 * Returns null if the response cannot be parsed as the expected structure (F-03).
 *
 * @param {*} raw - raw return value from Redis EVAL/EVALSHA
 * @returns {{ allowed: boolean, currentCount: number, prevCount: number, weighted: number } | null}
 */
function parseScriptResult(raw) {
  if (
    !Array.isArray(raw) ||
    raw.length !== 4 ||
    raw.some(v => typeof v !== 'number' && typeof v !== 'string')
  ) {
    return null;
  }

  const [allowedFlag, currentCount, prevCount, weighted] = raw.map(Number);

  if ([allowedFlag, currentCount, prevCount, weighted].some(isNaN)) {
    return null;
  }

  return {
    allowed: allowedFlag === 1,
    currentCount,
    prevCount,
    weighted,
  };
}

/**
 * rateLimiter — factory function that returns an Express-compatible middleware.
 *
 * @param {object} options
 * @param {string}   options.namespace          - §5: logical namespace, max 64 chars, no spaces
 * @param {number}   options.limit              - §5: max requests per window, positive integer
 * @param {number}   options.windowSizeSeconds  - §5: window duration in seconds, positive integer
 * @param {Function} options.identifierFn       - §5: (req) => string | null, pure and synchronous
 * @param {object}   options.redisClient        - §5: pre-constructed ioredis client
 * @param {Function} [options.onLimitExceeded]  - §5: optional override for 429 response handler
 * @param {boolean}  options.failOpen           - §5: explicit fail-open (true) or fail-closed (false)
 * @param {object}   [options.metrics]          - §6.1: metrics emitter; defaults to no-op (D-05)
 * @returns {Function} Express middleware (req, res, next)
 */
function rateLimiter(options) {
  // Validate all options at initialization time (F-06, §5).
  // Any error here is thrown synchronously and must be caught by the caller
  // during application startup — not during request handling.
  validateOptions(options);

  const {
    namespace,
    limit,
    windowSizeSeconds,
    identifierFn,
    redisClient,
    onLimitExceeded,
    failOpen,
  } = options;

  // D-05: Use provided metrics emitter or fall back to no-op.
  const metrics = options.metrics || defaultMetrics;

  /**
   * Handle Redis failure according to the fail-open/fail-closed configuration.
   * Satisfies: F-01, F-02, F-03, R-18, R-19, R-20, R-23, R-24, R-25, R-39.
   *
   * @param {object} params
   * @param {Error}  params.err          - the Redis error
   * @param {string} params.errorType    - classified error type
   * @param {string} params.identifier   - for logging
   * @param {string} params.requestId    - for logging
   * @param {object} params.req
   * @param {object} params.res
   * @param {Function} params.next
   */
  function handleRedisFailure({ err, errorType, identifier, requestId, req, res, next }) {
    // Emit the appropriate error metric (R-45).
    // script_miss is handled separately in luaScript.js (R-46).
    const metricErrorType = errorType === ErrorType.SCRIPT_MISS
      ? 'script_miss'  // should not reach here; handled in evalScript
      : errorType;

    metrics.emit('ratelimit.redis.error', 1, { namespace, error_type: metricErrorType });
    metrics.emit('ratelimit.request.total', 1, { namespace, result: 'error' });

    if (failOpen) {
      // F-01 / F-02 fail-open path:
      // Allow the request through. Log at WARN (rate-limited to 1/sec). Emit failopen metric.
      // R-19: rate-limited warn log
      // R-53: WARN level for fail-open
      warnRateLimited({
        level: 'WARN',
        namespace,
        identifier,
        result: 'failopen',
        weighted_count: null,
        limit,
        window_id: null,
        redis_latency_ms: null,
        request_id: requestId,
        error_type: errorType,
        error_message: err ? err.message : 'unknown',
      });

      // R-47: emit failopen counter
      metrics.emit('ratelimit.failopen.request', 1, { namespace });

      // R-57: Do not cache a "Redis is down" state. next() is called unconditionally;
      // the next request will attempt Redis again via the normal path.
      next();
    } else {
      // F-01 / F-02 fail-closed path:
      // Return 503 with Retry-After: 10 as specified in §5.
      // R-39: 503 response body and headers
      log('WARN', {
        namespace,
        identifier,
        result: 'error',
        weighted_count: null,
        limit,
        window_id: null,
        redis_latency_ms: null,
        request_id: requestId,
        error_type: errorType,
        error_message: err ? err.message : 'unknown',
      });

      res.status(503)
        .set('Retry-After', '10')
        .set('Content-Type', 'application/json')
        .json({
          error: 'service_unavailable',
          message: 'Rate limiting is temporarily unavailable. Please retry shortly.',
          retryAfter: 10,
        });
    }
  }

  /**
   * The middleware function returned to the caller.
   * This is the hot path — every request passes through here.
   */
  return async function rateLimiterMiddleware(req, res, next) {
    // ── Step 1: Extract identifier ──────────────────────────────────────────
    // F-06: If identifierFn returns null/undefined, log ERROR and fail-open
    // for this specific request. Do NOT use a fallback identifier.
    let identifier;
    try {
      identifier = identifierFn(req);
    } catch (extractErr) {
      // identifierFn threw — treat as null return.
      identifier = null;
      log('ERROR', {
        namespace,
        identifier: null,
        result: 'error',
        weighted_count: null,
        limit,
        window_id: null,
        redis_latency_ms: null,
        request_id: req.id || req.headers['x-request-id'] || null,
        error_message: `identifierFn threw: ${extractErr.message}`,
      });
    }

    if (identifier === null || identifier === undefined) {
      // R-26: log ERROR, emit metric, fail-open for this request.
      // Do not apply rate limiting. Do not use a fallback identifier.
      metrics.emit('ratelimit.identifier.missing', 1, { namespace });
      log('ERROR', {
