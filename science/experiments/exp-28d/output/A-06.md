# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** Middleware library interface (not standalone service — see Open Items)
- [ ] **R-02** Per-identifier rate limiting
- [ ] **R-03** Sliding window algorithm
- [ ] **R-04** Redis-backed storage
- [ ] **R-05** Structured 429 response on limit exceeded
- [ ] **R-06** Identifier extracted via `identifierFn` (not `req.socket.remoteAddress`)

### §2 Algorithm
- [ ] **R-07** Approximated sliding window counter formula implemented exactly as specified
- [ ] **R-08** `current_window = floor(now / window_size)`
- [ ] **R-09** `previous_window = current_window - 1`
- [ ] **R-10** `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-11** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-12** Reject if `weighted_count >= limit`; allow and increment if `weighted_count < limit`
- [ ] **R-13** Lua script executed atomically via `EVALSHA` with `EVAL` fallback (F-07)
- [ ] **R-14** Lua script body matches normative specification exactly — no logic alterations
- [ ] **R-15** Return value indices: [0]=allowed/rejected, [1]=new current count, [2]=previous count, [3]=weighted

### §3 Storage
- [ ] **R-16** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-17** `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-18** TTL = `window_size * 2` seconds, set only on first write (`new_count === 1`)
- [ ] **R-19** TTL set inside Lua script (atomicity)

### §4 Failure Modes
- [ ] **R-20** F-01: Redis connection unavailable — fail-open or fail-closed per `failOpen` option
- [ ] **R-21** F-01: WARN log rate-limited to one line per second during Redis unavailability
- [ ] **R-22** F-01: Emit `ratelimit.redis.unavailable` metric
- [ ] **R-23** F-02: Redis command timeout enforced client-side at 100ms
- [ ] **R-24** F-02: Emit `ratelimit.redis.timeout` metric (distinct from unavailable)
- [ ] **R-25** F-03: Log ERROR with raw Redis response on unexpected data; fail-open/closed per config; no process crash; no internal details to client
- [ ] **R-26** F-07: Catch `NOSCRIPT` error, fall back to `EVAL`, re-cache SHA
- [ ] **R-27** F-07: Emit `ratelimit.redis.script_miss` metric
- [ ] **R-28** F-06: If `identifierFn` returns null/undefined at request time — log ERROR, fail-open for that request, do NOT use fallback identifier
- [ ] **R-29** F-06: Emit `ratelimit.identifier.missing` metric
- [ ] **R-30** Redis client: `ioredis` with `lazyConnect: true`, `enableOfflineQueue: false`
- [ ] **R-31** Redis client: connection timeout ≤ 200ms
- [ ] **R-32** No unhandled promise rejections from Redis operations
- [ ] **R-33** No event loop blocking

### §5 API Contract
- [ ] **R-34** `rateLimiter(options)` returns Express-compatible middleware `(req, res, next)`
- [ ] **R-35** Options: `namespace` (string, no spaces, max 64 chars)
- [ ] **R-36** Options: `limit` (positive integer)
- [ ] **R-37** Options: `windowSizeSeconds` (positive integer)
- [ ] **R-38** Options: `identifierFn` — pure, synchronous `(req) => string | null`
- [ ] **R-39** Options: `redisClient` — pre-constructed ioredis instance
- [ ] **R-40** Options: `onLimitExceeded` — optional override handler
- [ ] **R-41** Options: `failOpen` — required boolean, no default
- [ ] **R-42** Allowed request: call `next()`
- [ ] **R-43** Allowed request: set `X-RateLimit-Limit` header
- [ ] **R-44** Allowed request: set `X-RateLimit-Remaining` header (floor at 0)
- [ ] **R-45** Allowed request: set `X-RateLimit-Reset` header (`(current_window + 1) * window_size`)
- [ ] **R-46** Allowed request: set `X-RateLimit-Window` header
- [ ] **R-47** Rejected request: HTTP 429
- [ ] **R-48** Rejected request: `Retry-After` header = `ceil((current_window + 1) * window_size - now)`, minimum 1
- [ ] **R-49** Rejected request: `X-RateLimit-Limit`, `X-RateLimit-Remaining: 0`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-50** Rejected request: `Content-Type: application/json`
- [ ] **R-51** Rejected request: JSON body with `error`, `message`, `retryAfter`, `limit`, `window` fields
- [ ] **R-52** `retryAfter` in body matches `Retry-After` header exactly
- [ ] **R-53** Body must not include identifier, internal keys, Redis state, stack traces
- [ ] **R-54** `Retry-After` minimum value is 1 (never 0)
- [ ] **R-55** Fail-closed Redis error: HTTP 503 with `Retry-After: 10`, JSON body with `error`, `message`, `retryAfter`

### §6.1 Observability — Metrics
- [ ] **R-56** `ratelimit.request.total` counter with `namespace`, `result` tags
- [ ] **R-57** `ratelimit.request.allowed` counter with `namespace` tag
- [ ] **R-58** `ratelimit.request.rejected` counter with `namespace` tag
- [ ] **R-59** `ratelimit.redis.latency_ms` histogram with `namespace` tag, buckets: 1,5,10,25,50,100,250,500
- [ ] **R-60** `ratelimit.redis.error` counter with `namespace`, `error_type` tags
- [ ] **R-61** `ratelimit.redis.script_miss` counter with `namespace` tag
- [ ] **R-62** `ratelimit.failopen.request` counter with `namespace` tag
- [ ] **R-63** `ratelimit.identifier.missing` counter with `namespace` tag

### §6.1 Observability — Structured Logging
- [ ] **R-64** All log events are structured JSON
- [ ] **R-65** Required fields: `timestamp`, `level`, `component`, `namespace`, `identifier`, `result`, `weighted_count`, `limit`, `window_id`, `redis_latency_ms`, `request_id`

### §6.4 Health Check
- [ ] **R-66** Export `isAlive()` — synchronous, returns true if initialized
- [ ] **R-67** Export async `isReady()` — Redis PING with 200ms timeout
- [ ] **R-68** `isReady()`: PING success → `{status:"ready", redis:"ok"}`
- [ ] **R-69** `isReady()`: PING fail + failOpen true → `{status:"ready", redis:"degraded", mode:"fail-open"}`
- [ ] **R-70** `isReady()`: PING fail + failOpen false → `{status:"not_ready", redis:"unavailable"}`

### `src/luaScript.js`

*Satisfies: R-13, R-14, R-15, R-26, R-27*

```javascript
'use strict';

// §2 — Normative Lua script. DO NOT ALTER THE LOGIC OF THIS SCRIPT
// without re-running the load test suite defined in §6.5.
// The script body here must remain byte-for-byte equivalent to the
// normative specification in the design document.
const RATE_LIMIT_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

// In-process cache of the SHA1 returned by Redis after first LOAD or EVAL.
// This is process-local. On Redis restart the SHA is still valid (scripts are
// deterministic), but the cache may be evicted — F-07 handles that case.
let cachedSha = null;

/**
 * Execute the rate limit Lua script.
 *
 * Uses EVALSHA when a SHA is cached; catches NOSCRIPT and falls back to EVAL,
 * re-caching the SHA on success. This satisfies F-07.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string[]} keys   - [currentKey, previousKey]
 * @param {(string|number)[]} args - [limit, windowSize, now, elapsedFrac]
 * @param {{ emitScriptMiss: () => void }} metrics
 * @returns {Promise<[number, number, number, number]>}
 */
async function evalScript(redisClient, keys, args, metrics) {
  // Attempt EVALSHA if we have a cached SHA (F-07: performance path)
  if (cachedSha) {
    try {
      const result = await redisClient.evalsha(cachedSha, keys.length, ...keys, ...args);
      return result;
    } catch (err) {
      if (!err.message || !err.message.startsWith('NOSCRIPT')) {
        // Not a script-miss error — re-throw for the caller to handle
        throw err;
      }
      // NOSCRIPT: script was evicted. Fall through to EVAL below.
      // R-27: emit script_miss metric
      metrics.emitScriptMiss();
      cachedSha = null;
    }
  }

  // F-07 fallback: EVAL with full script body. Redis returns the SHA on EVAL,
  // but ioredis EVAL does not return the SHA directly — we load it separately.
  const result = await redisClient.eval(RATE_LIMIT_SCRIPT, keys.length, ...keys, ...args);

  // Re-cache the SHA for subsequent requests.
  // SCRIPT LOAD is fire-and-forget here; a failure does not affect correctness
  // (we will just use EVAL again next time, which is safe).
  redisClient.script('load', RATE_LIMIT_SCRIPT)
    .then((sha) => { cachedSha = sha; })
    .catch(() => { /* non-fatal; next request will EVAL again */ });

  return result;
}

/**
 * Pre-load the script into Redis at initialization time.
 * This is a best-effort optimization; failure is non-fatal.
 *
 * @param {import('ioredis').Redis} redisClient
 */
async function preloadScript(redisClient) {
  try {
    cachedSha = await redisClient.script('load', RATE_LIMIT_SCRIPT);
  } catch (_err) {
    // Non-fatal. evalScript will fall back to EVAL on first request.
    cachedSha = null;
  }
}

module.exports = { evalScript, preloadScript, RATE_LIMIT_SCRIPT };
```

**Compliance note:** R-13 (EVALSHA + EVAL fallback), R-14 (script body unaltered), R-15 (return value contract), R-26 (NOSCRIPT catch + EVAL fallback + SHA re-cache), R-27 (script_miss metric call).

### `src/validateOptions.js`

*Satisfies: R-34 through R-41*

```javascript
'use strict';

/**
 * Validates the options object passed to rateLimiter().
 * Throws a descriptive Error on any violation.
 * All fields are validated at initialization time, not at request time (§4, F-06).
 *
 * @param {object} options
 */
function validateOptions(options) {
  if (options === null || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be a non-null object');
  }

  // R-35: namespace — string, no spaces, max 64 chars
  if (typeof options.namespace !== 'string') {
    throw new Error('[ratelimit] options.namespace must be a string');
  }
  if (options.namespace.trim().length === 0) {
    throw new Error('[ratelimit] options.namespace must not be empty');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must not exceed 64 characters');
  }

  // R-36: limit — positive integer
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // R-37: windowSizeSeconds — positive integer
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // R-38: identifierFn — function
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // R-39: redisClient — must be provided (duck-type check for ioredis interface)
  if (
    options.redisClient === null ||
    options.redisClient === undefined ||
    typeof options.redisClient.eval !== 'function' ||
    typeof options.redisClient.evalsha !== 'function' ||
    typeof options.redisClient.ping !== 'function'
  ) {
    throw new Error(
      '[ratelimit] options.redisClient must be a pre-constructed ioredis client instance'
    );
  }

  // R-40: onLimitExceeded — optional, but if present must be a function
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // R-41: failOpen — required boolean, no default.
  // The design explicitly states "No default. Forces the operator to make a
  // conscious choice." We enforce this by requiring the field to be present
  // and strictly boolean.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'There is no default. This is a required operational decision (see design §7, D-01).'
    );
  }
}

module.exports = { validateOptions };
```

**Compliance note:** R-35 through R-41 validated at initialization time.

### `src/metrics.js`

*Satisfies: R-56 through R-63*

```javascript
'use strict';

// DECISION POINT D-05: Metrics backend
// ─────────────────────────────────────
// The design specifies metric names and semantics as normative (§6.1) but
// leaves the backend (StatsD, Prometheus, etc.) as an explicit decision point.
//
// SAFER DEFAULT CHOSEN: No-op metrics implementation.
// The metrics interface is fully defined here. The operator must supply a
// concrete implementation by passing a `metricsEmitter` object to the
// rateLimiter factory, or by replacing this module with a backend-specific
// implementation.
//
// To wire in a real backend:
//   1. Implement the interface below using your metrics client.
//   2. Pass it as `options.metricsEmitter` (see rateLimiter.js).
//   3. Remove the no-op stubs.
//
// The histogram buckets are normative per §6.1:
//   [1, 5, 10, 25, 50, 100, 250, 500] milliseconds.
//
// See Open Items: OI-02.

/**
 * Creates a metrics emitter bound to a namespace.
 * If a real emitter is provided, it delegates to it.
 * Otherwise, all operations are no-ops.
 *
 * @param {string} namespace
 * @param {object|null} backendEmitter - Optional real metrics backend.
 *   Must implement: increment(name, tags), histogram(name, value, tags).
 * @returns {object}
 */
function createMetrics(namespace, backendEmitter) {
  const tags = { namespace };

  function increment(name, extraTags = {}) {
    if (backendEmitter && typeof backendEmitter.increment === 'function') {
      backendEmitter.increment(name, { ...tags, ...extraTags });
    }
    // No-op if no backend is configured (D-05 decision point)
  }

  function histogram(name, value, extraTags = {}) {
    if (backendEmitter && typeof backendEmitter.histogram === 'function') {
      backendEmitter.histogram(name, value, { ...tags, ...extraTags });
    }
    // No-op if no backend is configured (D-05 decision point)
  }

  return {
    // R-56: ratelimit.request.total
    emitRequestTotal(result) {
      increment('ratelimit.request.total', { result });
    },

    // R-57: ratelimit.request.allowed
    emitRequestAllowed() {
      increment('ratelimit.request.allowed');
    },

    // R-58: ratelimit.request.rejected
    emitRequestRejected() {
      increment('ratelimit.request.rejected');
    },

    // R-59: ratelimit.redis.latency_ms (histogram, buckets: 1,5,10,25,50,100,250,500)
    emitRedisLatency(ms) {
      histogram('ratelimit.redis.latency_ms', ms);
    },

    // R-60: ratelimit.redis.error
    emitRedisError(errorType) {
      // errorType must be one of: 'timeout' | 'unavailable' | 'unexpected'
      increment('ratelimit.redis.error', { error_type: errorType });
    },

    // R-61: ratelimit.redis.script_miss
    emitScriptMiss() {
      increment('ratelimit.redis.script_miss');
    },

    // R-62: ratelimit.failopen.request
    emitFailOpen() {
      increment('ratelimit.failopen.request');
    },

    // R-63: ratelimit.identifier.missing
    emitIdentifierMissing() {
      increment('ratelimit.identifier.missing');
    },
  };
}

module.exports = { createMetrics };
```

**Compliance note:** R-56 through R-63 all implemented. D-05 decision point surfaced with no-op default and clear replacement instructions.

### `src/logger.js`

*Satisfies: R-64, R-65, R-21 (WARN rate-limiting)*

```javascript
'use strict';

// §6.1 — All log events must be structured JSON.
// Required fields per design: timestamp, level, component, namespace,
// identifier, result, weighted_count, limit, window_id, redis_latency_ms,
// request_id.
//
// NOTE TO OPERATOR (§6.1, Log volume management):
// INFO-level logging of every allowed request will be high-volume in
// production. Confirm with the operator whether allowed-request logs are
// required. The design states rejections and errors must always be logged.
// This implementation logs all events at INFO/WARN/ERROR as specified.
// To suppress allowed-request INFO logs, set the LOG_ALLOWED_REQUESTS
// environment variable to 'false'. This is an operational tuning point
// not specified in the design — see Open Items: OI-03.
//
// NOTE TO OPERATOR (§6.1, PII):
// The identifier field is logged as-is. If identifiers contain PII
// (e.g., email addresses used as API keys), the logging pipeline must
// hash or mask this field before storage. This is flagged per the design's
// explicit requirement that the implementing engineer surface this concern.

/**
 * Emits a structured JSON log event to stdout.
 * All rate limiter log events go through this function.
 *
 * @param {string} level - 'INFO' | 'WARN' | 'ERROR'
 * @param {object} fields - Must include all required fields from §6.1
 */
function log(level, fields) {
  const event = {
    timestamp: new Date().toISOString(),
    level,
    component: 'ratelimit',
    ...fields,
  };
  // Write to stdout as a single JSON line. The logging infrastructure is
  // responsible for collection and routing.
  process.stdout.write(JSON.stringify(event) + '\n');
}

// R-21: WARN log during Redis unavailability must be rate-limited to one
// line per second to prevent log flooding.
// This is a per-process rate limit on the warn log, not per-identifier.
let lastRedisWarnTimestamp = 0;

/**
 * Emit a WARN log for Redis unavailability, rate-limited to once per second.
 *
 * @param {object} fields
 */
function warnRedisUnavailable(fields) {
  const now = Date.now();
  if (now - lastRedisWarnTimestamp >= 1000) {
    lastRedisWarnTimestamp = now;
    log('WARN', { ...fields, result: 'error' });
  }
  // Suppressed if within the 1-second window — this is the rate-limiting
  // behavior specified in §4, F-01.
}

module.exports = { log, warnRedisUnavailable };
```

**Compliance note:** R-64 (structured JSON), R-65 (required fields emitted by callers via `fields` object), R-21 (WARN rate-limited to 1/second).

### `src/rateLimiter.js`

*Satisfies: R-01 through R-06, R-07 through R-19, R-20 through R-33, R-34 through R-55, R-56 through R-65*

```javascript
'use strict';

const { validateOptions } = require('./validateOptions');
const { evalScript, preloadScript } = require('./luaScript');
const { createMetrics } = require('./metrics');
const { log, warnRedisUnavailable } = require('./logger');

// ─────────────────────────────────────────────────────────────────────────────
// §2 — Key construction
// ─────────────────────────────────────────────────────────────────────────────

/**
 * Builds the Redis key for a given window.
 * Format: ratelimit:{namespace}:{identifier}:{window_id}
 * R-16
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Computes the current window ID.
 * R-17: Math.floor(Date.now() / 1000 / windowSizeSeconds)
 *
 * @param {number} nowSeconds - Current Unix time in seconds
 * @param {number} windowSizeSeconds
 * @returns {number}
 */
function computeWindowId(nowSeconds, windowSizeSeconds) {
  return Math.floor(nowSeconds / windowSizeSeconds);
}

// ─────────────────────────────────────────────────────────────────────────────
// §5 — Response helpers
// ─────────────────────────────────────────────────────────────────────────────

/**
 * Sets the rate limit informational headers on the response.
 * Used for both allowed (R-43 through R-46) and rejected (R-49) requests.
 *
 * @param {object} res - Node.js/Express response object
 * @param {object} headerValues
 */
function setRateLimitHeaders(res, { limit, remaining, reset, windowSizeSeconds }) {
  res.setHeader('X-RateLimit-Limit', String(limit));
  // R-44: floor remaining at 0
  res.setHeader('X-RateLimit-Remaining', String(Math.max(0, remaining)));
  res.setHeader('X-RateLimit-Reset', String(reset));
  res.setHeader('X-RateLimit-Window', String(windowSizeSeconds));
}

/**
 * Sends the 429 Too Many Requests response.
 * R-47 through R-54.
 *
 * @param {object} res
 * @param {object} params
 */
function sendRejectedResponse(res, { limit, windowSizeSeconds, currentWindow, nowSeconds }) {
  // R-48: Retry-After = ceil((current_window + 1) * window_size - now)
  // R-54: minimum value is 1 (never 0)
  const resetTimestamp = (currentWindow + 1) * windowSizeSeconds;
  const retryAfterRaw = Math.ceil(resetTimestamp - nowSeconds);
  const retryAfter = Math.max(1, retryAfterRaw); // R-54: floor at 1

  setRateLimitHeaders(res, {
    limit,
    remaining: 0, // R-49: always 0 on rejection
    reset: resetTimestamp,
    windowSizeSeconds,
  });

  res.setHeader('Retry-After', String(retryAfter));
  res.setHeader('Content-Type', 'application/json'); // R-50

  res.statusCode = 429; // R-47

  // R-51, R-52, R-53: body fields; no identifier, no internal keys, no stack traces
  const body = {
    error: 'rate_limit_exceeded',
    message: 'Too many requests. Please retry after the indicated delay.',
    retryAfter, // R-52: matches Retry-After header exactly
    limit,
    window: windowSizeSeconds,
  };

  res.end(JSON.stringify(body));
}

/**
 * Sends the 503 Service Unavailable response (fail-closed Redis error path).
 * R-55.
 *
 * @param {object} res
 */
function sendServiceUnavailableResponse(res) {
  res.setHeader('Retry-After', '10');
  res.setHeader('Content-Type', 'application/json');
  res.statusCode = 503;

  const body = {
    error: 'service_unavailable',
    message: 'Rate limiting is temporarily unavailable. Please retry shortly.',
    retryAfter: 10,
  };

  res.end(JSON.stringify(body));
}

// ─────────────────────────────────────────────────────────────────────────────
// §5 — Middleware factory
// ─────────────────────────────────────────────────────────────────────────────

/**
 * Creates a rate limiter middleware.
 *
 * DECISION POINT D-03 (identifier / IP extraction):
 * ──────────────────────────────────────────────────
 * The design states (§1, Assumption 2) that the real client IP must be
 * extracted from X-Forwarded-For or a trusted header — NOT from
 * req.socket.remoteAddress. The design explicitly defers the extraction
 * logic to the caller via identifierFn.
 *
 * SAFER DEFAULT: This middleware does not implement any IP extraction.
 * The operator MUST supply an identifierFn that correctly extracts the
 * identifier from the appropriate header for their proxy configuration.
 * This extraction logic must be audited before deployment (§1, Assumption 2).
 *
 * Example identifierFn for a single trusted proxy:
 *   identifierFn: (req) => {
 *     const xff = req.headers['x-forwarded-for'];
 *     return xff ? xff.split(',')[0].trim() : req.socket.remoteAddress;
 *   }
 *
 * @param {object} options
 * @param {string} options.namespace
 * @param {number} options.limit
 * @param {number} options.windowSizeSeconds
 * @param {function} options.identifierFn
 * @param {import('ioredis').Redis} options.redisClient
 * @param {function} [options.onLimitExceeded]
 * @param {boolean} options.failOpen
 * @param {object} [options.metricsEmitter] - D-05: optional metrics backend
 * @returns {function} Express-compatible middleware (req, res, next)
 */
function rateLimiter(options) {
  // R-34 through R-41: validate all options at initialization time
  validateOptions(options);

  const {
    namespace,
    limit,
    windowSizeSeconds,
    identifierFn,
    redisClient,
    onLimitExceeded,
    failOpen,
    metricsEmitter = null, // D-05: no-op if not provided
  } = options;

  // R-56 through R-63: metrics bound to this namespace
  const metrics = createMetrics(namespace, metrics
