# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] **R-01** Redis client must use `ioredis` with `lazyConnect: true` and `enableOfflineQueue: false`
- [ ] **R-02** Identifier extraction logic must be implemented (D-03 decision point — flagged)
- [ ] **R-03** System does not authenticate requests, geolocate, or manage Redis infrastructure

### §2 Algorithm
- [ ] **R-04** Approximated sliding window counter algorithm implemented as specified
- [ ] **R-05** `current_window = floor(now / window_size)`, `elapsed_fraction = (now % window_size) / window_size`
- [ ] **R-06** `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] **R-07** Reject if `weighted_count >= limit`; increment and allow if `weighted_count < limit`
- [ ] **R-08** Lua script executed atomically via `EVALSHA` with `EVAL` fallback (F-07)
- [ ] **R-09** Lua script logic must match the normative script exactly — no alterations

### §3 Storage
- [ ] **R-10** Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] **R-11** `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] **R-12** TTL set to `window_size * 2` on first write only (`if new_count == 1`)
- [ ] **R-13** Redis config requirements documented (maxmemory-policy, version ≥ 3.2) — operational, not code

### §4 Failure Modes
- [ ] **R-14** F-01: Redis connection unavailable → fail-open or fail-closed per D-01; log WARN (rate-limited to 1/sec); emit `ratelimit.redis.unavailable`
- [ ] **R-15** F-02: Redis command timeout → same as F-01; emit `ratelimit.redis.timeout`; 100ms client-side timeout enforced
- [ ] **R-16** F-03: Redis returns unexpected data → log ERROR with raw response; apply D-01 behavior; do not crash; do not surface internals to client
- [ ] **R-17** F-04: Clock skew — no active defense; emit `window_id` in logs for detection
- [ ] **R-18** F-05: Memory exhaustion — handled by Redis eviction policy (operational); evicted key treated as zero count
- [ ] **R-19** F-06: Identifier extraction returns null → log ERROR; fail-open for that request; do not use fallback identifier
- [ ] **R-20** F-07: NOSCRIPT error → catch and fall back to `EVAL`; re-cache SHA; emit `ratelimit.redis.script_miss`
- [ ] **R-21** F-08: Process restart — no special handling required; state in Redis

### §5 API Contract
- [ ] **R-22** `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] **R-23** Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] **R-24** Allowed request: call `next()`; set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] **R-25** `X-RateLimit-Remaining` = `limit - weighted_count`, floored at 0
- [ ] **R-26** `X-RateLimit-Reset` = `(current_window + 1) * window_size` (Unix seconds)
- [ ] **R-27** Rejected request: HTTP 429; `Retry-After`, `X-RateLimit-*` headers; `Content-Type: application/json`
- [ ] **R-28** `Retry-After` = `ceil((current_window + 1) * window_size - now)`; minimum value 1
- [ ] **R-29** 429 body: `{ error, message, retryAfter, limit, window }`; `retryAfter` matches header; no identifier/internals/stack traces
- [ ] **R-30** Redis unavailable + fail-closed: HTTP 503; `Retry-After: 10`; body `{ error, message, retryAfter }`
- [ ] **R-31** `onLimitExceeded` override invoked instead of default 429 handler when provided

### §6.1 Observability — Metrics
- [ ] **R-32** Emit `ratelimit.request.total` (counter, tags: namespace, result)
- [ ] **R-33** Emit `ratelimit.request.allowed` (counter, tag: namespace)
- [ ] **R-34** Emit `ratelimit.request.rejected` (counter, tag: namespace)
- [ ] **R-35** Emit `ratelimit.redis.latency_ms` (histogram, tag: namespace)
- [ ] **R-36** Emit `ratelimit.redis.error` (counter, tags: namespace, error_type)
- [ ] **R-37** Emit `ratelimit.redis.script_miss` (counter, tag: namespace)
- [ ] **R-38** Emit `ratelimit.failopen.request` (counter, tag: namespace)
- [ ] **R-39** Emit `ratelimit.identifier.missing` (counter, tag: namespace)
- [ ] **R-40** Metrics backend is D-05 decision point — implement stub interface

### §6.1 Observability — Logging
- [ ] **R-41** All log events structured JSON with required fields: timestamp, level, component, namespace, identifier, result, weighted_count, limit, window_id, redis_latency_ms, request_id
- [ ] **R-42** Identifier must be logged (PII flag to operator documented)
- [ ] **R-43** Fail-open log at WARN; Redis unavailable log rate-limited to 1/sec
- [ ] **R-44** Rejections and errors always logged; allowed-request log volume is D-06 decision point

### §6.2 Graceful Degradation
- [ ] **R-45** Rate limiter must never crash the Node.js process
- [ ] **R-46** Rate limiter must never block the event loop
- [ ] **R-47** Fail-open must be explicit and logged; silent fail-open is not acceptable
- [ ] **R-48** Recovery from Redis outage must be automatic (no cached "down" state)

### §6.4 Health Check
- [ ] **R-49** Export `isAlive()` → always returns true if process is running and middleware initialized
- [ ] **R-50** Export async `isReady()` → Redis PING with 200ms timeout; returns status per fail-open/fail-closed config
- [ ] **R-51** Health check must not run on the hot path

### `src/luaScript.js` — Normative Lua Script and Execution

Satisfies: **R-08, R-09, R-20**

```javascript
'use strict';

/**
 * NORMATIVE LUA SCRIPT — DO NOT ALTER LOGIC.
 *
 * Design ref: §2 "Lua script (normative)".
 * The design states: "The implementing engineer must not alter the logic of
 * this script without re-running the load test suite defined in §6.5."
 *
 * This script is reproduced exactly as specified. Any modification requires
 * a full load test cycle (LT-01 through LT-07).
 */
const RATE_LIMIT_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

/**
 * In-memory cache of the script SHA returned by Redis after first EVAL.
 * Used to switch from EVAL to EVALSHA on subsequent calls (F-07).
 * This is process-local state. On process restart it resets to null,
 * which is correct — the first call will re-EVAL and re-cache.
 *
 * Design ref: §4 F-07.
 */
let cachedScriptSha = null;

/**
 * Execute the rate limit Lua script against Redis.
 *
 * Implements F-07: attempts EVALSHA first; on NOSCRIPT error falls back to
 * EVAL, caches the returned SHA, and emits ratelimit.redis.script_miss.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string[]} keys   - [currentKey, previousKey]
 * @param {(string|number)[]} args - [limit, windowSize, now, elapsedFrac]
 * @param {object} metrics  - metrics interface (see src/metrics.js)
 * @param {string} namespace
 * @returns {Promise<[number, number, number, number]>} - [allowed, currentCount, previousCount, weighted]
 */
async function executeRateLimitScript(redisClient, keys, args, metrics, namespace) {
  /**
   * Attempt EVALSHA if we have a cached SHA.
   * On NOSCRIPT, fall back to EVAL and re-cache.
   * Design ref: §4 F-07.
   */
  if (cachedScriptSha !== null) {
    try {
      const result = await redisClient.evalsha(cachedScriptSha, keys.length, ...keys, ...args);
      return parseScriptResult(result);
    } catch (err) {
      if (isNoscriptError(err)) {
        // Script was evicted from Redis cache. Fall through to EVAL below.
        // Emit script_miss metric as required by design §6.1.
        metrics.increment('ratelimit.redis.script_miss', { namespace });
        cachedScriptSha = null;
        // Fall through to EVAL.
      } else {
        throw err; // Re-throw non-NOSCRIPT errors for caller to handle.
      }
    }
  }

  // EVAL path: used on first call or after NOSCRIPT recovery.
  // ioredis.eval signature: eval(script, numkeys, ...keys, ...args)
  const result = await redisClient.eval(RATE_LIMIT_SCRIPT, keys.length, ...keys, ...args);

  // Cache the SHA for future calls via EVALSHA.
  // We derive the SHA by loading the script. ioredis does not return the SHA
  // from EVAL directly, so we use SCRIPT LOAD to obtain and cache it.
  //
  // DESIGN GAP NOTE: The design specifies "Re-cache the script SHA after
  // successful EVAL" (§4 F-07) but does not specify the mechanism.
  // ioredis's eval() does not return the SHA. We use SCRIPT LOAD here.
  // This adds one extra Redis round-trip on NOSCRIPT recovery only — not
  // on the hot path. This is the safest correct implementation.
  try {
    cachedScriptSha = await redisClient.script('LOAD', RATE_LIMIT_SCRIPT);
  } catch (loadErr) {
    // If SCRIPT LOAD fails, we simply won't cache the SHA this time.
    // The next call will EVAL again. This is degraded but not incorrect.
    // Do not throw — the rate limit operation itself succeeded.
    cachedScriptSha = null;
  }

  return parseScriptResult(result);
}

/**
 * Parse and validate the array returned by the Lua script.
 *
 * Expected: [allowed(0|1), currentCount, previousCount, weighted]
 * Design ref: §2 "Return value" specification; §4 F-03.
 *
 * Throws a typed error if the result cannot be parsed, allowing the caller
 * to apply F-03 behavior.
 */
function parseScriptResult(result) {
  if (
    !Array.isArray(result) ||
    result.length !== 4 ||
    (result[0] !== 0 && result[0] !== 1)
  ) {
    const err = new Error('Unexpected Lua script result structure');
    err.code = 'RATELIMIT_UNEXPECTED_RESULT';
    err.rawResult = result;
    throw err;
  }
  // result[3] (weighted) is returned as a Redis bulk string when it is a
  // float. Parse all values to numbers.
  return [
    Number(result[0]),
    Number(result[1]),
    Number(result[2]),
    Number(result[3]),
  ];
}

/**
 * Detect NOSCRIPT errors from ioredis.
 * ioredis surfaces these as Error objects with message containing "NOSCRIPT".
 */
function isNoscriptError(err) {
  return err && typeof err.message === 'string' && err.message.includes('NOSCRIPT');
}

module.exports = { executeRateLimitScript, RATE_LIMIT_SCRIPT };
```

**Compliance note:** R-08 (EVALSHA + EVAL fallback), R-09 (script logic unaltered), R-20 (NOSCRIPT recovery + script_miss metric) satisfied.

### `src/logger.js` — Structured JSON Logger

Satisfies: **R-41, R-42, R-43, R-44**

```javascript
'use strict';

/**
 * Structured JSON logger.
 *
 * All log events emitted by the rate limiter must be structured JSON with
 * the required fields specified in §6.1.
 *
 * Required fields on every event (§6.1):
 *   timestamp, level, component, namespace, identifier, result,
 *   weighted_count, limit, window_id, redis_latency_ms, request_id
 *
 * DECISION POINT D-06: Allowed-request log volume
 *
 * The design states: "The implementing engineer should confirm with the
 * operator whether allowed-request logs are required or whether only
 * rejections and errors should be logged at INFO."
 *
 * DEFAULT CHOSEN: Log only rejections, errors, and fail-open events at INFO
 * or above. Allowed requests are logged at DEBUG level. This is the safer
 * default for production log volume. To enable allowed-request INFO logging,
 * set options.logAllowedRequests = true when constructing the Logger.
 *
 * OPERATOR NOTE (§6.1, R-42): The `identifier` field is logged on every
 * event. If identifiers contain PII (e.g., email addresses used as API keys),
 * the logging pipeline MUST hash or mask this field before storage. This is
 * an infrastructure concern per the design, but the implementing engineer
 * must flag it to the operator before production deployment.
 */
class Logger {
  /**
   * @param {object} options
   * @param {boolean} [options.logAllowedRequests=false] - D-06 decision point
   * @param {function} [options.write] - injectable write function for testing; defaults to process.stdout.write
   */
  constructor(options = {}) {
    // D-06: Default is false (do not log allowed requests at INFO).
    this._logAllowedRequests = options.logAllowedRequests === true;
    this._write = options.write || ((line) => process.stdout.write(line + '\n'));
  }

  /**
   * Emit a structured log event.
   *
   * @param {string} level - 'INFO' | 'WARN' | 'ERROR' | 'DEBUG'
   * @param {object} fields - required fields per §6.1
   */
  _emit(level, fields) {
    const entry = {
      timestamp: new Date().toISOString(), // ISO8601 per §6.1
      level,
      component: 'ratelimit',             // static per §6.1
      ...fields,
    };
    try {
      this._write(JSON.stringify(entry));
    } catch (_) {
      // If JSON serialization fails (e.g., circular reference in fields),
      // do not crash the process. Design req: R-45.
    }
  }

  /**
   * Log an allowed request.
   * Logged at DEBUG by default (D-06). Set logAllowedRequests=true for INFO.
   */
  logAllowed(fields) {
    // D-06: Only emit at INFO if operator has opted in.
    const level = this._logAllowedRequests ? 'INFO' : 'DEBUG';
    this._emit(level, { result: 'allowed', ...fields });
  }

  /**
   * Log a rejected request (429). Always logged at INFO per design.
   * Design ref: §6.1 "Rejections and errors must always be logged."
   */
  logRejected(fields) {
    this._emit('INFO', { result: 'rejected', ...fields });
  }

  /**
   * Log a fail-open event. Always logged at WARN per design.
   * Design ref: §4 F-01, §6.2 "Fail-open must be explicit and logged."
   */
  logFailOpen(fields) {
    this._emit('WARN', { result: 'failopen', ...fields });
  }

  /**
   * Log an error event. Always logged at ERROR per design.
   * Design ref: §4 F-03, F-06.
   */
  logError(fields) {
    this._emit('ERROR', { result: 'error', ...fields });
  }

  /**
   * Log a warning event (e.g., Redis unavailable, rate-limited to 1/sec).
   * Design ref: §4 F-01.
   */
  logWarn(fields) {
    this._emit('WARN', fields);
  }
}

module.exports = { Logger };
```

**Compliance note:** R-41 (structured JSON, required fields), R-42 (identifier logged, PII flag documented), R-43 (fail-open WARN, unavailable rate-limited — rate limiting implemented in `rateLimiter.js`), R-44 (D-06 decision point) satisfied.

### `src/rateLimiter.js` — Main Middleware Factory

Satisfies: **R-01 through R-03, R-04 through R-07, R-10 through R-12, R-14 through R-19, R-21 through R-48**

```javascript
'use strict';

const { executeRateLimitScript } = require('./luaScript');
const { MetricsClient }          = require('./metrics');
const { Logger }                 = require('./logger');
const { RedisErrorType }         = require('./errors');

/**
 * DECISION POINT D-01: Fail-Open vs. Fail-Closed
 *
 * The design makes this an EXPLICIT REQUIRED OPTION — `failOpen` must be
 * supplied by the caller with no default. The design states: "No default.
 * Forces the operator to make a conscious choice." (§5 Options table)
 *
 * This is enforced in validateOptions() below: if `failOpen` is not a
 * boolean, initialization throws immediately.
 *
 * The safer default if validation were bypassed would be fail-open (allow
 * traffic), but the design prohibits any default here. The validation
 * enforces this.
 */

/**
 * DECISION POINT D-03: Identifier Extraction / X-Forwarded-For
 *
 * The design states: "The real client IP must be extracted from
 * X-Forwarded-For or a trusted header — not req.socket.remoteAddress.
 * This extraction logic must be implemented and audited before deployment."
 * (§1 Assumption 2, §7 D-03)
 *
 * The design places this logic in `identifierFn`, which is caller-supplied.
 * The rate limiter itself does not implement IP extraction — it is
 * identifier-agnostic per §1 "What This System Does Not Do".
 *
 * ACTION REQUIRED BEFORE DEPLOYMENT: The caller must supply an `identifierFn`
 * that correctly extracts the real client IP from the trusted proxy header.
 * A naive implementation that reads req.headers['x-forwarded-for'] without
 * validating the proxy chain is a security vulnerability (IP spoofing).
 * See §7 D-03 for the full decision specification.
 */

/**
 * Rate at which "Redis unavailable" WARN logs are emitted.
 * Design ref: §4 F-01: "rate-limited to one log line per second to prevent
 * log flooding."
 */
const REDIS_UNAVAILABLE_LOG_INTERVAL_MS = 1000;

/**
 * Validate the options object at initialization time.
 * Design ref: §5 Options table — all fields required unless marked optional.
 * Design ref: §4 F-06 — identifier extraction failure must be a hard config
 * error surfaced at startup, not at request time (for structural issues).
 *
 * @param {object} options
 * @throws {Error} if any required option is missing or invalid
 */
function validateOptions(options) {
  if (!options || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must be 64 characters or fewer');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }

  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  if (!options.redisClient || typeof options.redisClient.eval !== 'function') {
    throw new Error('[ratelimit] options.redisClient must be an ioredis client instance');
  }

  // onLimitExceeded is optional per §5.
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // failOpen is REQUIRED with NO DEFAULT. Design ref: §5, D-01.
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'This is a required operational decision (see design §7 D-01). ' +
      'true = allow all requests when Redis is unavailable. ' +
      'false = return 503 to all requests when Redis is unavailable.'
    );
  }
}

/**
 * Build the Redis key for a given namespace, identifier, and window ID.
 * Design ref: §3 Key Strategy.
 *
 * Key format: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * Note: The identifier is used as-is. Normalization (e.g., lowercasing IPs,
 * stripping ports) is the responsibility of the caller's identifierFn.
 * Design ref: §8 Note N-02 (referenced in §3).
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Compute window IDs and timing values from the current time.
 * Design ref: §2 Algorithm specification; §3 Note on window_id arithmetic.
 *
 * @param {number} nowMs - current time in milliseconds (Date.now())
 * @param {number} windowSizeSeconds
 * @returns {{ nowSeconds, currentWindow, previousWindow, elapsedFraction, resetTimestamp }}
 */
function computeWindowTiming(nowMs, windowSizeSeconds) {
  // Design ref: §3 Note: "compute window_id as Math.floor(Date.now() / 1000 / window_size_in_seconds)"
  const nowSeconds      = nowMs / 1000;
  const currentWindow   = Math.floor(nowSeconds / windowSizeSeconds);
  const previousWindow  = currentWindow - 1;
  // Design ref: §2 "elapsed_fraction = (now % window_size) / window_size"
  // where now is in seconds.
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;
  // Design ref: §5 "X-RateLimit-Reset = (current_window + 1) * window_size"
  const resetTimestamp  = (currentWindow + 1) * windowSizeSeconds;

  return { nowSeconds, currentWindow, previousWindow, elapsedFraction, resetTimestamp };
}

/**
 * Compute the Retry-After value in seconds.
 * Design ref: §5 "Retry-After = ceil((current_window + 1) * window_size - now)"
 * Design ref: §6.3 "A Retry-After value of 0 is a defect; minimum value is 1."
 *
 * @param {number} resetTimestamp - Unix seconds when window resets
 * @param {number} nowSeconds
 * @returns {number} integer seconds, minimum 1
 */
function computeRetryAfter(resetTimestamp, nowSeconds) {
  const retryAfter = Math.ceil(resetTimestamp - nowSeconds);
  // Design ref: §6.3 minimum value is 1.
