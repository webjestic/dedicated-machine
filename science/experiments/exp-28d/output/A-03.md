# Rate Limiter Implementation

## 1. Requirement Checklist

Derived from the design document before any code is written.

### §1 Overview / Assumptions
- [ ] R-01: Middleware library interface (not standalone service — see Open Items)
- [ ] R-02: Per-identifier rate limiting
- [ ] R-03: Redis-backed (shared across all Node.js instances)
- [ ] R-04: `X-Forwarded-For` / trusted header extraction (D-03 decision point)
- [ ] R-05: `ioredis` client with `lazyConnect: true`, `enableOfflineQueue: false`

### §2 Algorithm
- [ ] R-06: Approximated sliding window counter algorithm
- [ ] R-07: `current_window = floor(now / window_size)`
- [ ] R-08: `elapsed_fraction = (now % window_size) / window_size`
- [ ] R-09: `weighted_count = previous_count * (1 - elapsed_fraction) + current_count`
- [ ] R-10: Reject if `weighted_count >= limit`
- [ ] R-11: Lua script executed atomically via `EVAL`/`EVALSHA` — normative script must not be altered
- [ ] R-12: Lua return value structure: `[allowed, current_count, previous_count, weighted]`

### §3 Storage
- [ ] R-13: Key format: `ratelimit:{namespace}:{identifier}:{window_id}`
- [ ] R-14: `window_id = Math.floor(Date.now() / 1000 / windowSizeSeconds)`
- [ ] R-15: TTL = `window_size * 2`, set only on first write (`new_count === 1`)
- [ ] R-16: TTL set via Lua script (already in normative script)

### §4 Failure Modes
- [ ] R-17: F-01 — Redis connection unavailable: fail-open or fail-closed per D-01
- [ ] R-18: F-01 — WARN log rate-limited to 1/second during Redis unavailability
- [ ] R-19: F-01 — Emit `ratelimit.redis.unavailable` metric
- [ ] R-20: F-02 — Redis command timeout (100ms client-side), emit `ratelimit.redis.timeout`
- [ ] R-21: F-03 — Unexpected Redis response: ERROR log, fail-open/closed, no crash, no client exposure
- [ ] R-22: F-04 — Clock skew: no active defense; emit `window_id` in logs for detection
- [ ] R-23: F-05 — Memory exhaustion: handled by Redis `allkeys-lru` config (operational, not code)
- [ ] R-24: F-06 — Identifier extraction failure: ERROR log, fail-open for that request, no silent fallback
- [ ] R-25: F-07 — Lua script eviction: catch `NOSCRIPT`, fall back to `EVAL`, re-cache SHA
- [ ] R-26: F-08 — Process restart: no special handling required (state in Redis)

### §5 API Contract
- [ ] R-27: `rateLimiter(options)` returns Express middleware `(req, res, next)`
- [ ] R-28: Options: `namespace`, `limit`, `windowSizeSeconds`, `identifierFn`, `redisClient`, `onLimitExceeded` (optional), `failOpen` (required, no default)
- [ ] R-29: Namespace validation: no spaces, max 64 chars
- [ ] R-30: Allowed request: call `next()`, set `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`, `X-RateLimit-Window` headers
- [ ] R-31: `X-RateLimit-Remaining` floored at 0
- [ ] R-32: `X-RateLimit-Reset = (current_window + 1) * window_size`
- [ ] R-33: Rejected request: HTTP 429, `Retry-After`, all X-RateLimit headers, `Content-Type: application/json`, structured body
- [ ] R-34: `Retry-After = ceil((current_window + 1) * window_size - now)`, minimum 1
- [ ] R-35: 429 body: `error`, `message`, `retryAfter`, `limit`, `window` — no identifier, no internal keys, no stack traces
- [ ] R-36: `retryAfter` in body matches `Retry-After` header exactly
- [ ] R-37: Fail-closed Redis unavailable: HTTP 503, `Retry-After: 10`, structured body
- [ ] R-38: `onLimitExceeded` override hook

### §6 Operational Requirements
- [ ] R-39: Metrics: `ratelimit.request.total` (counter, tags: namespace, result)
- [ ] R-40: Metrics: `ratelimit.request.allowed` (counter, tag: namespace)
- [ ] R-41: Metrics: `ratelimit.request.rejected` (counter, tag: namespace)
- [ ] R-42: Metrics: `ratelimit.redis.latency_ms` (histogram, tag: namespace, buckets: 1,5,10,25,50,100,250,500)
- [ ] R-43: Metrics: `ratelimit.redis.error` (counter, tags: namespace, error_type)
- [ ] R-44: Metrics: `ratelimit.redis.script_miss` (counter, tag: namespace)
- [ ] R-45: Metrics: `ratelimit.failopen.request` (counter, tag: namespace)
- [ ] R-46: Metrics: `ratelimit.identifier.missing` (counter, tag: namespace)
- [ ] R-47: Structured JSON logging on every rate limiter event (fields specified in §6.1)
- [ ] R-48: Liveness check: `isAlive()` function
- [ ] R-49: Readiness check: `isReady()` async function with Redis PING, 200ms timeout, fail-open/closed aware response
- [ ] R-50: Rate limiter must never crash the Node.js process
- [ ] R-51: Rate limiter must never block the event loop
- [ ] R-52: Recovery from Redis outage must be automatic (no cached "down" state)

### `src/lua-script.js`

Implements: R-11, R-12, R-25 (F-07 NOSCRIPT recovery)

```javascript
'use strict';

// §2 — Normative Lua script. DO NOT ALTER without re-running load test suite (§6.5).
// Any change to this script invalidates the load test results and requires a full re-run
// of LT-01 through LT-07 before deployment.
const LUA_SCRIPT = `
local current_key   = KEYS[1]
local previous_key  = KEYS[2]
local limit         = tonumber(ARGV[1])
local window_size   = tonumber(ARGV[2])
local now           = tonumber(ARGV[3])
local elapsed_frac  = tonumber(ARGV[4])

local current_count  = tonumber(redis.call('GET', current_key) or 0)
local previous_count = tonumber(redis.call('GET', previous_key) or 0)
local weighted       = previous_count * (1 - elapsed_frac) + current_count

if weighted >= limit then
  return {0, current_count, previous_count, weighted}
end

local new_count = redis.call('INCR', current_key)
if new_count == 1 then
  redis.call('EXPIRE', current_key, window_size * 2)
end

return {1, new_count, previous_count, weighted}
`;

// SHA1 of the script above, computed at load time and cached.
// Invalidated and recomputed on NOSCRIPT error (F-07).
let cachedSha = null;

/**
 * Execute the normative rate limit Lua script.
 *
 * Implements F-07: attempts EVALSHA first; on NOSCRIPT error, falls back to EVAL
 * with the full script body, re-caches the SHA, and emits ratelimit.redis.script_miss.
 *
 * @param {import('ioredis').Redis} redisClient
 * @param {string} currentKey
 * @param {string} previousKey
 * @param {number} limit
 * @param {number} windowSizeSeconds
 * @param {number} nowSeconds - Unix timestamp in seconds (float acceptable)
 * @param {number} elapsedFraction
 * @param {Function} onScriptMiss - called with no args when NOSCRIPT is caught (for metrics)
 * @returns {Promise<[number, number, number, number]>} [allowed, currentCount, previousCount, weighted]
 */
async function evalScript(
  redisClient,
  currentKey,
  previousKey,
  limit,
  windowSizeSeconds,
  nowSeconds,
  elapsedFraction,
  onScriptMiss
) {
  const args = [
    limit,
    windowSizeSeconds,
    nowSeconds,
    elapsedFraction,
  ];

  // Attempt EVALSHA if we have a cached SHA. This is the hot path.
  if (cachedSha !== null) {
    try {
      const result = await redisClient.evalsha(
        cachedSha,
        2,              // number of KEYS
        currentKey,
        previousKey,
        ...args
      );
      return parseScriptResult(result);
    } catch (err) {
      if (!isNoscriptError(err)) {
        // Not a script-miss error — propagate to caller for F-01/F-02/F-03 handling.
        throw err;
      }
      // NOSCRIPT: fall through to EVAL below.
      // R-44 / F-07: notify caller to emit ratelimit.redis.script_miss metric.
      if (typeof onScriptMiss === 'function') {
        onScriptMiss();
      }
      cachedSha = null; // Invalidate; will be re-set after successful EVAL.
    }
  }

  // EVAL with full script body. Also used on first call before SHA is cached.
  const result = await redisClient.eval(
    LUA_SCRIPT,
    2,              // number of KEYS
    currentKey,
    previousKey,
    ...args
  );

  // Re-cache the SHA so subsequent calls use EVALSHA.
  // We derive the SHA by loading the script; ioredis does not expose the SHA
  // from EVAL directly. We use SCRIPT LOAD to obtain and cache it.
  // This is a best-effort fire-and-forget; failure here is non-fatal.
  redisClient.script('LOAD', LUA_SCRIPT).then((sha) => {
    cachedSha = sha;
  }).catch(() => {
    // SHA re-caching failed. Next call will EVAL again. Non-fatal.
  });

  return parseScriptResult(result);
}

/**
 * Pre-load the Lua script into Redis and cache the SHA.
 * Call this at middleware initialization for best performance.
 * Failure is non-fatal — evalScript will fall back to EVAL.
 *
 * @param {import('ioredis').Redis} redisClient
 * @returns {Promise<void>}
 */
async function loadScript(redisClient) {
  try {
    cachedSha = await redisClient.script('LOAD', LUA_SCRIPT);
  } catch (_err) {
    // Non-fatal. evalScript will use EVAL on first call.
    cachedSha = null;
  }
}

/**
 * Parse and validate the Lua script return value.
 * §2 specifies: [allowed(0|1), current_count, previous_count, weighted]
 *
 * Throws if the result is not the expected 4-element array (F-03 trigger).
 *
 * @param {unknown} result
 * @returns {[number, number, number, number]}
 */
function parseScriptResult(result) {
  if (
    !Array.isArray(result) ||
    result.length !== 4 ||
    result.some((v) => typeof v !== 'number' && typeof v !== 'string')
  ) {
    // F-03: unexpected data — throw so caller can handle per fail-open/closed config.
    const err = new Error('Unexpected Lua script return value');
    err.code = 'RATELIMIT_UNEXPECTED_RESPONSE';
    err.rawResult = result;
    throw err;
  }
  return result.map(Number);
}

/**
 * @param {Error} err
 * @returns {boolean}
 */
function isNoscriptError(err) {
  // ioredis surfaces NOSCRIPT as an error with message starting with 'NOSCRIPT'
  return err && typeof err.message === 'string' && err.message.startsWith('NOSCRIPT');
}

module.exports = { evalScript, loadScript, LUA_SCRIPT };
```

**Compliance note:** R-11 (normative Lua script, unaltered), R-12 (return value structure), R-25 (F-07 NOSCRIPT recovery with EVAL fallback and SHA re-cache).

### `src/logger.js`

Implements: R-47 (structured JSON logging), R-18 (WARN log rate-limited to 1/second), R-22 (window_id in log fields)

```javascript
'use strict';

// §6.1 — Structured JSON logging. All fields specified in §6.1 are required on every event.
// DECISION POINT D-05 (partial): The logging destination is not specified by the design.
// This implementation writes to process.stdout as newline-delimited JSON.
// To route to a different sink (e.g., Winston, Pino), replace the `emit` function below.
// The field schema is normative and must not change.

/**
 * Emit a structured log event per §6.1 schema.
 *
 * @param {object} fields
 * @param {string} fields.level - 'INFO' | 'WARN' | 'ERROR'
 * @param {string} fields.namespace
 * @param {string|null} fields.identifier
 * @param {string} fields.result - 'allowed' | 'rejected' | 'error' | 'failopen'
 * @param {number|null} fields.weighted_count
 * @param {number} fields.limit
 * @param {number} fields.window_id
 * @param {number|null} fields.redis_latency_ms
 * @param {string|null} fields.request_id
 * @param {string} [fields.error_detail] - internal only, never surfaced to client
 */
function log(fields) {
  const entry = {
    timestamp: new Date().toISOString(),
    level: fields.level,
    component: 'ratelimit',
    namespace: fields.namespace,
    identifier: fields.identifier,
    result: fields.result,
    weighted_count: fields.weighted_count,
    limit: fields.limit,
    window_id: fields.window_id,
    redis_latency_ms: fields.redis_latency_ms,
    request_id: fields.request_id,
  };

  // Internal error detail is included in the log but must never be forwarded to clients.
  // §5 — "The body must not include... stack traces."
  if (fields.error_detail !== undefined) {
    entry.error_detail = fields.error_detail;
  }

  // §6.1 — "If the identifier contains PII (e.g., email addresses used as API keys),
  // the logging pipeline must hash or mask it before storage. This is an infrastructure
  // concern, not a rate limiter concern, but the implementing engineer must flag it
  // to the operator."
  // FLAG TO OPERATOR: If identifiers contain PII, configure your log pipeline to
  // hash or mask the `identifier` field before storage. This implementation logs
  // the raw identifier as required for incident investigation (§6.1).

  process.stdout.write(JSON.stringify(entry) + '\n');
}

// §4 F-01 — WARN log rate-limited to one line per second during Redis unavailability.
// This prevents log flooding when Redis is down and every request triggers a log event.
let lastRedisUnavailableLogMs = 0;
const REDIS_UNAVAILABLE_LOG_INTERVAL_MS = 1000;

/**
 * Log a Redis-unavailable WARN event, rate-limited to once per second.
 * §4 F-01 requirement: "Log a WARN-level event per request (rate-limited to one log
 * line per second to prevent log flooding)."
 *
 * Note: The design says "per request" but also "rate-limited to one log line per second."
 * These are contradictory. This implementation logs at most once per second (the
 * rate-limiting clause takes precedence as it is the safety constraint).
 * See Open Items: OI-01.
 *
 * @param {object} fields - same schema as log()
 */
function logRedisUnavailable(fields) {
  const now = Date.now();
  if (now - lastRedisUnavailableLogMs >= REDIS_UNAVAILABLE_LOG_INTERVAL_MS) {
    lastRedisUnavailableLogMs = now;
    log({ ...fields, level: 'WARN' });
  }
}

module.exports = { log, logRedisUnavailable };
```

**Compliance note:** R-47 (structured JSON, all required fields), R-18 (rate-limited WARN log), R-22 (window_id emitted in every log entry).

### `src/validation.js`

Implements: R-28, R-29 (options validation), R-05 (ioredis client requirements)

```javascript
'use strict';

/**
 * Validate the options object passed to rateLimiter().
 * §5 — All fields are required unless marked optional.
 * Throws synchronously on invalid configuration so errors surface at
 * initialization time, not at request time.
 *
 * @param {object} options
 * @throws {Error} on invalid configuration
 */
function validateOptions(options) {
  if (options === null || typeof options !== 'object') {
    throw new Error('[ratelimit] options must be an object');
  }

  // namespace: string, no spaces, max 64 chars (§5, R-29)
  if (typeof options.namespace !== 'string' || options.namespace.length === 0) {
    throw new Error('[ratelimit] options.namespace must be a non-empty string');
  }
  if (/\s/.test(options.namespace)) {
    throw new Error('[ratelimit] options.namespace must not contain spaces');
  }
  if (options.namespace.length > 64) {
    throw new Error('[ratelimit] options.namespace must not exceed 64 characters');
  }

  // limit: positive integer (§5)
  if (
    typeof options.limit !== 'number' ||
    !Number.isInteger(options.limit) ||
    options.limit <= 0
  ) {
    throw new Error('[ratelimit] options.limit must be a positive integer');
  }

  // windowSizeSeconds: positive integer (§5)
  if (
    typeof options.windowSizeSeconds !== 'number' ||
    !Number.isInteger(options.windowSizeSeconds) ||
    options.windowSizeSeconds <= 0
  ) {
    throw new Error('[ratelimit] options.windowSizeSeconds must be a positive integer');
  }

  // identifierFn: synchronous function (§5)
  if (typeof options.identifierFn !== 'function') {
    throw new Error('[ratelimit] options.identifierFn must be a function');
  }

  // redisClient: must be provided; we check for the ioredis interface surface
  // we rely on (§5, §4 F-01 requirement for lazyConnect/enableOfflineQueue).
  if (options.redisClient === null || typeof options.redisClient !== 'object') {
    throw new Error('[ratelimit] options.redisClient must be an ioredis Redis instance');
  }
  if (
    typeof options.redisClient.eval !== 'function' ||
    typeof options.redisClient.evalsha !== 'function' ||
    typeof options.redisClient.script !== 'function' ||
    typeof options.redisClient.ping !== 'function'
  ) {
    throw new Error(
      '[ratelimit] options.redisClient does not appear to be an ioredis Redis instance ' +
      '(missing eval, evalsha, script, or ping methods)'
    );
  }

  // §4 F-01 — ioredis must be configured with enableOfflineQueue: false.
  // We cannot inspect ioredis internal options directly in all versions, so we
  // emit a startup warning rather than throwing. The operator must verify this.
  // FLAG: Verify that the ioredis client was constructed with:
  //   { lazyConnect: true, enableOfflineQueue: false, commandTimeout: 100 }
  // Failure to set enableOfflineQueue: false will cause queued commands to
  // execute in a burst on Redis reconnect, producing incorrect counts.

  // onLimitExceeded: optional, must be function if provided (§5)
  if (
    options.onLimitExceeded !== undefined &&
    typeof options.onLimitExceeded !== 'function'
  ) {
    throw new Error('[ratelimit] options.onLimitExceeded must be a function if provided');
  }

  // failOpen: required boolean, no default (§5)
  // Design: "Required. Explicit declaration of fail-open (true) or fail-closed (false)
  // behavior. No default. Forces the operator to make a conscious choice."
  if (typeof options.failOpen !== 'boolean') {
    throw new Error(
      '[ratelimit] options.failOpen must be explicitly set to true or false. ' +
      'There is no default. See §7 D-01 of the design document.'
    );
  }
}

module.exports = { validateOptions };
```

**Compliance note:** R-28 (all options validated), R-29 (namespace constraints enforced), R-05 (ioredis interface surface checked; offline queue warning flagged).

### `src/middleware.js`

Implements: R-06 through R-16 (algorithm and storage), R-17 through R-26 (failure modes), R-27 through R-38 (API contract), R-39 through R-46 (metrics emission), R-47 (logging), R-50 through R-52 (operational requirements)

```javascript
'use strict';

const { evalScript, loadScript } = require('./lua-script');
const metrics = require('./metrics');
const { log, logRedisUnavailable } = require('./logger');
const { validateOptions } = require('./validation');

/**
 * Build the Redis key for a given identifier and window ID.
 * §3 Key Strategy: ratelimit:{namespace}:{identifier}:{window_id}
 *
 * @param {string} namespace
 * @param {string} identifier
 * @param {number} windowId
 * @returns {string}
 */
function buildKey(namespace, identifier, windowId) {
  return `ratelimit:${namespace}:${identifier}:${windowId}`;
}

/**
 * Compute window IDs and timing values for the current moment.
 * §2 Algorithm, §3 Storage Note.
 *
 * UNIT TEST REQUIREMENT (§3): Verify this arithmetic with fixed timestamps
 * before shipping. See compliance matrix for test reference.
 *
 * @param {number} windowSizeSeconds - positive integer
 * @returns {{
 *   nowSeconds: number,
 *   currentWindowId: number,
 *   previousWindowId: number,
 *   elapsedFraction: number,
 *   resetTimestamp: number
 * }}
 */
function computeWindowTiming(windowSizeSeconds) {
  // §3 Note: window_id = Math.floor(Date.now() / 1000 / window_size_in_seconds)
  // Date.now() returns milliseconds; divide by 1000 to get seconds.
  const nowMs = Date.now();
  const nowSeconds = nowMs / 1000; // float, used for elapsed_fraction precision

  // R-07: current_window = floor(now / window_size)
  const currentWindowId = Math.floor(nowSeconds / windowSizeSeconds);

  // §2: previous_window = current_window - 1
  const previousWindowId = currentWindowId - 1;

  // R-08: elapsed_fraction = (now % window_size) / window_size
  const elapsedFraction = (nowSeconds % windowSizeSeconds) / windowSizeSeconds;

  // R-32: reset_timestamp = (current_window + 1) * window_size
  const resetTimestamp = (currentWindowId + 1) * windowSizeSeconds;

  return { nowSeconds, currentWindowId, previousWindowId, elapsedFraction, resetTimestamp };
}

/**
 * Classify a Redis error for metric tagging and logging.
 * §4 F-01, F-02, F-03 — distinct error types with distinct metrics.
 *
 * @param {Error} err
 * @returns {'timeout'|'unavailable'|'unexpected'}
 */
function classifyRedisError(err) {
  if (!err) return 'unexpected';

  const msg = (err.message || '').toLowerCase();
  const code = err.code || '';

  // ioredis surfaces command timeouts with specific messages/codes.
  // 'Command timed out' is the ioredis commandTimeout message.
  if (
