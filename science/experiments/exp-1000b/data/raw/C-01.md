Two engineers. Same task. Same model. Same seniority label — ten years, detail-oriented, thorough.

One gets a competent code review. The other gets the race condition that only fires under specific timing, the infrastructure failure mode three layers below the visible code.

I ran twelve experiments to find out why.

The answer isn't what most practitioners expect. It's not how forcefully you encode the behavior. "Cannot help but catch every bug" sounds powerful. I know how that sentence ends. When you strip the domain vocabulary and keep the compulsion, output collapses to baseline. When you strip the compulsion and keep the domain vocabulary, the outputs hold. The compulsion framing was always arriving with dense domain vocabulary underneath it, taking credit for work it didn't do.

What actually shapes the consideration set — the range of failure modes the model will even reach for — is the technical substrate adjacent to the identity anchor.

Not "senior software engineer." That's orientation vocabulary. It labels a category without entering it. The model generates things consistent with senior-engineer-ness without running deep in any particular direction.

Mechanism vocabulary is different.

"Whether the write is fenced. Whether the lock is held for the duration of the operation. Whether the TTL arithmetic holds under GC pause."

Those tokens name the causal chain. They pre-weight the model's attention toward the failure-mode cluster before the first task token is processed. The consideration set is shaped before the task is read.

Orientation vocabulary produces 0/10 detection on target findings. Mechanism vocabulary produces 10/10.

The full article walks through the experimental anatomy: how fusion versus split configurations affect depth, why the detection rates move that far apart, and the precise grammatical principle that explains why entangling the behavioral drive with the domain vocabulary in a single sentence outperforms separating them across two.

The practical rule is simpler than the mechanism: write the technical substrate, not the credential.

Every orientation phrase has a mechanism equivalent. One tells the model where to look. The other tells it what the failure looks like when it gets there.
