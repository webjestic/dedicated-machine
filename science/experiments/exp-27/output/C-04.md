# Production Blockchain Implementation in Node.js

Let me start by identifying what a production deployment actually requires that typical "build me a blockchain" specs omit, then build accordingly.

## Production Gap Analysis

Before writing a line of code, here's what the stated requirement ("build me a blockchain in Node.js") does **not** mention but a production system requires:

| Gap | Risk if Ignored | My Decision |
|-----|----------------|-------------|
| Persistent storage | Node restart = chain lost | LevelDB via `level` |
| P2P networking | Single node = not a blockchain | libp2p-style TCP mesh |
| Mempool management | No transaction ordering/eviction | Priority queue with TTL |
| Fork resolution | Competing chains = split brain | Longest chain (most work) rule |
| Wallet / key management | No way to sign transactions | secp256k1 ECDSA |
| Transaction validation | Anyone can spend anyone's funds | UTXO model with sig verification |
| Mining difficulty adjustment | Chain halts or inflates | Epoch-based retargeting |
| DoS protection | Spam kills the node | Rate limiting + tx fee floor |
| API authentication | Open RPC = attack surface | Flagged — not implemented, noted below |
| Merkle proofs / SPV | Can't verify without full chain | Merkle tree on tx set |
| Replay protection | Tx reusable across forks | Chain ID in signature scope |
| Structured logging / observability | Blind in production | `pino` structured logs + metrics endpoint |
| Graceful shutdown | Corrupt writes on SIGTERM | Drain + flush on signal |
| Configuration management | Hardcoded values = ops nightmare | ENV + config file |

**Explicitly flagged gaps I am NOT implementing (and why):**
1. **RPC authentication / TLS** — Requires cert management infrastructure outside scope; flag for ops team
2. **Full UTXO set indexing** — Implemented in-memory with LevelDB persistence; a production system would want a dedicated UTXO database
3. **Peer discovery via DHT** — Using static peer list + simple gossip; production needs a DHT (Kademlia) or DNS seeds
4. **Slashing / finality** — This is PoW only; PoS/BFT finality is a separate design
5. **Mempool fee market** — Basic fee floor only; EIP-1559-style base fee not implemented

### `src/crypto/hash.js`

```javascript
import { createHash } from 'crypto';

/**
 * All hashing goes through this module.
 * Centralising it means we can swap SHA-256 for a different
 * primitive (e.g. Blake3) in one place.
 */

/**
 * Double-SHA256, matching Bitcoin's block/tx hashing convention.
 * Returns a hex string.
 */
export function hash256(data) {
  const first  = createHash('sha256').update(data).digest();
  const second = createHash('sha256').update(first).digest('hex');
  return second;
}

/**
 * Single SHA-256, used for Merkle tree internals and
 * anywhere we don't need the double-hash security margin.
 */
export function sha256(data) {
  return createHash('sha256').update(data).digest('hex');
}

/**
 * RIPEMD-160(SHA-256(data)) — standard address derivation.
 */
export function hash160(data) {
  const sha = createHash('sha256').update(data).digest();
  return createHash('ripemd160').update(sha).digest('hex');
}

### `src/crypto/merkle.js`

```javascript
import { sha256 } from './hash.js';

/**
 * Binary Merkle tree over an array of hex-string leaves.
 *
 * Production note: This implementation is vulnerable to the
 * CVE-2012-2459 duplicate-leaf attack present in early Bitcoin.
 * We guard against it by refusing trees with duplicate txids.
 */
export class MerkleTree {
  constructor(leaves) {
    if (!leaves || leaves.length === 0) {
      this.root = sha256('empty');
      this.layers = [];
      return;
    }

    // Guard: duplicate txids in a block are invalid
    const unique = new Set(leaves);
    if (unique.size !== leaves.length) {
      throw new Error('MerkleTree: duplicate leaves detected — invalid block');
    }

    this.layers = [leaves];
    this.root   = this._build(leaves);
  }

  _build(nodes) {
    if (nodes.length === 1) return nodes[0];

    const next = [];
    for (let i = 0; i < nodes.length; i += 2) {
      const left  = nodes[i];
      // If odd number of nodes, duplicate the last one.
      // This is safe here because we already rejected duplicate txids above.
      const right = nodes[i + 1] ?? nodes[i];
      next.push(sha256(left + right));
    }

    this.layers.push(next);
    return this._build(next);
  }

  /**
   * Generate a proof (array of {hash, position} objects) for a leaf at index.
   * Verifiable without the full tree — enables SPV clients.
   */
  getProof(index) {
    const proof = [];
    let idx = index;

    for (let i = 0; i < this.layers.length - 1; i++) {
      const layer     = this.layers[i];
      const isRight   = idx % 2 === 1;
      const siblingIdx = isRight ? idx - 1 : idx + 1;
      const sibling   = layer[siblingIdx] ?? layer[idx]; // duplicate rule

      proof.push({ hash: sibling, position: isRight ? 'left' : 'right' });
      idx = Math.floor(idx / 2);
    }

    return proof;
  }

  /**
   * Verify a Merkle proof without the full tree.
   */
  static verify(leaf, proof, root) {
    let computed = leaf;
    for (const { hash, position } of proof) {
      computed = position === 'left'
        ? sha256(hash + computed)
        : sha256(computed + hash);
    }
    return computed === root;
  }
}
```

### `src/crypto/wallet.js`

```javascript
import elliptic from 'elliptic';
import { hash160, sha256 } from './hash.js';

const EC = new elliptic.ec('secp256k1');

/**
 * Wallet — key generation, signing, verification, and address derivation.
 *
 * PRODUCTION GAPS (explicitly flagged):
 * 1. Private keys are held in memory as hex strings.
 *    Production: use HSM, encrypted keystore (BIP-38), or hardware wallet.
 * 2. No BIP-32 HD derivation — each wallet is a single keypair.
 *    Production: implement BIP-32/44 for account management.
 * 3. No BIP-39 mnemonic support.
 * 4. Address format is simplified (hash160 of pubkey, hex-encoded).
 *    Production: add Base58Check or Bech32 encoding.
 */
export class Wallet {
  constructor(privateKeyHex = null) {
    this.keyPair = privateKeyHex
      ? EC.keyFromPrivate(privateKeyHex, 'hex')
      : EC.genKeyPair();

    this.privateKey = this.keyPair.getPrivate('hex');
    this.publicKey  = this.keyPair.getPublic('hex');  // uncompressed, 130 chars
    this.address    = Wallet.publicKeyToAddress(this.publicKey);
  }

  static publicKeyToAddress(publicKeyHex) {
    // Standard: RIPEMD160(SHA256(pubkey))
    return hash160(Buffer.from(publicKeyHex, 'hex'));
  }

  /**
   * Sign arbitrary data (hex string of the data's hash).
   * Returns DER-encoded signature as hex.
   */
  sign(dataHash) {
    const sig = this.keyPair.sign(dataHash);
    return sig.toDER('hex');
  }

  /**
   * Verify a signature against a public key and data hash.
   * Static — verifiers don't need a private key.
   */
  static verify(dataHash, signatureHex, publicKeyHex) {
    try {
      const key = EC.keyFromPublic(publicKeyHex, 'hex');
      return key.verify(dataHash, signatureHex);
    } catch {
      return false;
    }
  }

  toJSON() {
    // NEVER serialise the private key in logs or API responses.
    return { address: this.address, publicKey: this.publicKey };
  }
}

### `src/core/transaction.js`

```javascript
import { hash256 } from '../crypto/hash.js';
import { Wallet }  from '../crypto/wallet.js';
import { config }  from '../config/index.js';

/**
 * Transaction model — UTXO-based.
 *
 * Each input references a previous output (txid + output index).
 * Each output specifies a recipient address and amount.
 * The signature covers: inputs, outputs, networkId, and timestamp
 * to provide replay protection across forks and time.
 *
 * PRODUCTION GAP: This is a simplified UTXO model.
 * A full implementation needs:
 * - Script system (P2PKH, P2SH, P2WPKH) for programmable outputs
 * - Segregated witness for transaction malleability fix
 * - Locktime / sequence numbers for time-locked transactions
 */
export class Transaction {
  constructor({ inputs, outputs, timestamp, signature, publicKey, txid }) {
    this.inputs    = inputs;    // [{ txid, outputIndex }]
    this.outputs   = outputs;   // [{ address, amount (BigInt) }]
    this.timestamp = timestamp ?? Date.now();
    this.signature = signature ?? null;
    this.publicKey = publicKey ?? null;

    // txid is computed deterministically from content
    this.txid = txid ?? this._computeTxid();
  }

  /**
   * The data committed to by the signature.
   * Includes networkId for replay protection.
   */
  _signingPayload() {
    return JSON.stringify({
      inputs:    this.inputs,
      outputs:   this.outputs.map(o => ({ address: o.address, amount: o.amount.toString() })),
      timestamp: this.timestamp,
      networkId: config.networkId,
    });
  }

  _computeTxid() {
    const payload = this._signingPayload() + (this.signature ?? '');
    return hash256(payload);
  }

  /**
   * Sign this transaction with a wallet.
   * Must be called before the transaction is broadcast.
   */
  sign(wallet) {
    const payload = this._signingPayload();
    const payloadHash = hash256(payload);
    this.signature = wallet.sign(payloadHash);
    this.publicKey = wallet.publicKey;
    this.txid = this._computeTxid();
    return this;
  }

  /**
   * Verify the signature is valid for this transaction's content.
   */
  verifySignature() {
    if (!this.signature || !this.publicKey) return false;
    const payload = this._signingPayload();
    const payloadHash = hash256(payload);
    return Wallet.verify(payloadHash, this.signature, this.publicKey);
  }

  /**
