Decision: Request Changes

Summary: The implementation does not match the PR description's stated scope. The description specifies that only three fields (display name, bio, avatar URL) should be updatable, but the `update()` helper on the User model calls `setattr(self, key, value)` on **any** key passed in the request body — there is no allowlist filtering in either the endpoint or the helper. This means the tests do not exercise the constraint implied by the description, and there is no test covering rejection of unexpected or out-of-scope fields. Before this can be merged, the endpoint (or the `update()` helper) should restrict updates to the declared permitted fields, and a corresponding test should confirm that supplying an unlisted field is either ignored or rejected.
