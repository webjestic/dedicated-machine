Credential strength is not the operative variable.

I ran twelve experiments on this. I know exactly where I was sitting when the pattern became impossible to ignore.

Two personas. Same credential. Same seniority. Same "cannot help but catch every bug" framing. Dramatically different outputs.

One finds the obvious issues. The other finds the race condition that only fires under specific timing, the infrastructure failure mode three layers below the code. Not because it has more experience. Because it arrives at the first token with a different search space.

Here's what the twelve experiments isolated: the operative variable isn't credential strength. It isn't compulsion intensity. It's the density of domain-specific vocabulary adjacent to the "You" anchor.

"Senior software engineer" activates a broad, shallow semantic neighborhood — web development, systems, databases, deployment, all pulling simultaneously, none dominating. The model generates something that performs like senior engineer output without running deep in any particular direction.

"You review distributed systems code for production readiness; before approving any merge, you check whether the write is fenced, whether the TTL arithmetic holds under GC pause, whether the commit boundary is where the failure mode actually lives" — that's a gravity well. Every clause is a mechanism token that pre-weights the model's attention toward the specific failure-mode cluster before the first task token is processed. Like the difference between pointing at a rack of tools and handing someone a torque wrench already set to spec — one gesture names a category, the other loads a constraint. One prompt sits flat on the page. The other has load-bearing weight before the task even starts.

The experiments tested compulsion framing directly. "Constitutionally unable to miss a bug." "Cannot help but catch every issue." Sounds forceful. I ran the numbers.

The data was unambiguous: compulsion framing without domain vocabulary collapses to the same floor as a basic dispositional persona. Add domain vocabulary and trait framing outperforms compulsion framing. The ceiling lifts. The floor disappears entirely.

Compulsion is a passenger. Domain vocabulary is the engine.

The practical distinction runs like this. Orientation vocabulary points toward a domain — "production readiness," "best practices," "thoroughness." Mechanism vocabulary enters it. "Whether two workers can both believe they hold the same lock." "Whether the TTL expires under a GC pause." Orientation produced 0/10 detection on target findings across all twelve experiments. Mechanism produced 10/10. Not 7. Not 9. Ten.

The difference isn't subtle. Uninitialized pointer versus live address.

Every credential you've ever written is orientation vocabulary. It names a category. It doesn't name a mechanism. That's the whole problem, and it's been the whole problem from the first prompt you ever thought was working.

The question the data doesn't answer here is why the specificity threshold exists where it does — the article has it.
