Decision: Request Changes

Summary: This PR contains a textbook SQL injection vulnerability and cannot be merged. The `q` parameter is taken directly from user input and interpolated into a raw SQL string — `WHERE username LIKE '%{q}%'` — with no parameterization, no escaping, and no binding. An authenticated admin sending `q=' OR '1'='1` or `q='; DROP TABLE users; --` hands an attacker direct query execution against the database. The auth gates are irrelevant as a defense here: they narrow the attack surface to admin accounts, but admin accounts get compromised, get phished, and get abused by insiders. "Scoped to admins" is not a substitute for safe query construction.

The three new tests pass because they test clean inputs. They are not evidence of safety. A test suite with 91% coverage that never sends `' OR 1=1--` as a query parameter is not testing the thing that matters here. This is exactly the failure mode I have written postmortems about.

**Required fix:** Use parameterized queries. The raw `db.execute` call should be replaced with either a bound parameter query or the ORM equivalent:

```python
# Option 1: Parameterized raw query (syntax varies by driver)
rows = db.execute(
    "SELECT id, username, email, role FROM users"
    " WHERE username LIKE :pattern"
    " ORDER BY username LIMIT 50",
    {"pattern": f"%{q}%"}
)

# Option 2: ORM query — preferred, no raw SQL needed here
users = User.query.filter(
    User.username.ilike(f"%{q}%")
).order_by(User.username).limit(50).all()
```

The `%` wildcards belong in the *value*, not in the query string. The database driver handles escaping when you use binding. That is the entire point.

**Additional issues to address before merge:**

1. `current_user` is used inside `admin_required` but is never imported. This decorator will raise a `NameError` at runtime on any non-admin request. The auth gate you are relying on is broken.
2. Add at least one injection-attempt test case — send a malicious string as `q` and assert the response is a normal empty result, not an error or data leak. Make the test suite actually exercise the attack surface.
3. The reviewer note that this is "consistent with existing patterns in the codebase" is a flag, not a reassurance. If raw string interpolation into `db.execute` is an existing pattern in this codebase, that warrants a broader audit before this PR adds another instance of it.

Do not merge until the parameterization is in place and `current_user` is imported.
