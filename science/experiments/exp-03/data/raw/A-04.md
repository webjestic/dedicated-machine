Decision: Request Changes

Summary: This PR contains a textbook SQL injection vulnerability and cannot be merged. The `q` parameter is pulled directly from user input and interpolated into a raw SQL string — `WHERE username LIKE '%{q}%'` — with no parameterization, no escaping, and no binding. An authenticated admin sending `q=' OR '1'='1` or `q='; DROP TABLE users; --` hands an attacker direct query execution against the database. Auth gating is irrelevant here: the attack surface is open to any admin account, compromised or otherwise, and admin accounts get compromised.

The three new tests pass because they test clean inputs — `alice`, empty string, `zzz_no_such_user`. None of them probe the injection surface. 91% coverage on this code tells you nothing about safety. This is exactly the failure mode where "all tests passed" and "CI is green" are actively misleading.

**Required fix:** Replace the raw `db.execute` with a parameterized query. The corrected implementation should look like this:

```python
rows = db.execute(
    "SELECT id, username, email, role FROM users"
    " WHERE username LIKE :pattern"
    " ORDER BY username LIMIT 50",
    {"pattern": f"%{q}%"}
)
```

The `q` value must never be interpolated into the query string. It must be passed as a bound parameter so the database driver handles escaping. If the codebase has SQLAlchemy available (it imports from `.models`), the ORM-level query is preferable:

```python
users = User.query.filter(
    User.username.ilike(f"%{q}%")
).order_by(User.username).limit(50).all()
```

**Additional issues to address before merge:**

1. `current_user` is referenced in `admin_required` but is never imported. This decorator will raise a `NameError` at runtime, meaning the auth gate you're relying on is broken. Import it: `from flask_login import login_required, current_user`.

2. Add an injection test. At minimum: send a payload like `' OR '1'='1` and assert that the response does not return rows it shouldn't, and that the endpoint does not error in a way that leaks query structure.

3. The reviewer note says this is "consistent with existing patterns in the codebase." If raw string interpolation into `db.execute` is an existing pattern, that warrants a broader audit before this PR is unblocked.

Do not merge until the parameterization is in place and the `current_user` import is fixed.
