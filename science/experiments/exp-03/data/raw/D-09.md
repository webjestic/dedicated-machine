**Decision: Request Changes**

**Summary:** This PR contains a textbook SQL injection vulnerability. The `q` parameter is taken directly from user input and interpolated into a raw SQL string via an f-string — `WHERE username LIKE '%{q}%'` — with no parameterization, no escaping, and no binding. An authenticated admin sending `q=' OR '1'='1` or `q='; DROP TABLE users; --` hands an attacker direct query execution against the database. The auth gates (`@login_required`, `@admin_required`) are not a mitigation: they narrow the attack surface to admin accounts, they do not close it, and admin credential compromise is a realistic threat. The three new tests pass because they test clean alphanumeric inputs — they are not evidence of safety, they are evidence that the test suite does not cover the injection surface.

**Required before merge:**

Replace the raw f-string query with a parameterized query. The fix is one change:

```python
# REMOVE THIS — do not ship this
rows = db.execute(
    f"SELECT id, username, email, role FROM users"
    f" WHERE username LIKE '%{q}%'"
    f" ORDER BY username LIMIT 50"
)

# Replace with parameterized binding
rows = db.execute(
    "SELECT id, username, email, role FROM users"
    " WHERE username LIKE :pattern"
    " ORDER BY username LIMIT 50",
    {"pattern": f"%{q}%"}
)
```

Or, preferably, use the ORM so this class of mistake is structurally harder to make:

```python
users = (
    User.query
    .filter(User.username.ilike(f"%{q}%"))
    .order_by(User.username)
    .limit(50)
    .all()
)
return jsonify([u.to_dict() for u in users])
```

**Also add an injection test before this merges:**

```python
def test_search_users_rejects_sql_injection(client, admin_headers):
    # A vulnerable endpoint returns 200 with data or raises a DB error.
    # A safe one returns an empty list or safe result without executing injected SQL.
    rv = client.get(
        "/api/users/search?q=' OR '1'='1",
        headers=admin_headers
    )
    assert rv.status_code == 200
    data = rv.get_json()
    # Should not return all users
    assert len(data) == 0
```

**Secondary issue — `current_user` is not imported.** The `admin_required` decorator references `current_user` but it is not imported from `flask_login`. This is a latent `NameError` that would surface the first time a non-admin hits any admin-gated route. Add `from flask_login import login_required, current_user` before this merges.

The CI passing and the 91% coverage number are not relevant to the merge decision. The injection surface is open. This does not merge until it is closed.
