Decision: Request Changes

Summary: The endpoint has several issues that need to be addressed before merging. The most critical is the use of synchronous `requests.post` inside an `async` route handler, which will block the event loop under load — this should be replaced with `httpx.AsyncClient` or equivalent. Additionally, the `message` field on `NotificationRequest` accepts arbitrary, unbounded user-controlled input with no length cap or sanitization, which is a concern given this endpoint is reachable during checkout. The failure test also patches the base `Exception` class rather than `requests.RequestException`, meaning it doesn't actually validate the intended error handling path.

**Specific issues to resolve:**

1. **Blocking I/O in async context** — `requests.post` is synchronous. In a FastAPI async handler this blocks the entire event loop. Replace with `httpx.AsyncClient` and `await client.post(...)`.

```python
# Replace this
import requests
response = requests.post(...)

# With this
import httpx
async with httpx.AsyncClient() as client:
    response = await client.post(...)
```

2. **Unbounded `message` field** — No length constraint on user-supplied input. A malicious or misconfigured caller could send arbitrarily large payloads downstream to the internal service.

```python
from pydantic import BaseModel, Field

class NotificationRequest(BaseModel):
    user_id: str = Field(..., max_length=64)
    order_id: str = Field(..., max_length=64)
    message: str = Field(..., max_length=512)
```

3. **Hardcoded internal service URL** — `"https://notifications.internal/send"` is hardcoded. This should be pulled from environment config or a settings object to support environment parity and secret management.

```python
from app.config import settings
url = settings.notification_service_url
```

4. **Weak error handling test** — The failure test patches `Exception` directly, bypassing the `requests.RequestException` catch block. This means the test passes for the wrong reason and doesn't validate the actual error path.

```python
# Use the correct exception type
from requests.exceptions import RequestException
with patch("api.notifications.requests.post",
           side_effect=RequestException("Connection refused")):
```

5. **No authentication on the endpoint** — There is no auth dependency on `/notify`. Given this is a government contracting environment, unauthenticated endpoints that trigger external service calls are not acceptable.

```python
@router.post("/notify", response_model=NotificationResponse, dependencies=[Depends(verify_token)])
```

6. **`notification_id` key access is unguarded** — `data["notification_id"]` will raise a `KeyError` if the upstream service returns an unexpected response shape. Add a `.get()` with a fallback or explicit validation.

```python
notification_id = data.get("notification_id")
if not notification_id:
    raise HTTPException(status_code=502, detail="Invalid response from notification service")
```
