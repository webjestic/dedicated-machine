Decision: Request Changes

Summary: The endpoint has several issues that need to be addressed before merging. The most critical is the use of synchronous `requests.post` inside an `async` route handler, which will block the event loop under load — this should be replaced with `httpx.AsyncClient` or equivalent. Additionally, the `message` field on `NotificationRequest` is unconstrained, meaning a caller can send arbitrarily large or malformed payloads directly to the internal service, and there is no authentication or authorization on the endpoint itself, which is a significant concern given our contracting context.

---

### Detailed Findings

#### 🔴 Critical — Blocking I/O in Async Handler

```python
# Current — blocks the event loop
async def send_notification(request: NotificationRequest) -> NotificationResponse:
    response = requests.post(...)
```

```python
# Required — use async HTTP client
import httpx

async def send_notification(request: NotificationRequest) -> NotificationResponse:
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "https://notifications.internal/send",
            json={...},
            timeout=5.0,
        )
```

Using `requests` (a synchronous library) inside an `async` def will starve the event loop on any concurrent traffic. This is a correctness and performance defect, not just a style issue.

---

#### 🔴 Critical — No Authentication or Authorization

The `/notify` endpoint is completely open. Any caller with network access can trigger notifications on behalf of arbitrary users. At minimum this needs:

- A verified caller identity (API key, JWT, or mTLS depending on deployment context)
- Ideally a dependency-injected auth guard

```python
from fastapi import Depends, Security
from fastapi.security import APIKeyHeader

API_KEY_HEADER = APIKeyHeader(name="X-Internal-API-Key")

async def verify_internal_key(api_key: str = Security(API_KEY_HEADER)):
    if api_key != settings.INTERNAL_API_KEY:
        raise HTTPException(status_code=403, detail="Forbidden")

@router.post("/notify", dependencies=[Depends(verify_internal_key)])
async def send_notification(...):
    ...
```

#### 🟡 Medium — Unconstrained `message` Field

```python
# Current — no bounds
message: str
```

```python
# Required — enforce reasonable limits
from pydantic import Field

message: str = Field(..., min_length=1, max_length=500)
```

Without this, a caller can forward arbitrarily large strings to the internal notification service. Depending on how that service handles input, this is at minimum a resource abuse vector.

---

#### 🟡 Medium — Hardcoded Internal Service URL

```python
"https://notifications.internal/send"
```

Service URLs must come from configuration, not be hardcoded. This makes environment promotion (dev → staging → prod) error-prone and untestable in isolation.

```python
from api.config import settings

response = await client.post(settings.NOTIFICATION_SERVICE_URL, ...)
```

---

#### 🟡 Medium — Error Logging May Leak Sensitive Data

```python
logger.error("Notification service error: %s", e)
```

Exception messages from `requests` / `httpx` can include full URLs, headers, and response bodies. Scrub or wrap before logging in a production context:

```python
logger.error("Notification service error for order %s: %s", request.order_id, type(e).__name__)
```

---

#### 🟠 Minor — Test Catches `Exception` Instead of `requests.RequestException`

```python
# Test patches with base Exception
side_effect=Exception("Connection refused")
```

The handler only catches `requests.RequestException`. The test is not actually exercising the real error path — it passes coincidentally. Update the test to use `requests.exceptions.ConnectionError` or the `httpx` equivalent after the client swap.

---

#### 🟠 Minor — `notification_id` Access Without Key Guard

```python
notification_id=data["notification_id"]
```

If the external service returns a 200 but omits `notification_id`, this raises an unhandled `KeyError` that will surface as a 500. Use `.get()` with a fallback or validate the response shape explicitly.

```python
notification_id = data.get("notification_id")
if not notification_id:
    raise HTTPException(status_code=502, detail="Invalid response from notification service")
```
